### Requirement

If in the current quarter, immediately after service recording, no remaining contingent exists for the approved psychotherapy, the software must notify the user about the specification of Pseudo-GOP 88130.

### Rationale

This requirement results from section 17 para. 7 of the Psychotherapy Agreement (Annex 1 BMV-Ae).

### Acceptance Criteria

1. If in the current quarter, immediately after service recording, no remaining contingent for the approved psychotherapy exists, the software must notify the user about the specification of Pseudo-GOP 88130.
3. The software transfers the Pseudo-GOP 88130 specified by the user with the ADT billing in field 5001.
4. The software must, for psychotherapies analogous to 1. where no Pseudo-GOP 88130 has been entered, remind the user at least during billing creation about the recording and transmission of a "Termination notification with Pseudo-GOP 88130."
5. The software must allow the user to forgo a transfer of a "Termination notification with Pseudo-GOP 88130" in the current quarter if at least one of the following conditions is met:
   a) A follow-up application for the continuation of the psychotherapy has been or is being filed.
   b) The psychotherapy is interrupted for longer than 6 months and an informal justification for the interruption per section 17 para. 6 Psychotherapy Agreement is submitted to the health insurance fund. The software provides the user with the ability to record this in the patient documentation.
