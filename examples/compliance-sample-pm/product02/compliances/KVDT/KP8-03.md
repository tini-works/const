### Requirement

The software must ensure that for each Hybrid-DRG, an operations code according to the OPS master data file is transmitted.

### Rationale

Hybrid-DRG services must always be coded with an official OPS code.

### Acceptance Criteria

1. The software ensures that for each Hybrid-DRG service (FK 5027), at least one OPS code is transmitted in FK 5035.
2. The software must ensure the following for the entered OPS code:
   a) The OPS code must exist in the OPS master data file (XML element `../opscode_liste/opscode/@V`).
   b) The OPS code must still be valid (the date must not be before or after the validity period of the XML element `../opscode_liste/opscode/gueltigkeit/@V`).
   c) If the entered OPS code is defined in the master data file with a side localization (`../opscode_liste/opscode/kzseite/@V="J"`), the system must request the side localization from the user.
      i. The system must suggest the corresponding side localizations for selection to the user.
      ii. The system transfers the side localization specification in FK 5041.

### Condition

Implementation of Hybrid-DRG billing according to the record type HDRG.
