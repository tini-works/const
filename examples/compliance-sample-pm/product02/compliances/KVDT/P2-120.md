### Requirement

If insurance card data records are read from an insurance card, the system must treat the data of the insurance card in the same quarter as "official" data. These data must be stored unchanged and transmitted with the billing.

### Rationale

Through a modification of read insurance card data records from a valid insurance card, rejections of the billing data record by the health insurers could occur in the subsequent billing process.

### Acceptance Criteria

1. After successful reading of an insurance card data record from an insurance card according to the specifications of the "Technische Anlage zur Anlage 4a" and the generation of a read-in date according to requirement [P2-140](P2-140.md), the following data of an insurance card have "official" character:

   | Personal Insurance Data | Protected Insurance Data | Street Address | PO Box Address | General Insurance Data |
   |---|---|---|---|---|
   | Geburtsdatum | Besondere Personengruppe | PLZ | PostfachPLZ | Versichertenart |
   | Vorname | DMP_Kennzeichnung | Ort | PostfachOrt | WOP(-Kennzeichen) |
   | Name | | Straße | Postfach | Versicherungsschutz Beginn (JJJJMMTT) |
   | Geschlecht | | Hausnummer | PostfachWohnsitzlaendercode | Versicherungsschutz Ende (JJJJMMTT) |
   | Vorsatzwort | | Anschriftenzusatz | | |
   | Namenszusatz | | Wohnsitzlaendercode | | |
   | Titel | | | | |
   | Versichertennummer bzw. Versicherten_ID | | | | |
   | Kostentraegername | | | | |
   | Kostentraegerkennung | | | | |

   a) The following requirements must be observed during the reading process:
      i. [P2-105](P2-105.md)
      ii. [P2-135](P2-135.md)
      iii. [P2-136](P2-136.md)
      iv. [P2-170](P2-170.md)
      v. [KP2-310](KP2-310.md)
      vi. [P2-470](P2-470.md) (Exception to "officiality" when reading a KVK for the gender field)
      vii. Specifications of Section 2.2.2.1 Zuordnung des Kostenträgers (VKNR, IK und Krankenkassenname)
2. The software must display, store, and transmit the fields with "official" character unchanged and on a per-patient basis with the billing.
   a) The software must exclude modification of fields with official character through both UI and programmatic means.
3. The "officiality" of the data listed in Table 4 is valid until the end of the quarter in which the insurance card was read.
4. If a successful reading process of an insurance card has occurred, the software must prevent the automatic adoption of insurance card data into the substitute procedure (cf. Section 2.2.3.1 Definition Ersatzverfahren) for the purpose of data modification.

### Notes

For Acceptance Criterion (2): Even if the patient data are no longer current, they must be transmitted unchanged in the billing.

When issuing panel physician forms, attention must be paid to which address data to use for the issuance. A distinction is made between referral and billing forms and the other panel physician forms (see P7-81/P7-82, "Anforderungskatalog Formularbedruckung" [KBV_ITA_VGEX_Anforderungskatalog_Formularbedruckung]).
