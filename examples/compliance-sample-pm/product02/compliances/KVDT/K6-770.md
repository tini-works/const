### Requirement

If (check) conditions of the EBM master data file are violated, the software may perform system-side changes to GNRs documented by the user as part of service documentation in real time.

### Rationale

Physicians may be supported during service recording to avoid incorrect entries and to increase the quality of billing data.

### Acceptance Criteria

1. If (check) conditions (section `//gnr/bedingung/` of the EBM master data file) are violated, the software may perform system-side changes to GNRs documented by the user as part of service documentation in real time -- at the time of recording services.
2. The user must be able to recognize the system-side changes to the service documentation at the time of the changes and reverse them.

### Notes

System-side changes (deleting, adding, replacing GNRs) regarding the services documented by the user may **not** be performed through real-time checks when rule mechanisms of the EBM master data file (section `//gnr/regel/` of the EBM master data file) are the basis. Effects of this rule checking on the service documentation may only be of an **advisory nature**.
