### Requirement

The software must enable the user to capture and manage the point value for calculating the estimated physician's fee on the patient receipt.

### Rationale

The contractual basis is § 305 (2) SGB V.

The point value serves as the basis for calculating the estimated physician's fee on the patient receipt per [P2-890](P2-890.md). In the normal case, the point value corresponds to the orientation value.

### Acceptance Criteria

1. The software pre-populates the entry field with the currently valid orientation value as a default value automatically.

### Notes

If a GOP is valued only in Euro, this Euro value is to be used.
