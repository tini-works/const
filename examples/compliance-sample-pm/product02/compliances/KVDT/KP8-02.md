### Requirement

The software must ensure that when entering/selecting the ICD-10-GM code for the primary diagnosis (FK 6009) and/or the secondary diagnosis (FK 6011) for Hybrid-DRG billing, only terminal ICD-10-GM codes are used. Furthermore, at least one primary code must be used.

### Rationale

According to the ICD-10-GM coding guidelines, coding must be as specific as possible. This means that the terminal (final) key numbers of the ICD-10-GM must be used. There is no exception for coding in the context of Hybrid-DRGs.

According to the ICD-10-GM coding guidelines and the German Coding Directives (DKR) (D012 Multiple coding), secondary codes must be combined with so-called primary codes and cannot stand alone.

### Acceptance Criteria

1. The software ensures that the entered ICD-10-GM code for the primary diagnosis (6009) and/or secondary diagnosis(es) (6011) exists in the ICD-10-GM master data file [SDICD] (the XML element `../diagnosen_liste/diagnose/icd-code/@V`).
2. The software ensures that for coded primary and/or secondary diagnoses, at least one primary code is specified. If only ICD codes with the notation markers (*) or (!) (so-called secondary codes) are present, the software must ensure the following:
   a) Generate a notice that specifying a primary code is required.
   b) Prevent the transfer of the exclusively secondary code into the billing file.
3. The software ensures that the ICD-10-GM code does not end with "-". If an ICD-10-GM code ends with "-", the software must ensure the following:
   a) Generate a notice indicating that the ICD-10-GM code is not terminal and therefore may not be used for billing.
   b) Prevent the transfer of the non-terminal ICD-10-GM code into the billing file.

### Condition

Implementation of Hybrid-DRG billing according to the record type HDRG.
