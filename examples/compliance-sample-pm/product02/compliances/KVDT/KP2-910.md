### Requirement

If OP services from EBM chapters 31.2 or 36.2 are captured as a GOP (GOP; FK 5001), the system must require the specification of OPS codes (OPS-Code; FK 5035) as a billing justification.

For documentation purposes and also as a billing justification, the specification of OPS codes for OP services from EBM chapters 31.2 or 36.2 must generally be made.

### Rationale

For services of outpatient surgery, by law since 01.04.2005, the procedure codes in their respectively valid version must also be applied in the outpatient setting for documentation and billing purposes.

### Acceptance Criteria

If in the [EBM-Stammdatei] for a GOP under `.../gnr_zusatzangaben/gnr_zusatzangaben_liste/gnr_zusatzangabe` the value `V="5035"` (OPS code) is stored and the OPS code is selected as the justification type, the system must ensure the following:

1. The system must require the user to provide an OPS code.
2. The system must verify the entered OPS code for correctness and transmit it in the billing file under **FK 5035**.
   a) Whether an OPS code applies as a billing justification for a GOP is determined by the assignment of an OPS code to a GOP in the [EBM-Stammdatei] under `.../begruendungen_liste/ops_liste`.
   b) If the OPS code entered by the user is defined in the [EBM-Stammdatei] with a laterality "L" or "R" (under `.../begruendungen_liste/ops_liste/.../ops/seite`), the system must request the user to specify the laterality. The system must present the corresponding lateralizations for selection. The system transmits the laterality specification in FK 5041.
   c) If the OPS code entered by the user is defined in the [EBM-Stammdatei] with a laterality "P" (under `.../begruendungen_liste/ops_liste/.../ops/seite`), the OPS code must be transmitted twice (in two field identifiers FK 5035) automatically by the software in the billing. The system must also automatically specify the lateralizations "L" once and "R" once. The system transmits the laterality specification in FK 5041.
3. If none of the OPS codes entered by the user in the [EBM-Stammdatei] is assigned to the GOP (cf. acceptance criterion 2 a)), the system must display a corresponding notice.
   a) The system must allow adoption of the OPS code(s) into the billing file in FK 5035 despite the notice.
   b) If additionally OPS code 5-983 is specified for an OPS code that is not among the OPS codes defined in the [EBM-Stammdatei] for a service number, the system must waive displaying the notice.

### Condition

Exempt from implementation of this requirement are software systems without APK.

### Notes

The OPS codes available for outpatient contractual physician care as billing justification for a service are visible in the GOS through the deposit of the corresponding OPS codes.

For simultaneous procedures, OPS codes must be documented for all performed procedures.

Irrespective of this, all currently valid OPS codes are provided in the OPS-Stammdatei [SDOPS] by KBV and can generally be specified by physicians for documentation and billing of a service.

Acceptance criterion 2c) first applies to treatment cases from 1 January 2026.
