## KP2-102 — Card Data of Rejected Health Insurance Card Displayed in Copyable Format

| Field | Value |
|-------|-------|
| **ID** | KP2-102 |
| **Type** | Conditional Mandatory |
| **Source** | KVDT v6.06 |
| **Section** | 2.2.1.2 Krankenversichertenkarte als Berechtigungsnachweis zur Inanspruchnahme ärztlicher Leistungen ab 01.01.2015 ungültig |
| **Status** | TBC |
| **Goals** | — |
| **Verification Method** | — |
| **Matched by** | — |
| **Refer** | [KP2-101](KP2-101.md), [P2-140](P2-140.md) |

### Requirement

The software allows the user to view the card data of a rejected Krankenversichertenkarte (corresponding to [KP2-101](KP2-101.md)) in "copyable" format.

### Rationale

The user must have the ability to view the rejected card data in "copyable" format, e.g. to avoid data loss after adopting insurance card data records from a mobile card terminal.

### Acceptance Criteria

1. Upon rejection of the Krankenversichertenkarte corresponding to [KP2-101](KP2-101.md), the user can have the rejected card data displayed in "copyable" format. An automatic adoption of the card data into the "Ersatzverfahren" (substitute procedure) (cf. Section 2.2.3.1 Definition Ersatzverfahren) is not permitted.
2. An "Einlesedatum" (read-in date) according to [P2-140](P2-140.md) must not be generated in this case.

### Condition

Excluded from implementation of this requirement are software systems without APK.
