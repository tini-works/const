### Requirement

The software must, for name and/or address changes in the current quarter that deviate from the data on the insurance card, be capable of storing and managing the deviating data separately, but must not transfer these into the billing file.

### Rationale

For billing purposes, the official data from the insurance card must be used, since the insured person/contractual data must also arrive at billing as they are registered with the health insurers.

### Acceptance Criteria

1. If name and/or address changes arise for an insured person in the current quarter that deviate from the information on the insurance card:
   a) the software must store and manage these name and address data separately
   b) the software must not transfer this data for billing purposes. The software must transfer the official data from the insurance card for billing purposes (see [P2-120](P2-120.md))
