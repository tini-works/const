### Requirement

The software must check and ensure that a documented OPS is not transmitted without side localization when the OPS code requires a side localization.

### Rationale

Per § 295, Absatz 1 SGB V, it is regulated that the coding of performed operations and other procedures must be done using the respectively valid version of the procedure code. The provided version of the [SDOPS] represents the respectively valid version of the procedure code for application in the scope of § 295 SGB V.

### Acceptance Criteria

1. The software checks and ensures that a documented OPS code is not transmitted without side localization when the OPS requires a side localization:
   a) When an OPS code to be documented is defined in the OPS master data file [SDOPS] with the identifier `../opscode_liste/opscode/kzseite` equal to "J", the software must ensure that **one** side localization is transmitted for the OPS code in the billing via FK 5041. (see also [KP2-910](KP2-910.md), 2))
   b) When an OPS code to be documented is defined in the OPS master data file [SDOPS] with the identifier `../opscode_liste/opscode/kzseite` equal to "N", the software must ensure that **no** side localization is transmitted via FK 5041 in the billing.

### Condition

Excluded from the implementation of this requirement are software systems without APK.
