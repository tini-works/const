### Requirement

New cost units must be able to be added as temporary Kostentraeger-Stammsaetze to the KT-Stammdatei/database.

### Acceptance Criteria

1. Required data for billing alongside the valid IK are:
   a) **Abrechnungs-VKNR** (`/kostentraeger/@V`)
   b) **KT-Abrechnungsbereich** (`/kostentraeger/kt_abrechnungsbereich_liste/kt_abrechnungsbereich/@V`)
   c) **Kassenname zur Bedruckung** (`/kostentraeger/kt_abrechnungsbereich_liste/kt_abrechnungsbereich/bedruckungsname/@V`)
   d) **Gebuehrenordnung** (`/kostentraeger/gebuehrenordnung/@V`)
2. Further data for the manually created cost unit substitute is also needed when the user in the same service quarter wants to search the KT-Stammdatei again for the cost unit by Kassensuchname `/kostentraeger/bezeichnung/suchname/@V` and Ortssuchname `/kostentraeger/ortssuchname_liste/ortssuchname/@V`.
3. For manual entry of the Abrechnungs-VKNR, the following system checks must be performed:
   a) Length: 5 digits
   b) Type: numeric
   c) Format rule: 017
4. Temporary data records from the KT-Stammdatei **from the previous quarter** may not be automatically assigned to the new KT-Stammdatei if the corresponding IKs of the temporary data records are **not** contained in the updated KT-Stammdatei.
5. The user must have the ability to modify temporary master data records or amend KTABs.
6. If a temporarily created IK in the user's KT-Stammdatei is not contained in the subsequent version of the KT-Stammdatei, this IK may be automatically assigned to the new KT-Stammdatei.
