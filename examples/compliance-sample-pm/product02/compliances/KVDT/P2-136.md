### Requirement

The software must comply with the transformation specifications for the name of the cost carrier according to the requirements of the "Technische Anlage zur Anlage 4a".

### Rationale

The contractual basis of this requirement is Anlage 4a BMV-Ä as well as the Technische Anlage zu Anlage 4a (BMV-Ä).

### Acceptance Criteria

1. If a KVK is read, the software must adopt the content of the field "KrankenKassenName" (Objekttag '80') according to the specifications of the "Technische Anlage zur Anlage 4a" [KBV_ITA_VGEX_Mapping_KVK] into field 4134. (cf. also [P2-105](P2-105.md))
2. If an eGK is read, the software must adopt the content of the element `..//Name` according to the specifications of the "Technische Anlage zur Anlage 4a" into field 4134, observing the following priority rules:
   - **Priority 1:** Content of element `UC_AllgemeineVersicherungsdatenXML/Versicherter/Versicherungsschutz/Kostentraeger/AbrechnenderKostentraeger/Name`
   - **Priority 2:** Content of element `UC_AllgemeineVersicherungsdatenXML/Versicherter/Versicherungsschutz/Kostentraeger/Name` (cf. also [P2-105](P2-105.md))
3. The software must not use the cost carrier name from the insurance card for issuance of panel physician forms; instead, the rules of requirements [P2-210](P2-210.md), [P2-220](P2-220.md), [P2-260](P2-260.md), or [P2-270](P2-270.md) must be observed.
4. The software allows the user to enter this data manually as well.

### Notes

The KVDT field 4134 ("Kostentraegername") can only be present in a record type 010x when an insurance card was actually read.

The cost carrier name must always be transmitted when reading an insurance card -- independent of the apparent subgroup. The functions [P2-120](P2-120.md) and [P2-136](P2-136.md) take precedence over context rule 777 of the KVDT data record description.

The restriction to specific apparent subgroups in rule 777 results from requirement [P2-140](P2-140.md) (8).
