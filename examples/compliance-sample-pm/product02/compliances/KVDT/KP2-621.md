### Requirement

According to the EBM billing provisions for genetic services, these must be coded with HGNC gene symbols when billed.

### Rationale

The requirement results from the EBM billing provisions for genetic services.

### Acceptance Criteria

1. The software system must maintain the data of the HGNC key table (OID) in the system.
2. The user must have the ability to search for an HGNC gene symbol and, if applicable, automatically adopt the selected value as a billing justification related to a service into field 5077 (HGNC gene symbol).
3. The software system must validate user-entered HGNC gene symbols during input (real-time check) for existence against the values of the HGNC key tables (element `/key/@DN`).
   a) If the HGNC gene symbol to be documented is **not** contained in the key table, the following applies:
      - i. The system displays a warning that the entered HGNC gene symbol does not exist in the key table.
      - ii. In this case, the software system must ensure that the replacement value "999999" is transmitted in the respective KVDT field 5077.
      - iii. When entering the replacement value "999999" in field 5077, the user must provide a free text for the gene name in field 5078 (Gen-Name).
   b) If the documented HGNC gene symbol is contained in the key table, the user must **not** have the ability to populate field 5078.
4. The software transmits with the billing the HGNC gene symbol specified by the user for a service in field 5077.
5. The software transmits with the billing the gene name specified by the user for a service ("Gen-Name") in field 5078, provided a value is present.
6. If in the [EBM-Stammdatei] for a GOP under `.../gnr_zusatzangaben/gnr_zusatzangaben_liste/gnr_zusatzangabe` the value V="5077" (HGNC gene symbols) is stored and the HGNC gene symbol is selected as the justification type, the software system must ensure the following:
   a) The software system must require the user to provide at least one HGNC gene symbol.
7. The contract physician or PVS manufacturer has no ability to extend the key table with additional HGNC symbols.

### Condition

Exempt from implementation of this requirement is software that is used exclusively for billing laboratory community services from EBM section 32.2 (pursuant to § 25 Abs. 2 and 3 BMV-Ä).

Software subject to this criterion exclusively supports billing of record type 0102 with form subgroup 28.
