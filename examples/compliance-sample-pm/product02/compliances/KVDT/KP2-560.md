### Requirement

The software must enable the user to input, store, and transfer the "original Auftragstext" of the referring physician from the "Auftrag" lines of Muster 6 and Muster 10.

### Rationale

Per "Richtlinien der Kassenärztlichen Bundesvereinigung für den Einsatz von IT-Systemen in der Arztpraxis zum Zweck der Abrechnung gemäß § 295 Abs. 4 SGB V", § 1 Datenverarbeitungstechnisches Abrechnungsverfahren, Absatz 1, all billing-relevant data must be electronically transferable.

Per [KBV_Erläuterung_Vordrucke], see Muster 6, Punkt 8, Muster 10, Punkt 13: the physician executing the order may only perform the services specified under "Auftrag".

Legal basis is further § 297 Abs. 2 SGB V.

### Acceptance Criteria

1. If a referral per Muster 6 with marked checkbox "Ausführung von Auftragsleistungen" (= Satzart 0102 with Scheinuntergruppe 21) or per Muster 10 (= Satzart 0102 with Scheinuntergruppe 27) is captured, the software must request from the user the adoption of the original Auftragstext from the "Auftrag" lines of the respective Muster.
2. The software transfers the Auftragstext specified by the user into field FK 4205 (Auftrag) with the billing.
3. The software does not pre-populate the entry field with a default value (e.g. "Laboruntersuchung") nor does the billing automatically transfer a default value (e.g. "Laboruntersuchung").
4. If a referral per Muster 6 with one of (1) deviating Scheinuntergruppen is captured, the software must also enable entry of any existing Auftragstext. Transfer by the software is analogous to (2).
5. If Auftragsinformationen are imported into the software via Satzart 8215 "Auftrag" (LDT 3), the software may automatically transfer the content into field FK 4205 (KVDT) with the billing.
6. The software may support the user via selection lists as input aid under the following conditions:
   a) The software must enable the user to maintain these selection lists.
   b) The visible entry field is initially empty; no pre-set value is displayed.
   c) An entry adopted from a selection list must be modifiable by the user at any time.
