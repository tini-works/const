### Requirement

1. When insurance card data is transferred from a mobile reader (return code 9500 or 9501 at RESET_CT) into a billing system, the approval number of the mobile card terminal (tag '92' of the extended VDT) must be transmitted in the respective record type 010x of the KVDT (FK 4108), cf. also [P2-140](P2-140.md) (9) and Chapter "Mobiles Einsatzszenario" in "Mappingtabelle_KVK" [KBV_ITA_VGEX_Mapping_KVK].
2. If an approval number is already present, it is overwritten by the currently transmitted approval number (only one approval number per data record "010x").
3. The software must, when the same insurance card of the same insured person from the same health insurer of the same physician in the current quarter was first read via a mobile reader, and then a further APK is read via a stationary reader, overwrite the already existing read-in date (field FK 4109) in the respective record type "010x" (cf. also [P2-150](P2-150.md)). The software does not transmit field FK 4108 ("Zulassungsnummer") with the billing.
