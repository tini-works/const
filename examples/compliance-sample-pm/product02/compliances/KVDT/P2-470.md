### Requirement

The software must comply with the transformation rules for gender according to the requirements of the "Technische Anlage zur Anlage 4a" and allow the user to manually enter a gender specification.

### Rationale

The contractual basis of this requirement is Anlage 4a BMV-Ae as well as the "Technische Anlage zu Anlage 4a" (BMV-Ae).

### Acceptance Criteria

1. The software does not pre-populate the input field for gender (Feld 3110) with a default value.
   a) Automated determination of gender based on the first name or other identifying characteristics may be performed by the software to support the user. The user must always have the ability to overwrite the suggested value.
2. If an eGK is read, the software must adopt the content of the element `..//Geschlecht` into field 3110 according to the rules of the "Technische Anlage zur Anlage 4a" [KBV_ITA_VGEX_Mapping_KVK]. (see also [P2-105](P2-105.md))
3. If a KVK is read, the software must request the user to enter a gender specification.
4. During manual entry or Ersatzverfahren, the software must request the user to enter a gender specification.
5. The software must transfer the gender captured by the user in field 3110.
