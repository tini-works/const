### Requirement

The software must enable the Ersatzverfahren and all cases in which no insurance card is presented or can be presented, allowing input, storage, and transfer of all existing insured person data into the billing.

### Rationale

According to "Richtlinien der Kassenärztlichen Bundesvereinigung für den Einsatz von IT-Systemen in der Arztpraxis zum Zweck der Abrechnung gemäß § 295 Abs. 4 SGB V", § 1 Datenverarbeitungstechnisches Abrechnungsverfahren, Absatz 1, all billing-relevant data must be electronically transferable.

The legal basis for "Auftrags- und Konsiliaruntersuchung" is Anlage 4a BMV-Ae, Anhang 1, Punkt 3, as well as "Datenübernahme ohne persönlichen Arzt-Patienten-Kontakt" per Anlage 4a BMV-Ae, Anhang 1, Punkt 4.

The legal basis for Ersatzverfahren is Anlage 4a BMV-Ae, Anhang 1, Punkte 2.4 and 2.5 as well as Anlage 4a BMV-Ae, Anhang 1, Punkt 2.3.

3. The software notifies the user that all data listed in the table above can be captured and should be captured if available. The notification should occur without interrupting the workflow.
4. The software transfers the insured person data captured by the user per the table into the billing.

### Notes

For further processing of the above data, the same rules generally apply -- with the exception of restrictions regarding "official status" -- as for a card reading process via a card terminal.
