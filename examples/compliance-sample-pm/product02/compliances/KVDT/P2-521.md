### Requirement

The software must provide the user with the ability to incorporate treatment cases from previous quarters (so-called "Nachzüglerfälle") into the current quarterly billing.

### Rationale

For retroactive billing of treatment cases from previous quarters, users must have the ability to integrate these treatment cases into a current billing.

### Acceptance Criteria

1. If the quarter of the treatment case (FK 4101) lies before the current Abrechnungsquartal (FK 9204) and the case was not billed, the software must enable the user to bill as a Nachzüglerfall in the current quarter.
