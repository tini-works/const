## P2-501 — Creation of Additional "010x" Data Records in the Same Quarter for the Same Patient

### Requirement

The software must allow the user to create additional arbitrary 010x data records with the same and/or different Satzart, Scheinuntergruppe as needed, even if the patient was already recorded in the current quarter.

### Rationale

Due to the separate remuneration for treatments resulting from Terminvermittlungen or special constellations (e.g. open consultation hours) per § 87a Abs. 3 Satz 5 SGB V (i. d. F. des TSVG und GKV-FinStG), the billed services are to be marked by Schein.

### Acceptance Criteria

1. The user has the ability to create additional arbitrary 010x data records for a patient in the current quarter with the same and/or different Satzart, Scheinuntergruppe in the system.
2. If the insurance card is re-read, an already existing Einlesedatum in all 010x data records of the current quarter must be updated (see [P2-150](P2-150.md)).
