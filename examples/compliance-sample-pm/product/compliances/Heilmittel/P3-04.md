### Requirement

The user must be given the ability to issue a prescription based on previous prescriptions.

### Rationale

To simplify the prescribing process, it must be possible to access previous prescriptions. In doing so, all captured information from the previous prescriptions must be transferable to the new prescription.

### Acceptance Criteria

1. The user must have the ability to adopt all information from a previous prescription of the respective patient into the current prescription.
   a. The prescription history prior to October 1, 2020 must be retrievable in the software system. Access to data prior to October 1, 2020 for the purpose of creating a new prescription does not need to be implemented.
2. When adopting data from previous prescriptions, it must be ensured that these are based on the current master data.
3. All information as per [P3-02](P3-02.md) must be provided/transferred to the new prescription, except for the following:
   a. Issue date
   b. Cancellation date of the prescription for cancelled prescriptions
4. The identification of a special prescribing need or long-term remedy need must be re-verified after adoption as per [P3-18](P3-18.md).
5. After adoption of information into the current prescription, this information must be editable.

### Notes

Due to the revision of the Remedy Directive effective October 1, 2020, the information to be documented has changed significantly. For this reason, it is not mandatory to adopt information from prescriptions created based on the requirements catalog prior to version 2.0. It is nevertheless possible to adopt information from previous prescriptions using intelligent algorithms.
