### Requirement

The software system enables the user to store individual approvals of a long-term remedies need on a patient-specific basis.

### Rationale

The user must have the option to store individual approvals of a long-term remedies need pursuant to § 32 Abs. 1a SGB V, which are issued by the respective health insurance fund upon patient request, on a patient-specific basis and to use them within the scope of prescriptions.

### Acceptance Criteria

1. The user must be given the option in the software to store and process one or more long-term approvals issued by the health insurance fund per patient (e.g., patient number or insured person ID) with the following information:
   a. One or two ICD-10-GM code(s): The entered ICD-10-GM codes must correspond to the values of the ICD-10-GM master data file from KBV (SDICD XML Element ../diagnose/icd_code/@V).
   b. Diagnosis group: The entered diagnosis groups must correspond to the values of the remedies master data file (SDHM XML Element ../kapitel/diagnosegruppe/@V).
   c. One or up to three priority remedies and one supplementary remedy (optional entry): The entered remedies must correspond to the values of the remedies master data file (SDHM XML Element ../diagnosegruppe/heilmittelverordnung/vorrangiges_heilmittel_liste/vorrangiges_heilmittel/name/@V or SDHM XML Element ../diagnosegruppe/heilmittelverordnung/ergaenzendes_heilmittel_liste/ergaenzendes_heilmittel/name/@V or SDHM XML Element ../diagnosegruppe/heilmittelverordnung/standardisierte_heilmittel_kombination/name/@V) according to the selected diagnosis group.
   d. Valid-until date (validity date of the approval with the option "valid indefinitely").
   e. Cost bearer identifier of the approving cost bearer according to the KVDT reference design as per Technischer Anlage zur Anlage 4a (BMV-A) [KBV_ITA_VGEX_Mapping_KVK].
2. When capturing the cost bearer, the software must pre-populate the cost bearer identifier from the current insured person data.
3. In addition, the user must be given the option to store and process further entries per free text.
4. The software automatically evaluates the validity of the approval upon accessing the approval data, based on the validity date of the approval.
5. The software automatically evaluates the validity of the approval upon accessing the approval data, based on a possible change of cost bearer (cf. P2-220 [KBV_ITA_VGEX_Anforderungskatalog_KVDT]).
6. The software automatically evaluates the validity of the approval upon accessing the approval data, based on a possible dissolution or invalidity of the cost bearer (cf. P2-230 and P2-P260 [KBV_ITA_VGEX_Anforderungskatalog_KVDT]).

### Notes

The patient-specific approval of long-term remedies need is only valid for one specific health insurance fund. This means that approvals are retained during a health insurance fund merger.

It is recommended to offer a possibility to adopt the required information from an already documented approval by means of an action for the respective patient.
