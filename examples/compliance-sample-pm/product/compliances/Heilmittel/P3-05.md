### Requirement

The software checks whether the prescription has been completed in accordance with the present requirements catalog and provides notifications regarding fields not properly filled.

### Rationale

In order to meet the legislative objective and avoid formal errors, it must be ensured that the user is notified when the prescription is not completely or formally correctly filled in. The user must also have the option to print out incompletely filled prescriptions, since handwritten completion is permissible in individual cases, e.g., in the case of home visits.

### Acceptance Criteria

1. Upon completion of the prescription process (printing or saving), the user is to be notified, based on whether all fields have been filled according to the requirements, via a notification display.
2. This notification must contain information about which information does not conform to the requirements of the requirements catalog.
3. If none of the exception situations from [P3-08](P3-08.md), [P3-16](P3-16.md), [P3-24](P3-24.md), and [P3-11](P3-11.md) apply, completion of the prescription process must be permitted after notification display.
4. The following fields on the prescription are defined as mandatory fields, with exceptions defined in the referenced requirements to be considered accordingly:
    - Data of the personal data field per Technical Annex to Annex 4a (BMV-Ae) [KBV_ITA_VGEX_Mapping_KVK].
    - Remedy area (Physiotherapy, Podological Therapy, Voice-, Speech-, Language- and Swallowing Therapy, Occupational Therapy, or Nutritional Therapy)
    - Treatment-relevant diagnosis(es) (per [P3-12](P3-12.md))
    - Diagnosis group (per [P3-07](P3-07.md))
    - Key symptom (per [P3-08](P3-08.md))
    - Remedy per catalog specification (per [P3-19](P3-19.md) and [P3-16](P3-16.md) Blank prescription)
    - Treatment units (per [P3-11](P3-11.md))
    - Therapy frequency (per [P3-13](P3-13.md))
    - Home visit (per [P3-14](P3-14.md))
