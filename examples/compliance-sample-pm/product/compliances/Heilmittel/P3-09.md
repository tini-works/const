### Requirement

The software enables control over the status of the prescription, determining whether the prescription must be assigned to a new or an existing prescription case.

### Rationale

With the definition of a prescription case per § 7 of the HeilM-RL, it is possible to electronically define the status of the prescription, i.e., whether it is a new or existing prescription case. In § 7 paragraph 1 of the HeilM-RL, unambiguous criteria for review are defined. The prescription case and the guiding treatment quantity refer (per § 7 para. 3 of the HeilM-RL) to the respective prescribing physician.

### Acceptance Criteria

The software determines, based on the following defined criteria, whether the prescription is to be assigned to a new or an existing prescription case (see Figure 2).

The following dependencies apply:

1. Existence of a prior prescription for a female or male patient
2. Time interval to the prior prescription
3. Selected diagnosis group, and
4. Three-digit ICD-10-GM code (i.e., the first three digits of the ICD-10-GM code are identical) compared to the first prescription of the prescription case (initial prescription)
   a. At least one ICD-10-GM code of the prescription must be identical in the first three digits with one of the ICD-10-GM codes of the first prescription of the prescription case.
5. LANR of the physician who issued the prior prescription

**Flowchart (Figure 2 -- Assignment of the Prescription to a Prescription Case):**

The decision flow proceeds as follows: Start by checking whether a prior prescription exists. If no prior prescription exists, the result is "New prescription case." If a prior prescription exists, check whether the issue date is older than 6 months. If older than 6 months, check whether the LANR is identical to the prior prescription -- if no, result is "New prescription case"; if yes, continue. Next check whether the diagnosis group and ICD-10-GM code (three-digit) are identical to the initial prescription -- if no, result is "New prescription case"; if yes, check whether the prior prescription is a blank prescription. If the prior prescription is a blank prescription, check whether a blank prescription is being issued -- if yes, result is "Same prescription case"; if no, result is "New prescription case." If the prior prescription is not a blank prescription, check whether a blank prescription is being issued -- if yes, result is "New prescription case"; if no, result is "Same prescription case."

### Notes

In the context of a prescription where two ICD-10-GM codes are specified as treatment-relevant diagnoses, it is possible that this prescription may be assigned to two different prescription cases, provided a prior prescription exists with one of the two treatment-relevant diagnoses.

When a prior prescription exists where two ICD-10-GM codes are specified as treatment-relevant diagnoses, it is possible that this prescription may be assigned to two different prescription cases, provided a follow-up prescription is issued with only one of the two treatment-relevant diagnoses.
