### Requirement

There is a mandatory requirement to use the valid BVB/LHM master data file. The data records of the BVB/LHM master data file must not be modified in content.

### Rationale

Pursuant to § 73 Abs. 10 Satz 1 SGB V, the software must provide, in addition to the information from the Remedies Directive, also information about special prescription needs pursuant to § 106b Absatz 2 Satz 4 SGB V. This is ensured through the information in the BVB/LHM master data file. The information in the BVB/LHM master data file consists of agreement contents between GKV-SV and KBV, i.e., contents of the Remedies Directive of the G-BA, which are binding and therefore may only apply in unaltered form.

### Acceptance Criteria

1. The BVB/LHM master data file from KBV must be integrated into the software and used in the respectively valid version.
2. The software provides the user with the option to view the status of the integrated master data file.
3. The data of the BVB/LHM master data file must not be modified in content.

### Notes

The display of the status of the integrated master data file can be implemented by showing the user the file name or the validity information of the file (SDHMA XML Element //ehd/header/service_tmr/@V), indicating which data source the used data originates from.

This requirement does not mandate that software systems must work directly with the structure of the provided file. The data of the file may be transferred to another format, e.g., in the form of databases.
