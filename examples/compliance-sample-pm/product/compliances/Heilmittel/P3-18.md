## P3-18 — Criteria for Prescriptions Based on Special Prescription Need / Long-Term Remedy Need

| Field | Value |
|-------|-------|
| **ID** | P3-18 |
| **Type** | Mandatory |

| **Status** | TBC |
| **Goals** | — |

| **Matched by** | — |
| **Refer** | [P3-01](P3-01.md), [P3-09](P3-09.md) |

### Requirement

The software provides the capability to process criteria of a special prescription need and a long-term remedy need and to inform the user accordingly, i.e., to perform the necessary documentation.

### Rationale

The software for prescribing remedies must provide the physician, in addition to the information of the Heilmittel-Richtlinie including the regulations on long-term remedy need, information about special prescription needs according to § 106b paragraph 2 sentence 4 SGB V, according to § 73 SGB V. Additional specifications/notes or restrictions are to be displayed.

According to § 106 b para. 2 sentence 5 SGB V, regional special prescription needs may also be agreed upon between the state associations of health insurance funds and the Associations of Statutory Health Insurance Physicians.

### Acceptance Criteria

1. If a selected ICD-10-GM code fulfills the criteria for a special prescription need and/or long-term remedy need per the BVB-/LHM-Stammdatei (SDHMA XML-Element ../verordnungsbedarf/icd_code/@V corresponds to the selected code and the XML-Element ../verordnungsbedarf/heilmittel_liste/heilmittel/sekundaercode/@V is not defined), the Heilmittel-Diagnosegruppen are to be visibly marked separately in the entry dialog upon selection, indicating those where it is a special supply need/long-term remedy need.
    a. If the patient's age does not fulfill an optional existing age criterion (per SDHMA XML-Element ../verordnungsbedarf/heilmittel_liste/heilmittel/untere_altersgrenze/@V and/or ../verordnungsbedarf/heilmittel_liste/heilmittel/obere_altersgrenze/@V), this marking must be omitted.
    b. If, due to an incomplete or missing date of birth of the patient, the fulfillment of an age condition cannot be unambiguously assessed (e.g., if the date of birth is unknown), this marking must be omitted.
    c. For special prescription needs that apply only regionally (per SDHMA XML-Element ../verordnungsbedarf/geltungsbereich_kv/@V), this marking must be omitted.

    Note: Within a prescription where two ICD-10-GM codes are specified as treatment-relevant diagnosis, it is possible that the prerequisites for a special prescription need or long-term remedy need are fulfilled, each defined via one of the ICD-10-GM codes. In this case, the Heilmittel-Diagnosegruppen are marked, indicating those where it is a special supply need/long-term remedy need, whether defined via one or the other ICD-10-GM codes.

2. If two selected ICD-10-GM codes fulfill the criteria for a special prescription need and/or long-term remedy need per the BVB-/LHM-Stammdatei (SDHMA XML-Elements ../verordnungsbedarf/icd_code/@V and ../verordnungsbedarf/heilmittel_liste/heilmittel/sekundaercode/@V), the Heilmittel-Diagnosegruppen are to be visibly marked separately in the entry dialog upon selection, indicating those where it is a special supply need/long-term remedy need.

    Note: The selection order of the two ICD-10-GM codes is irrelevant.
    a. If the patient's age does not fulfill an optional existing age criterion (per SDHMA XML-Element ../verordnungsbedarf/heilmittel_liste/heilmittel/untere_altersgrenze/@V and/or ../verordnungsbedarf/heilmittel_liste/heilmittel/obere_altersgrenze/@V), this marking must be omitted.
    b. If, due to an incomplete or missing date of birth of the patient, the fulfillment of an age condition cannot be unambiguously assessed (e.g., if the date of birth is unknown), this marking must be omitted.
    c. For special prescription needs that apply only regionally (per SDHMA XML-Element ../verordnungsbedarf/geltungsbereich_kv/@V), this marking must be omitted.

3. If a selected ICD-10-GM code matches the single stored ICD-10-GM code from a valid patient-specific long-term approval, the Heilmittel-Diagnosegruppen are to be visibly marked separately in the entry dialog upon selection, indicating those associated with these approvals.

4. If two selected ICD-10-GM codes match the two stored ICD-10-GM codes from a valid patient-specific long-term approval, the Heilmittel-Diagnosegruppen are to be visibly marked separately in the entry dialog upon selection, indicating those associated with these approvals.

    Note: Within a prescription where two ICD-10-GM codes are specified as treatment-relevant diagnosis, it is possible that both ICD-10-GM codes match two different valid patient-specific long-term approvals. In this case, the Heilmittel-Diagnosegruppen are marked, indicating those where it is a long-term remedy need, whether established by one or the other valid patient-specific long-term approval.

5. If the prerequisites for at least a special prescription need and/or long-term remedy need are fulfilled via one or two ICD-10-GM codes and the diagnosis group (per SDHMA XML-Element ../kapitel/diagnosegruppe_liste/diagnosegruppe/@V), the prescription is to be visibly marked as special prescription need and/or long-term remedy need in the entry dialog upon selection of the diagnosis group.
    a. If the patient's age does not fulfill an optional existing age criterion (per SDHMA XML-Element ../verordnungsbedarf/heilmittel_liste/heilmittel/untere_altersgrenze/@V and/or ../verordnungsbedarf/heilmittel_liste/heilmittel/obere_altersgrenze/@V), this marking must be omitted.
    b. If, due to an incomplete or missing date of birth of the patient, the fulfillment of an age condition cannot be unambiguously assessed (e.g., if the date of birth is unknown), this marking must be omitted.
    c. For special prescription needs that apply only regionally (per SDHMA XML-Element ../verordnungsbedarf/geltungsbereich_kv/@V), this marking must be omitted.

    Note: Within a prescription where two ICD-10-GM codes are specified as treatment-relevant diagnosis, it is possible that both the prerequisites for a special prescription need as well as a long-term remedy need are fulfilled, each defined via one of the ICD-10-GM codes and the diagnosis group. In this case, the prescription is marked accordingly as special prescription need and long-term remedy need.

6. If the prerequisites for a special prescription need or long-term remedy need are fulfilled solely via the diagnosis group (per SDHMA XML-Element ../kapitel/diagnosegruppe_liste/diagnosegruppe/@V), the prescription is to be visibly marked as special prescription need or long-term remedy need in the entry dialog, independent of the selected ICD-10 code, upon selection of the diagnosis group.
    a. If the patient's age does not fulfill an optional existing age criterion (per SDHMA XML-Element ../verordnungsbedarf/heilmittel_liste/heilmittel/untere_altersgrenze/@V and/or ../verordnungsbedarf/heilmittel_liste/heilmittel/obere_altersgrenze/@V), this marking must be omitted.
    b. If, due to an incomplete or missing date of birth of the patient, the fulfillment of an age condition cannot be unambiguously assessed (e.g., if the date of birth is unknown), this marking must be omitted.
    c. For special prescription needs that apply only regionally (per SDHMA XML-Element ../verordnungsbedarf/geltungsbereich_kv/@V), this marking must be omitted.

7. For special prescription needs that apply only regionally, the markings per points (1), (2), (5), and (6) are to be displayed to those users who are active in the respective KV area (per SDHMA XML-Element ../verordnungsbedarf/geltungsbereich_kv/@V).

8. If the prescription establishes a special prescription need that is subject to the restriction of an acute event (SDHMA XML-Element ../verordnungsbedarf/heilmittel_liste/heilmittel/zeitraum_akutereignis/@V is present), the user is to be enabled to enter the patient-related acute event date in the format DD.MM.YYYY and to display it on subsequent prescriptions.
    a. If the special prescription need is not subject to the restriction of an acute event, no entry must be possible.
    b. If no acute event is entered by the user, this is not to be treated as a special prescription need. The marking of the prescription as special prescription need per points (5) and (6) must be omitted.
    c. If an acute event is entered by the user, the system must check whether the current prescription date falls within the permissible time period after the acute event (SDHMA XML-Element ../verordnungsbedarf/heilmittel_liste/heilmittel/zeitraum_akutereignis/@V). If this is not the case, the marking of the prescription as special prescription need per points (5) and (6) must be omitted.

    Note: Within a prescription where two ICD-10-GM codes are specified as treatment-relevant diagnosis, it is possible that the prescription establishes two special prescription needs, both subject to the restriction of an acute event. In this case, the user is enabled to enter patient-related acute events for two diagnoses assigned to the individual diagnoses and these are displayed on subsequent prescriptions.

9. For prescriptions that establish at least a long-term remedy need and/or at least a special prescription need and are subject to further specifications, the corresponding notes per the BVB-/LHM-Stammdatei (SDHMA XML-Element ../verordnungsbedarf/heilmittel_liste/heilmittel/hinweistext/@V) are to be displayed to the user without interrupting the workflow.

    Note: Within a prescription where two ICD-10-GM codes are specified as treatment-relevant diagnosis, it is possible that the prescription establishes multiple long-term remedy needs and/or special prescription needs, the further specifications of which are subject to multiple corresponding notes. In this case, the user is shown multiple corresponding notes.

10. If patient-specific long-term approvals per function [P3-01](P3-01.md) are stored in the software and are valid, prescriptions are to be visibly marked as long-term approved in the entry dialog.
    a. A valid patient-specific approval of long-term remedy need exists for a prescription to be issued when the information present in [P3-01](P3-01.md) point (1) regarding the approval:
        - one or two ICD-10-GM codes and the diagnosis group match the corresponding information of the prescription,
        - all remedies specified on the prescription match the existing approval information, if at least one remedy is specified there,
        - the validity end date has not been reached at the time of the prescription,
        - no change of the cost bearer without a merger relationship to the previous cost bearer has taken place, and no dissolution or invalidity of the approving cost bearer is determined.

### Notes

For special prescription needs with the specification "laengstens (...) nach Akutereignis" (at most (...) after acute event), the prescription date of the first remedy prescription assigned to the ICD-10-GM code or to the assigned ICD-10-GM code combination for the consideration and the deadline calculation within the scope of the economic efficiency review according to § 106b para. 1 SGB V is authoritative. A pre-population with the date of the first prescription of this prescription case (per [P3-09](P3-09.md)) is recommended.
