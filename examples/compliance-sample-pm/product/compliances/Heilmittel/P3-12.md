### Requirement

The user must be given the ability to enter information in the "Treatment-Relevant Diagnosis/Diagnoses" field.

### Rationale

The user should be offered different options for specifying the treatment-relevant diagnosis/diagnoses and, if applicable, further findings as per § 13 of the HeilM-RL. This ensures the provision of patient-specific information to the remedy provider.

### Acceptance Criteria

1. The software adopts the corresponding ICD-10 plain text/texts from the ICD-10-GM Master Data File (SDICD XML-Element ../diagnose/bezeichnung/@V) based on the selection of the ICD-10 code(s) (per [P3-06](P3-06.md)) into the "Treatment-Relevant Diagnosis/Diagnoses" field.
2. The user must have the following options for populating the "Treatment-Relevant Diagnosis/Diagnoses" field, replacing or supplementing the ICD-10 plain text(s) from point (1):
   a. Adoption of texts on diagnoses and/or findings from the patient documentation,
   b. Entry of free text.
3. The user must be able to freely combine all options for populating the "Treatment-Relevant Diagnosis/Diagnoses" field from point (2) a. and b.
4. After adoption and/or entry of texts into the "Treatment-Relevant Diagnosis/Diagnoses" field, the user must have the ability to edit them.
5. When specifying multiple texts, these must be separated by a ";" separator character.
