### Requirement

There is a mandatory requirement to use the valid blank prescription master data file. The data records of the blank prescription master data file must not be modified in content.

### Rationale

The blank prescription master data file provided by GKV-SV contains information about indications for which a prescription pursuant to § 13a of the HeilM-RL is possible, made available in electronically readable form. The information in the blank prescription master data file constitutes an agreement of the GKV-SV pursuant to § 125a SGB V, which is binding and therefore may only apply in unaltered form.

### Acceptance Criteria

1. The blank prescription master data file from GKV-SV must be integrated into the software and used in the respectively valid version.
2. The user must have the option in the software to view the status of the integrated master data file.
3. The data of the blank prescription master data file must not be modified in content.

### Notes

The display of the status of the integrated master data file can be implemented by showing the user the file name or the validity information of the file (SDG XML Element //ehd/header/service_tmr/@V), indicating which data source the used data originates from.

Changes to the blank prescription master data file are published by GKV-SV at the following deadlines at https://www.gkv-datenaustausch.de/leistungserbringer/aerzte/aerzte.jsp and can be retrieved by PVS manufacturers at the earliest on November 15, 2020:

- Version valid for Q1 of the year: publication by November 15 of the previous year
- Version valid for Q2 of the year: publication by February 15 of the year
- Version valid for Q3 of the year: publication by May 15 of the year
- Version valid for Q4 of the year: publication by August 15 of the year
