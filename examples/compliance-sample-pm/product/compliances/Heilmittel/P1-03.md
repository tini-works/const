### Requirement

The user must be given the option to set default values in the software's user settings for individual prescription data (fields on the prescription form).

### Rationale

To reduce the effort involved in issuing a prescription, the user should have the option to pre-populate certain prescription data (fields).

### Acceptance Criteria

1. The software allows the user to set default values only for the following prescription data (fields on the prescription form):
   - Home visit
   - Therapy report
   - Remedies discipline
2. In the initial software configuration at delivery, none of the aforementioned fields may have a default value set.
3. The user must actively set the default values for the fields.
4. The default values for the fields must be applicable globally (valid for all patients).
5. When creating a prescription, the user can override the default values for the fields.
