## P3-731 — Mapping of Form Selection in an Electronic Prescription

### Requirement

For an electronic prescription, the form selection made must be mapped via a predefined combination of FHIR elements.

### Rationale

For the electronic prescription, the underlying paper-based form plays no role. However, for the pharmacy at the time of dispensing, it must be apparent which prescribing situation is involved. This is mapped via a combination of existing identifiers.

### Acceptance Criteria

The prescribing software must, for an electronic prescription, store the following information in the FHIR dataset depending on the form selection per [P3-730](P3-730.md):

**FHIR Mapping:**

| | Coverage Profile | Medication Profile |
|---|---|---|
| **FHIR Profile** | KBV_PR_FOR_Coverage | KBV_PR_ERP_Medication_PZN / KBV_PR_ERP_Medication_Ingredient / KBV_PR_ERP_Medication_Compounding / KBV_PR_ERP_Medication_FreeText |
| **Element** | Coverage.type.coding.code | Medication.extension(https://fhir.kbv.de/StructureDefinition/KBV_EX_ERP_Medication_Category).valueCoding.code |

**Mapping of form selection to FHIR element codes:**

| Paper-based Form Selection | Coverage.type.coding.code | Medication_Category code | Additional Notes |
|---|---|---|---|
| BtM-Rezept* | GKV | 01 | |
| T-Rezept* | GKV | 02 | |
| Arzneiverordnungsblatt (Muster 16) | Depending on insurance status / prescribing situation: GKV, BG, SKT, or UK | 00 | If a statutorily insured patient requests cost reimbursement per § 25a Abs. 5 BMV-Ae, the value "Kostenerstattung" is to be specified in element payor.display in the profile KBV_PR_FOR_Coverage. If a statutorily insured patient receives a pharmaceutical prescription per § 25a Abs. 4 BMV-Ae, the value "ohne Versicherungsnachweis" is to be specified in element payor.display in the profile KBV_PR_FOR_Coverage. |
| Privat-Rezept | Depending on insurance status: SEL or PKV | 00 | |

### Notes

\* In stage 1 of the electronic prescription, BtM-Rezept and T-Rezept are not yet supported.
