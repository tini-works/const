### Requirement

The prescribing software must implement prescriptions of products based on the pharmaceutical master data ([P2-110](P2-110.md)).

### Rationale

After searching for and selecting a product, prescribing that product is the next step in the prescribing process.

### Acceptance Criteria

1. The prescribing software must implement a prescription of pharmaceuticals and other products included in the pharmaceutical supply pursuant to § 31 SGB V from the pharmaceutical master data.
2. When prescribing from the pharmaceutical master data, modification of the data adopted on the basis of the PZN by the contracted physician is not permitted.
3. Acceptance criterion 2 does not apply to prescriptions of pharmaceuticals with ATC code N07BC (agents for the treatment of opioid dependence). In this case, the prescribing software must ensure that when the data adopted on the basis of the PZN is modified, the PZN reference is automatically removed.
4. For a paper-based prescription, only a multiple of the package size of one product (one PZN) may be prescribed per prescription line. Prescribing different package sizes in one prescription line is not permitted.
   - The multiple of the package size may only be specified as a whole number.
   - If only a portion of a finished pharmaceutical product to which a PZN is assigned (Merkmal 023) is to be prescribed (partial quantity), the prescribing software directs the user to the need for a free-text prescription in accordance with [P3-624](P3-624.md).
     - Acceptance criterion 3 of requirement [P3-624](P3-624.md) does not apply.
5. For an electronic prescription, only a multiple of the package size of one product may be prescribed in one prescription. Prescribing different package sizes in one prescription is not permitted.
   - When no number of packages has been selected, the field (MedicationRequest.dispenseRequest.quantity.value in profile KBV_PR_ERP_Prescription or SupplyRequest.quantity.value in profile KBV_PR_ERP_PracticeSupply) is to be populated with 1.
   - The multiple of the package size may only be specified as a whole number.
   - If only a portion of a finished pharmaceutical product to which a PZN is assigned (Merkmal 023) is to be prescribed (partial quantity), the prescribing software directs the user to the need for a free-text prescription in accordance with [P3-624](P3-624.md).
     - Acceptance criterion 3 of requirement [P3-624](P3-624.md) does not apply.
6. When prescribing a multiple of a package size, an additional marking (e.g., an additional exclamation mark) is not required and must therefore not be applied.
7. For printing a paper-based prescription, [**P3-721**](P3-721.md) must be observed.
