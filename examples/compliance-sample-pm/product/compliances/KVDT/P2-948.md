### Requirement

The software must offer the user a function for searching and identifying existing patient master data in the patient master data store.

### Rationale

For data protection reasons, it may occur in the context of KV-physician communication that no patient name is used, but instead only the insured person's ID is used.

### Acceptance Criteria

1. The software must offer the user the ability to search for existing patients in the patient master data store by entering at least the following search criteria:
   a) Insured person's ID
   b) Insurance number
   c) SKT additional information
   d) Date of birth

### Notes

The use of additional search criteria such as last name, first name, etc. can also be offered to the user.
