### Requirement

The software must ensure that once an eGK has been read for an insured person of the Bundespolizei Heilfürsorge (BPol), no further outdated KVKs can be read.

### Rationale

The cost carrier Bundespolizei Heilfürsorge plans to replace the old KVKs with current eGKs.

### Acceptance Criteria

1. When an eGK is first read for an insured person of the BPol (VKNR = 74860), the software subsequently rejects the reading of an outdated KVK (also for VKNR 27860) in the current quarter and all subsequent quarters, and informs the user that the insured person already has an eGK.

### Condition

Excluded from implementation of this requirement are software systems without APK.

### Notes

As part of the eGK rollout, the BPol has introduced the new VKNR = 74860. Until the complete exchange of all KVKs within a transition period, the VKNR 27860 remains valid in the cost carrier master data file.

The cost carrier with VKNR 27860 will be marked as invalid in the cost carrier master data file as of October 1, 2025. Please also note requirement [P2-230](P2-230.md).
