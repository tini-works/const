### Requirement

1. The following fields of the KT master or KT change data record with the field identifiers:
   a) `/kostentraeger/@V`,
   b) `/kostentraeger/kt_abrechnungsbereich_liste/kt_abrechnungsbereich/bedruckung sname/@V`,
   c) `/kostentraeger/ik_liste/ik/@V`,
   d) `/kostentraeger/gebuehrenordnung/@V`,
   e) `/kostentraeger/kt_abrechnungsbereich_liste/kt_abrechnungsbereich/@V`,
   f) `/kostentraeger/bezeichnung/suchname/@V`,
   g) `/kostentraeger/ortssuchname_liste/ortssuchname/@V`,
   h) `/kostentraeger/gueltigkeit/@V`
   i) `/kostentraeger/ik_liste/ik/gueltigkeit/@V`,
   j) `/kostentraeger/existenzbeendigung/aufnehmender_kostentraeger/@V`,
   k) `/kostentraeger/unz_kv_geltungsbereich_liste/unz_kv_geltungsbereich/@V` **and**
   l) `/kostentraeger/kt_abrechnungsbereich_liste/kt_abrechnungsbereich/gueltigkei t/@V`

   have an "official" character, meaning they must not be changeable by the user (display fields only!).

2. The address fields (`/kostentraeger/adresse_liste/adresse/*`) can be changed freely while observing format specifications.
   a) Address changes of a cost carrier that are not included in a subsequent version of the KT master data file must also be retained after importing the new KT master data into the PVS.
   b) The software marks the cost carrier when manual corrections have been made and gives the user the option to restore/activate the original data according to the KT master data file.
