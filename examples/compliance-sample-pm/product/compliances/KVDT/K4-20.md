### Requirement

1. For implausible entries, **warnings** must be issued. The plausibility check is based on the following table:

| Digit(s) of Identifier | Meaning | Checks |
|---|---|---|
| 1 | Identifier for the patient's origin: 1 = NRW, 0 = other federal states | If content of digit 1 = 0, the case is to be billed directly with the responsible health insurance fund |
| 2-7 | Date of application at the health insurance fund | Check for date format; valid format is DDMMYY. Application date must be <= 1st service day |
| 8 | Running number of the day | Value range: 1, 2, ..., 9 |
| 9-15 | IK of the health insurance fund | Check via KT master file |
| 16-20 | PLZ of the issuing health insurance fund | Check via PLZ master file |
| 21-27 | Alphanumeric internal identifier of the health insurance fund | 1-7 digits |

6. However, it must also be possible to store an implausible identifier if the user confirms the entry.
