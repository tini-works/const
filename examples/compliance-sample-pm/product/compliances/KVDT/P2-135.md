### Requirement

The software must comply with the transformation specifications for the WOP (Wohnort-Kennzeichen) according to the requirements of the "Technische Anlage zur Anlage 4a" and enable the user to manually enter a WOP identifier.

### Rationale

The contractual basis of this requirement is Anlage 4a BMV-Ä as well as the Technische Anlage zu Anlage 4a (BMV-Ä).

The WOP must not be determined programmatically to avoid incorrect assignments.

### Acceptance Criteria

1. If a Krankenversichertenkarte (KVK) is read, the software must transform the content of the field "VKNR/WOP" (Objekttag '8F') according to the specifications of the "Technische Anlage zur Anlage 4a" [KBV_ITA_VGEX_Mapping_KVK] and transmit it in field 3116. (cf. also [P2-105](P2-105.md)).
   a) If no WOP identifier is coded on the KVK, field 3116 is not transmitted.
2. If an eGK is read, the software must adopt the content of the element `../WOP` according to the specifications of the "Technische Anlage zur Anlage 4a" into field 3116. (cf. also [P2-105](P2-105.md))
3. The software must provide the user during manual entry with the ability to manually enter the WOP identifier. (cf. also [P2-400](P2-400.md))
   a) The software does not pre-populate the WOP identifier input field (e.g. based on the PLZ of the insured person).
4. The software must transmit the WOP identifier entered by the user in field 3116.

### Notes

Due to storage space issues on the Krankenversichertenkarte, a dual assignment of the VKNR field (Objekttag '8F', VKNR or WOP identifier) occurred, so that alternatively a residence identifier (WOP) in the format 000nn (nn = KV region) can be contained.
