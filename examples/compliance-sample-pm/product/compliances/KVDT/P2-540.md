### Requirement

If in the current quarter the Versichertenart (FK 3108) of the same patient of the same insurer changes, the software must automatically create a new "010x" data record for billing.

### Rationale

Per § 21 Absatz 1 Satz 4 BMV-Ae, in case of a change of Versichertenart in the quarter, the Versichertenart at the beginning of the quarter is to be used for billing. For documentation of the Versichertenart change, a separation of data records takes place.

### Acceptance Criteria

1. If the Versichertenart (FK 3108) of a patient changes in the current quarter, the software must generate an additional "010x" data record.
2. The system may inform the user about the creation of a new "010x" data record.
3. When creating the new data record due to the change of Versichertenart, no changes to the already existing Einlesedatum of the insurance card (FK 4109) may take place. An update of the Einlesedatum takes place only for the newest data record.
   a) The software must store and transmit the Einlesedatum for the respective cases "true to data record" (possibly multiple times) (see [P2-150](P2-150.md)).
