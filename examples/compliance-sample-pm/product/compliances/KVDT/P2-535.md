### Requirement

If in the current quarter the Besondere Personengruppe (FK 4131) of the same patient of the same insurer changes, the software must automatically create a new "010x" data record for billing.

### Rationale

The treatment of patients designated by a Besondere Personengruppe (FK 4131) fundamentally leads to specific remuneration and/or billing regulations from the point in time when membership in the respective Besondere Personengruppe (FK 4131) becomes known, so that a separation of billing data is necessary.

### Acceptance Criteria

1. If the Besondere Personengruppe (FK 4131) of a patient changes in the current quarter, the software must generate an additional "010x" data record.
2. The software may inform the user about the creation of a new "010x" data record.
3. When creating the new data record due to the change of the Besondere Personengruppe, no changes to the already existing Einlesedatum of the insurance card (FK 4109) may take place. An update of the Einlesedatum takes place only for the newest data record.
   a) The software must store and transmit the Einlesedatum for the respective cases "true to data record" (possibly multiple times) (see [P2-150](P2-150.md)).
