### Requirement

The additional information required by the KV-Specifika master data file in record type "kvx3" in the context of ADT billing of SKT cases must only be entered by the user when the information is actually available in the medical practice (e.g., printed on a paper billing certificate, for example, printing of the SKT additional information in print line 6 in the "Insured Person No." field of the personal details form).

The data required according to the definitions in fields 9402, 9403, and 9404 of the KV-Specifika file in fields "4123", "4124", "4125", and "4126" of the ADT file are therefore not mandatory fields in the sense of the KVDT data set description for billing!

The user must be prompted by a system warning to enter the additional billing information according to FKs 9402, 9403, 9404, record type "kvx3". Further processing must be possible at any time after the warning. A "billing-preventing" error message and/or the capture and transmission or automatic generation and transmission of undefined "substitute values" is not permitted.

**Annotation:** A KV may arrange for a specification of SKT data via record type "kvx8"; however, this data is not programmatically evaluable (e.g., file reference numbers at social welfare offices are not necessarily required).
