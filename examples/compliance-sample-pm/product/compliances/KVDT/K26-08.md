### Requirement

The software adopts the data from the digital form automatically into the system and processes them as described in the acceptance criteria.

### Rationale

To support the physician in their work, the system must correctly process the contents from the received form.

### Acceptance Criteria

The system adopts the data of the digital form faithfully into the billing (see [KP2-570](KP2-570.md)). This means:

1. The user is given the option to create a new billing case (in the ADT data package with record type 0102).
2. In this billing case, no reading date is generated.
3. The insured person's data and the other data of the digital form are automatically adopted into the billing case. The rules derived from the KVDT data record description apply, such that data adoption for digital forms follows the same rules as data adoption for paper forms.
4. The data of the digital form for which no adoption rule from the KVDT data record description exists are automatically adopted on a case-by-case basis at the corresponding location in the system. The adoption rules from data adoption of paper forms apply.
5. The adopted data are editable by the user at any time.
