### Requirement

If in the current quarter the combination of Abrechnungs-VKNR (FK 4104) and Kostenträger-Abrechnungsbereich (KTAB) (FK 4106) of a patient changes, the software must automatically create a new "010x" data record for billing.

### Rationale

The combination of VKNR and KTAB uniquely identifies a cost unit.

### Acceptance Criteria

1. If the VKNR (FK 4104) - KTAB (FK 4106) combination of a patient changes in the current quarter, the software must generate an additional "010x" data record.
   a) The software must not create a new "010x" data record when only the Kostenträgerkennung (FK 4111) changes but the Abrechnungs-VKNR does not simultaneously change.
2. The software may inform the user about the creation of a new "010x" data record.
3. When creating the new data record due to an insurance switch, no changes to the already existing Einlesedatum of the insurance card (FK 4109) may take place. An update of the Einlesedatum takes place only for the newest data record.
   a) The software must store and transmit the Einlesedatum for the respective cases "true to data record" (possibly multiple times) (see [P2-150](P2-150.md)).

### Notes

Per this requirement, it must be possible for the same patient at one Betriebsstätte/Arztnummer in the same quarter to bill multiple cost units.
