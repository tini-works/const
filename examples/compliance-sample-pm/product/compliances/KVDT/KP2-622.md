### Requirement

The software system enables the user to capture, manage, and use HGNC gene symbol chains. HGNC gene symbol chains are multiple HGNC gene symbols grouped together into a group.

### Rationale

Through the use of HGNC gene symbol chains, the effort for billing documentation of extensive genetic examinations shall be reduced.

### Acceptance Criteria

1. Each HGNC gene symbol chain must have a unique identifier.
2. HGNC gene symbol chains are to be stored patient-independently in the software system.
3. When creating HGNC gene symbol chains, only valid HGNC gene symbols from the current HGNC key table may be used.
   a) That is, the software must check the HGNC gene symbols entered or selected by the user during input or selection for existence in the HGNC key table (real-time check). The HGNC gene symbols must correspond to a value of the HGNC key table (element `/key/@DN`).
4. The user has the ability in the software system to display all defined HGNC gene symbol chains.
5. The user has the ability in the software system to edit self-defined HGNC gene symbol chains:
   a) add further HGNC gene symbols to a defined HGNC gene symbol chain,
   b) remove HGNC gene symbols from a chain, or
   c) remove the entire HGNC gene symbol chain.
6. During quarter changes or when importing a new HGNC key table, the software system must automatically verify all defined HGNC gene symbol chains to ensure that only valid HGNC gene symbols are used.
   a) When the software system determines that individual HGNC gene symbols of a chain are no longer valid according to the HGNC key table, the user must be prompted to update the HGNC gene symbol chain.
