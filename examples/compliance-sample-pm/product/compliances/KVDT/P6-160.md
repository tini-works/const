### Requirement

If the data of an insurance card is the basis for billing -- regardless of whether the insurance card was read in or captured via the substitute procedure -- the software must not apply the additional information or restrictions defined with the record type "kvx3".

### Rationale

Since 01.04.2000, insurance cards have also been introduced for special population groups (e.g., BVG) of the "Other Cost Carriers" (see definition from 2.2.1.2) as a replacement for the "paper" billing certificates.

These insurance cards fundamentally do not contain any additional SKT billing information, such as a file reference number. The additional information optionally defined with the record type "kvx3" is not factually present in a medical practice and is also not relevant for billing in these special cases (see data set description SDKV, chapter explanation for record type "kvx3" [KBV_ITA_VGEX_Datensatzbeschreibung_SDKV]). The additional information and restrictions regarding permitted record types and permitted insured person types demanded with the record type "kvx3" therefore do not apply correspondingly in these cases.

### Acceptance Criteria

1. The software displays a (warning) notice for capturing the additionally required billing information or restrictions according to record type "kvx3" when:
   a) Field FK 4109 is not present **and**
   b) The serial number of the VKNR >= 800 (and the cost carrier billing area (KTAB) = 00-09) or the serial number of the VKNR < 800 and the KTAB is not equal to 00 **and**
   c) Neither field FK 3105 nor field FK 3119 is present.
