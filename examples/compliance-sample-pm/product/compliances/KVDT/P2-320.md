## P2-320 — Setting the KT-Abrechnungsbereich (KTAB; FK 4106) Depending on Besondere Personengruppe (FK 4131)

### Requirement

The system must support the user in selecting KTABs depending on the read or captured Besondere Personengruppe.

### Rationale

Treatment of patients designated by a Besondere Personengruppe leads to specific remuneration and/or billing regulations.

### Acceptance Criteria

1. The system must ensure that for identifiers of the Besonderen Personengruppen (FK 4131) the KTAB (FK 4106) may only have a specific set of values:
   a) If FK 4131 = "00", then FK 4106 may be one of the permitted values "00", "01", "02", "03", "04", "05", "06", "07", "08" or "09".
   b) If FK 4131 = "04", then FK 4106 must be either "00" or "09".
   c) If FK 4131 = "06", then FK 4106 must be either "02" or "09".
   d) If FK 4131 = "07" or "08", then FK 4106 must be either "01" or "09".
2. Automatic pre-population of FK 4106 content by the system **is possible**:
   a) If FK 4131 = "00", then automatic pre-population of FK 4106 with content "00".
   b) If FK 4131 = "04", then automatic pre-population of FK 4106 with content "00".
   c) If FK 4131 = "06", then automatic pre-population of FK 4106 with content "02".
   d) If FK 4131 = "07" or "08", then automatic pre-population of FK 4106 with content "01".
3. Automatic pre-population of FK 4106 content by the system **is required**:
   a) If FK 4131 = "09", then an automatic pre-population of FK 4106 with content "00" must occur.
4. The user must have the ability to change the pre-populated values per acceptance criteria 2 and 3.
