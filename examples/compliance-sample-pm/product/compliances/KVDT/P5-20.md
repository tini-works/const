### Requirement

The **communication record** generated by the KVDT validation module must be appended to the KVDT file (for billing of ADT, KADT, and SADT data packages) before encryption.

### Notes

When billing HDRG data packages, no communication record is appended to the billing file.

**Annotation:** After validating the KVDT file, the **KVDT validation module (XPM)** generates the so-called **communication record**. The communication record contains information about the preceding validation run. The name and location of this communication record can be configured via the "Kommunikationssatz" switch in the configuration file of the KVDT validation module, see chapter "Kommunikationssatz", KVDT-specific supplement to the KBV validation module XPM handbook [KBV_ITA_AHEX_Handbuch_Pruefmodul_KVDT].

As part of encryption, the **crypto module (XKM)** can also "append" the communication record of the KVDT file. To do this, the Komusatz.txt file (with default configuration) must be passed to the crypto module via the optional parameter -i or the Pruefinfo switch, see [KBV_ITA_AHEX_Handbuch_Kryptomodul].

When using the KBV validation assistant, the Komusatz.txt file is automatically appended with unchanged default configuration.
