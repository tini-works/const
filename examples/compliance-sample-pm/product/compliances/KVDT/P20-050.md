### Requirement

Based on the facility-individual material-analyte-GOP combinations, a further functionality is to be implemented so that a statement about the presence of the RV certificate can be managed -- for example as an options field -- with the following values:

a) Certificate present
b) Certificate not present
c) pnSD/uu

It is generally assumed that the RV certificates required for performing the analyses are present, so that all analyses are pre-assigned with the default value "Certificate present" and need only be confirmed once.

In addition, it shall be possible to also apply the other values "Certificate not present" and "pnSD/uu" collectively for all analyses with a single action.

Furthermore, it must be possible to individually mark certificates that deviate from an overall marking.

The facility-individual RV certificates must be permanently stored and editable.

Depending on the value of FK 0301 (see [KP20-030](KP20-030.md)), the RV certificates (FK 0305) are to be assigned the following default values:

| pnSD/uu-Info (FK 0301) | | Default for RV Certificate (FK 0305) | |
|---|---|---|---|
| Code | Meaning | Code | Meaning |
| 0 | no pnSD/uu | 1 | Certificate present |
| 1 | exclusively pnSD/uu | 2 | pnSD/uu analysis |
| 2 | partially pnSD/uu | 1 | Certificate present |
