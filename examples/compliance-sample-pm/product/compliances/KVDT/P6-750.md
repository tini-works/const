### Requirement

The software must provide the user with the data of the EBM master data file of their responsible KV as the primary EBM data basis.

### Rationale

The software vendors receive both the regional, KV-specific fee numbers and content-extended EBM master data files of all Kassenärztliche Vereinigungen as well as the nationwide unified EBM master data file of the KBV as part of the quarterly update.

Due to the defined EBM delivery process, however, short-term decisions of the Bewertungsausschuss may be reflected in the nationwide unified EBM master data file but not in the regional EBM master data files.

### Acceptance Criteria

1. The software provides the user with the data of the EBM master data file of their responsible KV as the primary EBM data basis.
2. If the user records a fee number as part of billing for service documentation that does not exist in the regional EBM master data file, the software can access the nationwide unified EBM master data file.
