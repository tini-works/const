### Requirement

Through appropriate organizational measures, it must be ensured that the user can use the currently valid KVDT validation module and KBV crypto module (XKM) in a timely manner for billing within their software.

### Rationale

To ensure data quality and guarantee billing processing, the KVDT validation module must be used to validate billing files.

Likewise, to ensure data protection and due to the various ways of submitting billing data, every billing file must be encrypted with the defined billing keys.

### Acceptance Criteria

1. The software ensures that the user can use the currently valid version of the KBV crypto module (XKM) in time for billing.
2. The software ensures that:
   a) For billing of ADT, KADT, and/or SADT data packages, the valid billing key "Oeffentlich_KV_VXX.pub" (working mode "Abrechnungs_Verschluesselung") is always used.
   b) For billing of the HDRG data package, the valid billing key "Oeffentlich_HDRG_VXX.pub" (working mode "HDRG_Verschluesselung") is always used.
3. The software ensures that the user can use the currently valid version of the KVDT validation module (KVDT-XPM) in time for billing.
4. Alternatively to acceptance criteria 1 and 3, the software can provide the user with the currently valid version of the KBV validation assistant in time for billing.

### Notes

If the software provides the user with the KBV validation assistant, the assistant ensures the correct use of the keys.
