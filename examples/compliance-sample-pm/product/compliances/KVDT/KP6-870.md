### Requirement

The software must check and ensure that a procedure code entered by the user exists in the current OPS master data file.

### Rationale

Per § 295, Absatz 1 SGB V, it is regulated that the coding of performed operations and other procedures must be done using the respectively valid version of the procedure code. The provided version of the SDOPS represents the respectively valid version of the procedure code for application in the scope of § 295 SGB V.

### Acceptance Criteria

1. The software checks whether the specified procedure code exists as content of the XML element `../opscode_liste/opscode/@V` of the current OPS master data file.
2. If the procedure code entered by the user is not present in the OPS master data file, the software must ensure the following:
   a) It must be apparent to the user that the procedure code does not exist in the OPS master data file and therefore may not be used for billing.
   b) Prevent the transfer of the non-existent procedure code into the billing file.

### Condition

Excluded from the implementation of this requirement are software systems without APK.
