### Requirement

To allow users to test new functions in the software or generally test software behavior, the user must have the ability to store fictitious insured persons. Billing of services for fictitious insured persons is to be prevented.

### Rationale

Health insurers that provide practices with data of fictitious insured persons for connectivity tests of TI-Fachanwendungen in production operation expect that no real contractual physician services are billed for these fictitious insured persons. To ensure that accidental billing of services for fictitious insured persons does not occur, the software should prevent the adoption of billing data of fictitious insured persons into the ADT-Abrechnungsdatensaetze.

### Acceptance Criteria

1. The software must provide the user with the ability to mark data as fictitious during manual entry of patient master data.
   a) Data marked as fictitious can use all functions of the software.
2. The software must automatically prevent the adoption of captured services for fictitious insured persons into the ADT-Abrechnungsdatensaetze within the KV-Abrechnung, without requesting user confirmation.
