### Requirement

The software supports the entry/population of the data fields defined in the record type HDRG according to the defined structure.

### Rationale

The software must provide the user with the ability to generate a file for billing that complies with the specifications of the record type HDRG.

### Acceptance Criteria

1. The software supports manual entry of the fields defined in the record type HDRG, or their automatic population, provided that data fields can be automatically filled (e.g., by importing data from read-in cards or by adopting data from master data).
   a) Compliance with rules and cardinalities is ensured by the software.

### Condition

Implementation of Hybrid-DRG billing according to the record type HDRG.
