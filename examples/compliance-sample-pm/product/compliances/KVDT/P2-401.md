### Requirement

In the Ersatzverfahren, the default value "00" is transferred for the Besondere Personengruppe.

### Rationale

Since the transfer of the Besondere Personengruppe has been mandatory since July 1, 2018, the user should be supported during entry in the Ersatzverfahren.

If the entry occurs in the Ersatzverfahren after an insurance switch (e.g. because the new eGK has not yet been delivered), the pre-population must -- regardless of the field value at the old insurer -- be set with the default value "00".

### Acceptance Criteria

1. In the Ersatzverfahren, the Besondere Personengruppe (FK 4131) is pre-populated as follows:
   a) If a value already exists for the patient in the software (e.g. from the previous quarter), the input field for the Besondere Personengruppe is pre-populated with the existing value.
   b) If no value exists for the patient in the software, or the patient does not yet exist in the software, or there is a simultaneous insurance switch (see [P2-530](P2-530.md)), then the Besondere Personengruppe is pre-populated with the default value "00" for billing (display in the input field is not mandatory).
2. The user may overwrite the pre-populated value in the Ersatzverfahren.
3. If the user does not enter a Besondere Personengruppe during data entry, the software transfers either preferentially the already known value or the default value "00" in FK 4131 into the billing.
