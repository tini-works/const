### Requirement

If, following a first physician-patient contact on the same treatment day, further physician-patient contacts occur, the user decides according to the EBM provisions whether to perform a **day separation**.

For the "Day Separation" function, the following applies:

1. An automatic "day separation" by the billing system is **not** permitted; rather, the billing system must offer the user an explicit "DAY SEPARATION" function.
2. If a day separation is performed, an additional treatment day with the same date (FK 5000) must be transmitted, containing only the fee schedule numbers rendered during this physician-patient contact.
3. If a day separation is performed, the billing system must ensure that:
   a) for the **first** GNR of the **first** physician-patient contact, a time (FK 5006) can be subsequently entered,
   b) for **further** physician-patient contacts, the specification of a time (FK 5006) for the respective **first** GNR **must** be made.
4. If the KV-specific rule defined via the KV-Spezifika-Stammdatei regarding point 3.a) for day separation is not observed by the user, a **warning** must be issued.
5. If the rule regarding point 3.b) for day separation is not observed by the user, an **error message** must be issued.
