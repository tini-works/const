### Requirement

The software must enable the user to process billing data from different quarters true to the quarter.

### Rationale

Since the deadlines for submitting billing do not fall directly on quarter boundaries, the software must enable the user to process billing data from different quarters true to the quarter.

### Acceptance Criteria

1. The software must provide the ability to bill the previous quarter also in subsequent quarters.
2. If processing of the previous quarter has not yet been fully completed, the software must provide the ability to also process the previous quarter in a subsequent quarter.
3. If data entry for the new quarter takes place while billing for a previous quarter has not yet been closed, the software must keep all data for Satzarten "010x" true to the quarter on a multiple basis. This applies in particular to the date of the last Einlesetag of the insurance card in the quarter (FK 4109).
