### Requirement

The software provides the user with the ability to generate a billing file for Hybrid-DRG billing at any time.

### Rationale

Hybrid-DRG billing is not linked to the quarterly reference period and must therefore be possible at any point in time.

### Acceptance Criteria

1. The software provides the user with the ability to create a billing file at any time.
   a) The user has the ability to select which Hybrid-DRG billing cases should be included in the billing file.
   b) The software validates the generated billing data (record type Hybrid-DRG) against the latest KVDT validation module.
      i. After successful validation, the software encrypts the billing file (Hybrid-DRG) with the "Hybrid-DRG" mode of the XKM in the latest version and with the latest key.
      **Note:** Alternatively, the KBV validation assistant can be used instead of the validation module and the XKM.
   c) The software marks the billed billing cases accordingly in the software.
2. The user must have the ability to view the path of the generated billing file.

### Condition

Implementation of Hybrid-DRG billing according to the record type HDRG.
