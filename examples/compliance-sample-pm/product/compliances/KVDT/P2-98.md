### Requirement

The software must validate date entries entered by the user for correctness. Invalid date entries are rejected by the software system.

### Rationale

Incorrect date entries lead to problems in the billing process.

### Acceptance Criteria

1. The software allows the user to enter only valid date entries.
2. The software transmits the date entered by the user in the billing in the format JJJJMMTT, observing the defined value range in the corresponding fields.
3. Requirements (1) and (2) also apply to the fields FK 4125 (Gültigkeitszeitraum von ... bis) and FK 4233 (Stationäre Behandlung von ... bis...), with the date format being JJJJMMTTJJJJMMTT.
