### Requirement

If in the current quarter the official insured person data of date of birth, gender, name, or address on the insurance card of the same patient change, the software must adopt these changes for billing.

### Rationale

Changes to registration data of insured persons lead to adjustments of the official insured person data on the insurance cards. Either new insurance cards are issued for the insured persons, or the insured person data are updated through the Online-VSDM on the eGK.

### Acceptance Criteria

1. The software must adopt the official fields per Table 4 from the insurance card unchanged.
2. If in the current quarter for the same patient the Besondere Personengruppe ([P2-535](P2-535.md)), the insurer ([P2-530](P2-530.md)), and the Versichertenart ([P2-540](P2-540.md)) remain unchanged, the software must automatically adopt changes to the official fields 3100, 3101, 3102, 3103, 3104, 3107, 3109, 3110, 3112, 3113, 3114, 3115, 3120, 3121, 3122, 3123, and 3124 due to a name, gender, date of birth, or address change upon reading the insurance card into all existing "010x" data records of the patient in the current quarter.
3. If the automatic data adoption per acceptance criterion (2) occurs, the software must not automatically create a new "010x" data record.
4. If the automatic data adoption per acceptance criterion (2) occurs, the software must update the Einlesedatum in all "010x" data records belonging to the insured person in the current quarter.

### Notes
