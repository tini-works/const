### Requirement

The software must display the product name of the connector to the user and transmit this information in the ADT billing.

### Rationale

The Kassenärztliche Vereinigungen can use this information to support practices with appropriate measures.

### Acceptance Criteria

1. The software must adopt the product name of the connector operated in a (subsidiary) practice site into the billing and store it in Field FK 0228 in the practice site data record ("besa") per practice site number or hospital IK in the KVDT file, if this information is retrievable via the external interface.
   a) The most recently available value within the billing quarter must be adopted in daily steps.
   b) If more than one connector is operated in a (subsidiary) practice site, the product name of the connector with the numerically highest product type version value must be adopted.
2. The software does not allow the user to manually enter the product name of the connector.

### Notes

The product name of the connector identifies the name of the product with the product type Konnektor of the telematics infrastructure, which can be retrieved via the external interface of the basic application Dienstverzeichnisdienst [gematik Spezifikation Konnektor]. In the response document of this service, the product name of the connector is contained in the element <ProductName> of the product information, which provides a short description of the connector model and is described using the XML Schema "ProductInformation.xsd" [gematik Übergreifende Spezifikation Operations und Maintenance].

It is possible that more than one connector is present in a practice site. The connector selected for billing transmission should correspond to the one meeting Acceptance Criterion 1 b) of requirement [P2-66](P2-66.md). The display should present the product names of all available connectors clearly.

If the character count of the connector's product name exceeds the maximum length of Field FK 0228, the product name must be split across multiple field instances upon adoption.
