### Requirement

The software supports the user during manual entry or Ersatzverfahren in searching for and selecting the correct cost unit.

### Rationale

Due to the high number of possible cost units for billing, the user must be supported in selecting a cost unit to prevent possible billing problems.

### Acceptance Criteria

1. If **no** IK for identification of a cost unit is available, then:
   a) The user must have the ability to search for a cost unit **at minimum** via:
      - the "Kassensuchname" from the KT-Stammdatei (XML element: `/kostentraeger/bezeichnung/suchname/@V`) **and/or**
      - the "Ortssuchname" from the KT-Stammdatei (XML element: `/kostentraeger/ortssuchname_liste/ortssuchname/@V`) or
      - the 5-digit VKNR.
   b) If the cost unit selected by the user has multiple valid IKs (`/kostentraeger/ik_liste/ik/@V`), the "Abrechnungs-IK" is always to be used for billing and issuance of contractual physician forms (the XML element contains the attribute `R="abrechnungs_ik"`).
   c) If a cost unit is selected by the user, then the rules for verifying billing capability of a cost unit with a KT-Abrechnungsbereich and for processing the Kassenname and VKNR per Kapitel 2.2.2.1 apply.

### Notes

In the data records of the KBV-supplied KT-Stammdatei, special search fields can be added by the software vendor or the practice to additionally facilitate finding a cost unit quickly.
