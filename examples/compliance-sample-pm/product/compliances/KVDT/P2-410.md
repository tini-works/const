### Requirement

The software supports the user during manual entry or Ersatzverfahren in identifying a cost unit in the Kostentraeger-Stammdatei.

### Rationale

This requirement results from § 1 of Anlage 6 BMV-Ae (Vertrag ueber den Datenaustausch auf Datentraegern). The user must be able to identify a cost unit in the Kostentraeger-Stammdatei to verify the billing capability of this cost unit.

### Acceptance Criteria

1. If an IK for identification of a cost unit is available, the software must provide the user the ability to identify a cost unit in the KT-Stammdatei through manual entry of this IK (see also [P2-200](P2-200.md)).
2. The software additionally provides the user with further search criteria such as VKNR, Kassenname, Kassensuchname and/or Ortssuchname for identifying a cost unit in the KT-Stammdatei.
3. If a cost unit could be identified, the software must verify the billing capability of this cost unit with a KT-Abrechnungsbereich and implement all rules and case distinctions per Kapitel 2.2.2.1 for processing the Kassenname and VKNR.
