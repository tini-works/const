## E5: Practice Software Core

**Goal:** Provide core PVS functionality — master data management, system configuration, HPM integration, and practice infrastructure — so that the system meets baseline certification requirements.

**Origin:** AKA Q1-26-1 catalog, sections 3.6 (VSST) and 3.10 (PSDV) — 66 obligations covering practice configuration, doctor/staff management, HPM connectivity, patient master data, and system infrastructure.

**Scope:**
- Practice master data (BSNR, doctors, staff, KV region assignment)
- Doctor qualification and LANR management per contract type
- HPM module integration (connectivity, version management, error handling)
- Patient master data import and validation (eGK, manual entry)
- Contract definition import and quarterly XML update processing
- System configuration per contract and KV region
- User role and permission management
- Audit trail infrastructure for all patient interactions
- Data backup, recovery, and retention policies

**Compliance impact:** BG-1a (AKA Certification), BG-3 (Contract Expansion), BG-4 (Risk Reduction)
