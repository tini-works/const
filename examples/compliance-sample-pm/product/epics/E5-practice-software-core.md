## E5: Practice Software Core

**Goal:** Provide core PVS functionality — system configuration, patient master data, insurance card processing, cost carrier management, and KBV master data files — so that the system meets baseline certification requirements for both AKA and KBV.

**Origin:** AKA Q1-26-1 catalog sections 3.6 (VSST) and 3.10 (PSDV) — 66 obligations covering practice configuration, doctor/staff management, HPM connectivity, patient master data, and system infrastructure. KVDT v6.06 sections 2.1, 2.2, 2.3.13, 2.5.4-2.5.9, Ch.7 — ~119 obligations covering practice infrastructure, insurance card processing, cost carrier resolution, master data file management, and RVSA configuration.

**Scope:**
- Practice master data (BSNR, doctors, staff, KV region assignment)
- Doctor qualification and LANR management per contract type
- HPM module integration (connectivity, version management, error handling)
- Patient master data import and validation (eGK, manual entry)
- Contract definition import and quarterly XML update processing
- System configuration per contract and KV region
- User role and permission management
- Audit trail infrastructure for all patient interactions
- Data backup, recovery, and retention policies
- **System basics** — ICD-10-GM and Formularbedruckung catalog references, data completeness, system date, pre-dating, substitute values (KVDT 2.1.1-2.1.4)
- **User and practice site management** — user/rights management, practice site management, partial billing, LANR uniqueness, BSNR uniqueness, TI connector data transmission (KVDT 2.1.5)
- **AVWG certification number** — capture and transmission of AMV certification number (KVDT 2.1.6)
- **Date handling** — capture of date entries with special field types (KVDT 2.1.8)
- **Fee schedule pre-population** — default Gebührenordnung (KVDT 2.1.9)
- **Insurance card processing** — certified card reader use, eGK read-in, KVK legacy handling, BPol insured data, official field processing, WOP identifier, cost carrier name, read-in date generation and legitimation, cost carrier obligation verification, data adoption into patient master, mobile reader approval number, VSDM update proof (KVDT 2.2.1)
- **Cost carrier resolution** — KT-Stammdatei lookup with 10 case branches (IK valid, fusion, dissolved, invalid, unknown, etc.), temporary records, insured data comparison, Besondere Personengruppe handling (KVDT 2.2.2)
- **Manual patient data entry** — Ersatzverfahren, eEB support, IK search aids, date of birth special ranges, Sonstige Kostenträger, Bundeswehr, PLZ existence check (KVDT 2.2.3-2.2.5)
- **Gender handling** — FK 3110 transformation rules per Technische Anlage 4a (KVDT 2.2.6)
- **Fictitious insured persons** — prevention of billing data adoption for test patients (KVDT 2.2.7)
- **Co-payment exemption** — verification after year change, automatic exemption for children (KVDT 2.2.1.11)
- **Patient data separation** — outpatient vs. inpatient area separation (KVDT 2.2.1.12)
- **Patient search** — search by Versicherten-ID, Versichertennummer, SKT, Geburtsdatum (KVDT 2.3.13)
- **RVSA configuration** — RV certificate management, RV participation determination, pnSD/uu device configuration, facility-specific analyte selection, analyte validation against service documentation, certificate marking per material-analyte-GOP, certificate overview dashboard (KVDT 2.5.4-2.5.9)
- **KT-Stammdatei (SDKT)** — deployment, binding, validity, update, official fields, temporary extension by user and vendor, dummy record VKNR 74799 (KVDT 7.1)
- **KV-Spezifika (SDKV)** — mandatory use, binding, validity, permitted billing areas, Scheinuntergruppe rules, SKT kvx3 handling, insurance card restrictions (KVDT 7.2)
- **Arztverzeichnis (SDAV)** — mandatory use for lab referral quality (KVDT 7.3)
- **PLZ-Stammdatei** — mandatory use, immutability, validity (KVDT 7.4)
- **EBM-Stammdatei (SDEBM/GO)** — mandatory use, validity quarter, modifiability, historization, KV-specific files, display function, real-time checks, final checks, existence check, SDEBM checks, gender reference for undetermined/diverse, billing support for sub-GOPs, change logging, change reversal, storage against rules (KVDT 7.5)
- **OPS-Stammdatei (SDOPS)** — mandatory use, validity, content immutability, existence check, side localization, OPS 5-622.5 special handling (KVDT 7.6)

**Compliance impact:** BG-1a (AKA Certification), BG-1b (KBV KVDT Certification), BG-3 (Contract Expansion), BG-4 (Risk Reduction)
