### User Story

As a **System**, the system must display the connector product name to the user and transmit it in the ADT billing,
so that KVen can identify the connector model in use and provide appropriate support.

### Compliance Context

The connector product name identifies the specific connector model deployed at a practice site. KVen need this information to provide targeted support and track connector deployments. The product name must be automatically retrieved and cannot be manually entered.

### Acceptance Criteria

**Scenario 1: Auto-adopt connector product name**
Given a connector is operated in a (subsidiary) practice site
When the billing quarter progresses
Then the software reads the product name from the connector via the external interface
AND stores it in FK 0228 in the besa data record per BSNR or hospital IK
AND adopts the most recently available value in daily steps
AND for multiple connectors, adopts the product name of the connector with the numerically highest product type version

**Scenario 2: No manual entry of product name**
Given the connector product name field
When the user attempts to edit it
Then the software does not allow manual entry of the connector product name
