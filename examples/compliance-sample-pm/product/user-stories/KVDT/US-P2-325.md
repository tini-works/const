### User Story

As a **Vertragsarzt**, I want **the system to alert me about the restricted benefit entitlement of asylum seekers (Besondere Personengruppe "09") when their eGK is read, without interrupting my workflow**,
so that **I am aware of the restricted scope of services per sections 4 and 6 AsylbLG before providing treatment**.

### Compliance Context

Sections 4 and 6 of the Asylbewerberleistungsgesetz (AsylbLG) regulate the scope of health care services for asylum seekers with less than 15 months of residence. The benefit entitlement differs from statutory health insurance (GKV) patients. Providing services beyond the restricted scope may result in non-reimbursable claims.

### Acceptance Criteria

**Scenario 1: Alert when eGK with Personengruppe "09" is read**
Given an eGK with Besondere Personengruppe "09" is read (after mapping per "Technische Anlage zur Anlage 4a")
When the card data is processed
Then the software must immediately alert the user to observe the restricted benefit entitlement per sections 4 and 6 AsylbLG

**Scenario 2: Alert when data transferred from mobile terminal**
Given data with Besondere Personengruppe "09" is transferred from a mobile card terminal into a PVS
When the data is processed
Then the same alert must be displayed

**Scenario 3: Alert during manual entry**
Given the user manually enters data with Besondere Personengruppe "09"
When the data is entered
Then the user should also be informed analogous to Scenario 1

**Scenario 4: User can deactivate the function**
Given the alert function is active by default
When the user configures the alert settings
Then the user must be able to deactivate the function

**Scenario 5: Non-interrupting notice**
Given the alert is triggered
When the notice is displayed
Then it must not interrupt the physician's workflow
