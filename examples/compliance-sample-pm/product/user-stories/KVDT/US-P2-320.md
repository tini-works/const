### User Story

As the **System**, the system must **restrict and auto-populate the KTAB (FK 4106) based on the Besondere Personengruppe (FK 4131), while allowing the user to change pre-populated values**,
so that **billing uses the correct contractual billing area for special patient groups**.

### Compliance Context

Treatment of patients designated by a Besondere Personengruppe leads to specific remuneration and billing regulations. Incorrect KTAB assignment for these groups results in billing rejections or incorrect reimbursement rates.

### Acceptance Criteria

**Scenario 1: KTAB value restriction based on Besondere Personengruppe**
Given an insurance card is read or data is entered
When FK 4131 (Besondere Personengruppe) is set
Then FK 4106 (KTAB) may only have specific permitted values:
And if FK 4131 = "00", then FK 4106 may be "00", "01", "02", "03", "04", "05", "06", "07", "08" or "09"
And if FK 4131 = "04", then FK 4106 must be "00" or "09"
And if FK 4131 = "06", then FK 4106 must be "02" or "09"
And if FK 4131 = "07" or "08", then FK 4106 must be "01" or "09"

**Scenario 2: Optional automatic pre-population of KTAB**
Given FK 4131 is set
When the system evaluates pre-population rules
Then if FK 4131 = "00", FK 4106 may be pre-populated with "00"
And if FK 4131 = "04", FK 4106 may be pre-populated with "00"
And if FK 4131 = "06", FK 4106 may be pre-populated with "02"
And if FK 4131 = "07" or "08", FK 4106 may be pre-populated with "01"

**Scenario 3: Required automatic pre-population for Personengruppe "09"**
Given FK 4131 = "09"
When the system processes the KTAB
Then automatic pre-population of FK 4106 with content "00" must occur

**Scenario 4: User can change pre-populated KTAB values**
Given FK 4106 has been pre-populated per Scenario 2 or 3
When the user reviews the value
Then the user must have the ability to change the pre-populated value
