### User Story

As a **System**, the system must correctly capture and transform gender data per the "Technische Anlage zur Anlage 4a" rules,
so that field 3110 is always filled accurately regardless of data source (eGK, KVK, or manual entry).

### Compliance Context

The contractual basis (Anlage 4a BMV-Ae and "Technische Anlage zu Anlage 4a") requires specific transformation rules for gender. Incorrect gender coding affects billing accuracy and patient data integrity.

### Acceptance Criteria

**Scenario 1: No default pre-population**
Given the user opens a new patient record
When the gender input field (Feld 3110) is displayed
Then it is not pre-populated with a default value

**Scenario 2: Automated gender suggestion with override**
Given the software determines a gender suggestion from the first name or other characteristics
When the suggestion is displayed
Then the user can always overwrite the suggested value

**Scenario 3: eGK reading — adopt per transformation rules**
Given an eGK is read
When the gender element `..//Geschlecht` is processed
Then the software adopts the content into field 3110 according to the rules of the "Technische Anlage zur Anlage 4a" [KBV_ITA_VGEX_Mapping_KVK]

**Scenario 4: KVK reading — prompt for gender**
Given a KVK is read
When the patient data is processed
Then the software requests the user to enter a gender specification

**Scenario 5: Manual entry or Ersatzverfahren — prompt for gender**
Given manual entry or Ersatzverfahren is used
When the patient data form is displayed
Then the software requests the user to enter a gender specification
AND the captured gender is transferred in field 3110
