### User Story

As the **System**, the system must **prevent capture of pre-dated GNRn and ICD-10-GM codes (dates beyond the system date) and display an error message when such entries are attempted**,
so that **billing data integrity is maintained and future-dated services or diagnoses cannot be recorded**.

### Compliance Context

Pre-dating violates the principles of proper data processing and could enable fraudulent billing. The system must enforce that no services or diagnoses are recorded for dates that have not yet occurred.

### Acceptance Criteria

**Scenario 1: Rejection of pre-dated GNR entry** [derived]
Given the user attempts to capture a GNR with a date beyond the system date
When the entry is submitted
Then the software must display an error message
And reject the entry

**Scenario 2: Rejection of pre-dated ICD-10-GM code entry** [derived]
Given the user attempts to capture an ICD-10-GM code with a date beyond the system date
When the entry is submitted
Then the software must display an error message
And reject the entry
