### User Story

As a **Abrechnungsverantwortliche** (billing manager), I want the system to automatically adopt data from the received digital Form 6 into the billing system faithfully,
so that referral data flows into billing cases without manual re-entry and following the same rules as paper form adoption.

### Compliance Context

Data from received digital referrals must be automatically adopted into the billing system to ensure data fidelity and reduce transcription errors. The adoption must follow the same rules as paper form adoption per KVDT data record description. The user must retain the ability to create new billing cases, edit adopted data, and the system must not generate a reading date for digital form cases.

### Acceptance Criteria

**Scenario 1: Create new billing case from digital form**
Given a digital Form 6 has been read per [K26-07](../../compliances/KVDT/K26-07.md)
When the user processes the received referral
Then the user is given the option to create a new billing case (ADT data package with record type 0102)
AND no reading date is generated in this billing case

**Scenario 2: Automatically adopt insured person and form data**
Given a new billing case is created from a digital form
When data adoption is performed
Then the insured person's data and other data of the digital form are automatically adopted into the billing case
AND the adoption follows the same rules as data adoption for paper forms per KVDT data record description

**Scenario 3: Adopt data without KVDT adoption rule**
Given data from the digital form has no adoption rule in the KVDT data record description
When data adoption is performed
Then the data is automatically adopted on a case-by-case basis at the corresponding location in the system
AND adoption rules from paper form data adoption apply

**Scenario 4: Adopted data is editable**
Given data has been adopted from a digital form
When the user views the adopted data
Then all adopted data are editable by the user at any time
