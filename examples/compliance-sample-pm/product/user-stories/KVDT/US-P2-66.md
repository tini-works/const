### User Story

As a **System**, the system must automatically read and transmit the connector product type version and ePA support status into the ADT billing,
so that KVen can verify connector availability and TI lump sum eligibility per Anlage 32 BMV-Ae and § 341 Abs. 6 SGB V.

### Compliance Context

Reimbursement of TI lump sums by KVen requires proof of connector deployment with a specific product type version. Additionally, per § 341 Abs. 6 SGB V, practices must demonstrate access to electronic patient record components. Without transmitting this data, lump sum reimbursements cannot be verified.

### Acceptance Criteria

**Scenario 1: Auto-adopt connector product type version**
Given a connector is operated in a (subsidiary) practice site
When the billing quarter progresses
Then the software reads the compatibility-relevant components of the product type version
AND stores them in FK 0224 in the besa data record per BSNR or hospital IK
AND adopts the currently available value in daily steps
AND for multiple connectors, adopts the numerically highest product type version

**Scenario 2: No manual entry of product type version**
Given the connector product type version field
When the user attempts to edit it
Then the software does not allow manual entry of the compatibility-relevant components

**Scenario 3: No user modification of TI application values (FK 0225)**
Given values are pre-populated for FK 0225
When the user attempts to modify them
Then the software does not allow modification prior to storing in the besa data record

**Scenario 4: User may modify system support values (FK 0226)**
Given values are pre-populated for FK 0226 (system support/practice equipment)
When the user reviews the values
Then the software allows the user to modify FK 0226 values before storing in the besa data record

**Scenario 5: ePA support detection and user acknowledgment**
Given the software checks ePA support status
When the system determines ePA support capability
Then the user can select between "keine ePA Unterstützung" and "Unterstützung von ePA Stufe 3 (ePA4all)"
AND the currently available information is pre-populated in daily steps
AND if no ePA support exists, the previous quarter's selection is adopted (defaulting to "keine ePA Unterstützung" if unknown)
AND changes from previous quarter values trigger user notification before ADT billing transmission

**Scenario 6: Transmit ePA support values**
Given ePA support values are determined
When the billing is transmitted per BSNR or hospital IK
Then "keine ePA Unterstützung" transmits FK 0225=11, FK 0226=0
AND "Unterstützung von ePA Stufe 3 (ePA4all)" transmits FK 0225=11, FK 0226=1
