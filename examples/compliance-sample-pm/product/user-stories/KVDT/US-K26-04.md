### User Story

As a **Vertragsarzt** (contract physician), I want to send the digital Form 6 via a secure transmission channel,
so that the electronic referral reaches the receiving physician securely and sensitive patient data is protected from unauthorized access.

### Compliance Context

Sensitive patient data in electronic referrals must be transmitted via secure channels to comply with data protection requirements. The BMV-Ae mandates that digital forms use only transmission channels meeting the security requirements of Annex 2b section 3 number 1. The software manufacturer must confirm implementation as part of the supplementary declaration.

### Acceptance Criteria

**Scenario 1: Send digital form via secure channel**
Given a digital Form 6 has been created per [K26-03](../../compliances/KVDT/K26-03.md)
When the user initiates sending
Then the form is sent via a secure transmission channel per [K26-05](../../compliances/KVDT/K26-05.md)

**Scenario 2: Manufacturer confirmation**
Given the sending function is implemented
When the software is evaluated
Then a corresponding implementation confirmation has been provided by the software manufacturer as part of the supplementary declaration
