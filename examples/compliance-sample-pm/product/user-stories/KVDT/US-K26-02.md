### User Story

As a **Vertragsarzt** (contract physician), I want the software to automatically populate the digital Form 6 when creating an electronic referral, with the ability to review and modify the data before population,
so that referral creation is efficient and data integrity is maintained.

### Compliance Context

Automated population of referral forms reduces manual entry errors and supports physician workflow. The physician must be able to review and modify data before population (except health insurance card data). Manual population of the PDF file is prohibited to ensure data consistency and format compliance.

### Acceptance Criteria

**Scenario 1: Auto-populate digital form fields**
Given the physician is creating an electronic referral per [K26-03](../../compliances/KVDT/K26-03.md)
When the referral is initiated
Then the software automatically populates the fields of the digital form

**Scenario 2: Display data for physician review before population**
Given the software is about to auto-populate the digital form
When the data is prepared
Then the data is displayed to the physician with the option to modify it
AND health insurance card data (per Chapter 2.2) is excluded from the modification option

**Scenario 3: Prevent manual PDF population**
Given the digital form PDF file exists
When the user attempts to populate the form
Then the user is not permitted to manually populate the PDF file of the digital form
