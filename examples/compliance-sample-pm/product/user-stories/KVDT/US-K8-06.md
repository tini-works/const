### User Story

As a **Abrechnungsverantwortliche** (billing manager), I want the software to integrate the Hybrid-DRG master data file (SDHDRG) and validate Hybrid-DRG entries against it,
so that practices can bill Hybrid-DRG services correctly with up-to-date master data support.

### Compliance Context

The Hybrid-DRG master data file maps data from Annexes 1 and 2 of the Hybrid-DRG remuneration agreement. Integration of this file supports practices in billing Hybrid-DRG services. The software must validate user-entered Hybrid-DRGs against the master data, warn about invalid entries, keep the file current, and allow users to view its contents and validity status.

### Acceptance Criteria

**Scenario 1: Validate Hybrid-DRG against master data**
Given the user enters a Hybrid-DRG in FK 5027
When the system checks the entry
Then the software verifies whether the Hybrid-DRG exists in `.../leistungsbereich/hybrid_drg_liste/hybrid_drg/@V` of the current master data file
AND if the Hybrid-DRG is not present, the software alerts the user with a warning

**Scenario 2: Ensure current SDHDRG is available**
Given the software has integrated the SDHDRG per [KBV_ITA_VGEX_Schnittstelle_SDHDRG]
When the system operates
Then the respectively current delivered SDHDRG is available in the software at any given time

**Scenario 3: View validity status and data**
Given the SDHDRG is integrated
When the user requests master data information
Then the user can view the validity status of the integrated master data file
AND the user can view the data according to the SDHDRG
