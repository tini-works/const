### User Story

As a **Abrechnungsverantwortliche** (billing manager), I want to generate a combined billing file across multiple practice sites,
so that all sites using the same PVS can submit a unified billing to the KV.

### Compliance Context

Combined billing across multiple practice sites is permitted under specific conditions to simplify the billing process for multi-site practices. The billing file must list all practicing physicians with their LANR in the besa data record. The combined file name must use the practice site number of the sender. Non-compliance could lead to rejected billing submissions.

### Acceptance Criteria

**Scenario 1: Combined billing with shared PVS** [derived]
Given multiple practice sites use the same PVS
When a combined billing is generated
Then all physicians practicing at the relevant practice sites are listed in the besa data record with their respective LANR
AND the billing file name is composed of the practice site number of the billing-generating/consolidating practice site

**Scenario 2: Data assembly method is flexible** [derived]
Given a combined billing is being assembled
When data is sourced from a central database or from individual billing files
Then the combined billing file is valid regardless of the assembly method
