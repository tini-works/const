### User Story

As the **System**, the system must **deploy the OPS master data file delivered in the current quarter starting from the beginning of the following quarter and keep it in use until a new version is available**,
so that **procedure coding always uses the valid version as required by section 295 Abs. 1 SGB V**.

### Compliance Context

Section 295 Abs. 1 SGB V requires coding with the valid version of the procedure code. Deploying the OPS master data at the wrong time -- too early or too late -- means procedures are coded against an incorrect version, risking billing rejections.

### Acceptance Criteria

**Scenario 1: Timely deployment of OPS master data**
Given the KBV has delivered an OPS master data file in the current quarter
When the following quarter begins
Then the software must deploy the new OPS master data file
And the file remains in use as long as a new version is not available

**Scenario 2: User can view master data validity status**
Given the OPS master data file is integrated
When the user requests the validity status
Then the software must display the validity status of the integrated master data file
