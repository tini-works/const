### User Story

As a **System**, The system must prevent users from modifying the data records of the PLZ master data file,
so that the integrity of KBV-provided postal code reference data is preserved.

### Compliance Context

The PLZ master data file from KBV is authoritative for postal code validations. User modifications would undermine the validation integrity, potentially allowing invalid postal codes to pass through and causing billing errors downstream.

### Acceptance Criteria

**Scenario 1: PLZ records are read-only** [derived]
Given the PLZ master data file has been imported
When the user attempts to modify any existing data record
Then the system prevents the modification
And the data records remain unchanged from the KBV delivery
