### User Story

As a **Abrechnungsverantwortliche** (billing manager), I want the system to prevent billing data of fictitious insured persons from being adopted into ADT-Abrechnungsdatensaetze,
so that health insurers are not billed for test patients used in connectivity tests.

### Compliance Context

Health insurers provide fictitious insured person data for TI connectivity tests in production. If services for these fictitious persons are accidentally included in billing submissions, it constitutes fraudulent billing. The software must automatically block this without relying on user confirmation, ensuring clean separation between test and production billing data.

### Acceptance Criteria

**Scenario 1: Mark patient data as fictitious during manual entry**
Given the user is manually entering patient master data
When the user creates or edits a patient record
Then the software provides the ability to mark the data as fictitious
AND data marked as fictitious can use all functions of the software

**Scenario 2: Prevent adoption of fictitious data into billing**
Given services have been captured for a fictitious insured person
When the KV-Abrechnung is generated
Then the software automatically prevents adoption of captured services for fictitious insured persons into the ADT-Abrechnungsdatensaetze
AND no user confirmation is requested for this prevention
