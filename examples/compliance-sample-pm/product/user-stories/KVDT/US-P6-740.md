### User Story

As a **System**, The system must provide EBM historization and quarter-faithful deployment since the EBM master data file itself does not contain historization,
so that previous quarter billing cases use the correct historical EBM data rather than the current quarter's fee schedule.

### Compliance Context

The EBM master data file has no built-in historization. When billing previous quarter cases (Vorquartalsfaelle), the system must apply the EBM data that was valid in that quarter. Using current-quarter data for past cases would result in incorrect fee calculations and billing rejections.

### Acceptance Criteria

**Scenario 1: EBM historization available for previous quarters** [derived]
Given the user is billing a case from a previous quarter
When the system applies EBM data
Then the system uses the EBM data that was valid in the case's quarter, not the current quarter's data

**Scenario 2: Quarter-faithful deployment** [derived]
Given a new EBM master data file is deployed
When the system manages EBM versions
Then the EBM master data file is deployed "quarter-faithfully" with correct quarter associations
