### User Story

As a **Abrechnungsverantwortliche**, I want to manually capture all insured person data when no insurance card is presented (Ersatzverfahren),
so that billing-relevant data is complete and electronically transferable per § 295 Abs. 4 SGB V.

### Compliance Context

When no insurance card can be read (Ersatzverfahren, Auftrags-/Konsiliaruntersuchung, or data capture without personal contact), the software must still enable full capture and transfer of insured person data into billing. Failure to support this violates § 1 of Anlage 6 BMV-Ae and would prevent proper billing for these patient encounters.

### Acceptance Criteria

**Scenario 1: All entry fields provided**
Given the user initiates the Ersatzverfahren or manual data capture
When the entry form is displayed
Then all fields per Table (FK 3100, 3120, 3101, 3102, 3103, 3104, 3105, 3119, 4133, 4110, 4111, 3116, 4132, 4131, 3108, 3110, 3107, 3112, 3113, 3109, 3114, 3121, 3122, 3123, 3124, 4123, 4124, 4126, 4125) are available for input

**Scenario 2: Minimum entries enforced**
Given the user is capturing insured person data manually
When the user attempts to save without the minimum entries (Name FK 3101, Vorname FK 3102, Geburtsdatum FK 3103, Kostentraegerkennung FK 4111, DMP-Kennzeichnung FK 4132, BesonderePersonengruppe FK 4131, Versichertenart FK 3108, Geschlecht FK 3110, PLZ FK 3112, PostfachPLZ FK 3121)
Then the software prevents saving and indicates the missing mandatory fields

**Scenario 3: Non-interrupting notification for optional fields**
Given the user is entering insured person data
When optional fields from the table are not yet filled
Then the software notifies the user that all listed data can and should be captured if available
AND the notification does not interrupt the workflow

**Scenario 4: Transfer to billing**
Given the user has completed manual data capture
When billing is generated
Then the software transfers all captured insured person data per the field table into the billing
