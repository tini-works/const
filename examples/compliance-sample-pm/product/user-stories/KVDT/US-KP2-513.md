### User Story

As a **System**, The system must **propose time-staggered surcharges based on the Vermittlungs-/Kontaktart, Tag der Terminvermittlung, and Leistungstag**,
so that **the correct TSS surcharge is automatically calculated and offered for billing adoption**.

### Compliance Context

Time-staggered surcharges for mediated appointments are differentiated by age class and time limits. Incorrect surcharge assignment leads to billing errors or lost revenue. The system must implement the defined algorithm to ensure compliant and accurate surcharge proposals.

### Acceptance Criteria

**Scenario 1: Surcharge assignment by age class and time limits**
Given the software has access to the GO-Stammdatei for the responsible KV
When a Schein has Vermittlungs-/Kontaktart, Tag der Terminvermittlung, and VP/GP/KP set
Then the software enables automatic assignment of the time-staggered surcharge by age classes and time limits
And the surcharges are defined in the GO-Stammdatei identified by `../kv/kennzeichen/arztpraxis/@V = true`

**Scenario 2: TSS-Akutfall within 1 day**
Given a Schein is marked as TSS-Akutfall (FK 4103 = 2)
And the day of Terminvermittlung (FK 4115) is present
And the VP/GP/KP is set
When FK 5000 minus FK 4115 is less than or equal to 1 calendar day
Then the software proposes surcharge identifier A for adoption

**Scenario 3: TSS-Akutfall exceeding 1 day**
Given a Schein is marked as TSS-Akutfall (FK 4103 = 2)
And the day of Terminvermittlung (FK 4115) is present
And the VP/GP/KP is set
When FK 5000 minus FK 4115 is greater than 1 calendar day
Then the user receives the warning: "The patient was not treated on the current day or following day and therefore cannot be marked as TSS-Akutfall."

**Scenario 4: TSS-Terminfall/HA-Vermittlungsfall/Routine-Termin within 4 days**
Given a Schein is marked as TSS-Terminfall, HA-Vermittlungsfall, or Routine-Termin (FK 4103 = 1, 3, or 6)
And the day of Terminvermittlung (FK 4115) is present
And the VP/GP/KP is set
When FK 5000 minus FK 4115 is less than or equal to 4 calendar days
Then the software proposes surcharge identifier B for adoption

**Scenario 5: TSS-Terminfall/HA-Vermittlungsfall/Routine-Termin 5-14 days**
Given a Schein is marked as TSS-Terminfall, HA-Vermittlungsfall, or Routine-Termin (FK 4103 = 1, 3, or 6)
And FK 4115 is present and VP/GP/KP is set
When FK 5000 minus FK 4115 is greater than 4 and less than or equal to 14 calendar days
Then the software proposes surcharge identifier C for adoption

**Scenario 6: TSS-Terminfall/HA-Vermittlungsfall/Routine-Termin 15-35 days**
Given a Schein is marked as TSS-Terminfall, HA-Vermittlungsfall, or Routine-Termin (FK 4103 = 1, 3, or 6)
And FK 4115 is present and VP/GP/KP is set
When FK 5000 minus FK 4115 is greater than 14 and less than or equal to 35 calendar days
Then the software proposes surcharge identifier D for adoption

**Scenario 7: TSS-Terminfall/HA-Vermittlungsfall/Routine-Termin exceeding 35 days**
Given a Schein is marked as TSS-Terminfall, HA-Vermittlungsfall, or Routine-Termin (FK 4103 = 1, 3, or 6)
And FK 4115 is present and VP/GP/KP is set
When FK 5000 minus FK 4115 is greater than 35 calendar days
Then the user receives the warning: "No time-staggered surcharge can be billed anymore, as the 35-calendar-day deadline has passed."

**Scenario 8: User confirmation and override**
Given the software proposes a time-staggered surcharge
When the user reviews the proposal
Then the software gives the user the ability to confirm the adoption
And the proposed surcharge can always be overridden by the user
