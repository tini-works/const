### User Story

As a **System**, the system must ensure that a (N)BSNR is transmitted only once in the besa data record of ADT, SADT, and KADT billing,
so that redundant BSNR entries do not cause processing errors at the KVen.

### Compliance Context

Since multiple physicians under one (N)BSNR can be listed in a single besa data record, transmitting the same BSNR multiple times is unnecessary and can cause processing errors. The besa data record structure must contain each BSNR exactly once.

### Acceptance Criteria

**Scenario 1: BSNR uniqueness in besa data record**
Given billing data is prepared for ADT, SADT, or KADT
When the billing file is generated
Then a BSNR occurs only once in field code 0201 of the besa data record
