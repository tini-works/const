### User Story

As a **Vertragsarzt**, I want **to create and maintain a Favoritenliste of frequently used physician records from the Kollegensuche**,
so that **I can quickly access essential Terminvermittlung data for recurring referral partners**.

### Compliance Context

The Kollegensuche supports physicians in finding colleagues for appointment mediation. A favorites list reduces repeated searching and ensures that essential referral data is readily available for compliant Terminvermittlung workflows.

### Acceptance Criteria

**Scenario 1: Favoritenliste creation**
Given the Kollegensuche feature is available
When the user chooses to create a Favoritenliste
Then the software enables the user to create a Favoritenliste with access to essential Terminvermittlung-relevant data

**Scenario 2: Marking search results as favorites**
Given the user has search results based on requirement [KP2-507]
When the user marks individual data records as favorites
Then those records are added to the Favoritenliste
