### User Story

As a **Vertragsarzt**, I want **to capture and transmit Melde-ID, Hash-String, and Hash-Wert from the implant register registration confirmation for each implant-related GOP**,
so that **proof of implant register notification is included in the billing per IRegG and IRegBV requirements**.

### Compliance Context

The Implant Register Act (IRegG) and IRegBV require healthcare institutions to report implant-related procedures to the implant register. Medical practices must transmit proof (Melde-ID, Hash-String, Hash-Wert) in billing. Non-compliance violates statutory reporting obligations.

### Acceptance Criteria

**Scenario 1: Service-related capture of implant register data**
Given an implant-related procedure has been registered in the implant register
When the user documents the service
Then the software provides the ability to capture one or more Melde-IDs with respective Hash-Strings and Hash-Wertes on a service-related basis

**Scenario 2: Melde-ID mandatory for configured GOPen**
Given the user is capturing a GOP where `@V="5050"` is stored in the EBM-Stammdatei
When the service is being documented
Then the software requires the "Melde-ID Implantateregister" (FK 5050)

**Scenario 3: Hash-String mandatory for configured GOPen**
Given the user is capturing a GOP where `@V="5051"` is stored in the EBM-Stammdatei
When the service is being documented
Then the software requires the "Hash-String Implantateregister" (FK 5051)

**Scenario 4: Hash-Wert mandatory for configured GOPen**
Given the user is capturing a GOP where `@V="5052"` is stored in the EBM-Stammdatei
When the service is being documented
Then the software requires the "Hash-Wert Implantateregister" (FK 5052)

**Scenario 5: Multiple Melde-IDs transmitted separately**
Given the user captures more than one Melde-ID
When the billing is created
Then each Melde-ID is transmitted in a separate FK 5050
And each Melde-ID includes the child elements Hash-String (FK 5051) and Hash-Wert (FK 5052)

**Scenario 6: Melde-ID transmitted in FK 5050**
Given the user has captured implant register data
When the billing is generated
Then the software transmits the Melde-ID in FK 5050, Hash-String in FK 5051, and Hash-Wert in FK 5052
