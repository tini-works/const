### User Story

As a **Abrechnungsverantwortliche** (billing manager), I want the software to implement TSS-Abrechnungsinformation functions per the Anforderungskatalog Terminservice so that services resulting from Terminvermittlungen are correctly marked and transmitted,
so that separate remuneration for Terminservice-based treatments is properly documented and billed.

### Compliance Context

Treatments arranged through the Terminservice (116117) require separate remuneration marking on billing documents. The Terminservice- und Versorgungsgesetz mandates that billed services be marked by Schein with relevant information transmitted per defined requirements. Non-compliance risks incorrect reimbursement and failed KBV certification.

### Acceptance Criteria

**Scenario 1: Implement TSS-Abrechnungsinformation functions**
Given the software supports Arzt-Patienten-Kontakt
When the user captures patients, manages the practice, or performs billing
Then the software provides TSS-Abrechnungsinformation functions per Kapitel 4 of the Anforderungsdokument "Anforderungskatalog Terminservice" version 3.0.0 [KBV_ITA_VGEX_Anforderungskatalog_116117_TSS]

**Scenario 2: Demonstrate implementation in certification**
Given the software has implemented TSS-Abrechnungsinformation
When the KBV Zertifizierungsverfahren "116117 Terminservice Abrechnungsinformation" is conducted
Then the software demonstrates the implementation of functionalities

**Scenario 3: Submit audit certificate**
Given the software has completed the audit for "116117 Terminservice Abrechnungsinformation" at kv.digital
When the KBV certification process is underway
Then the audit certificate is submitted to KBV within the certification process
