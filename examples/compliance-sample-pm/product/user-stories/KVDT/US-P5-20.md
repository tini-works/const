### User Story

As a **System**, The system must append the communication record (Kommunikationssatz) generated by the KVDT validation module to the KVDT file before encryption,
so that validation run information is included in the billing submission as required by the KVDT specification.

### Compliance Context

The communication record documents the validation run performed by the KVDT validation module. It must be appended to ADT, KADT, and SADT billing files before encryption. Missing communication records cause the billing file to be non-compliant. HDRG data packages are explicitly excluded from this requirement.

### Acceptance Criteria

**Scenario 1: Communication record appended for ADT/KADT/SADT** [derived]
Given the KVDT validation module has validated the billing file and generated a communication record
When the KVDT file is being prepared for encryption (ADT, KADT, or SADT data packages)
Then the communication record is appended to the KVDT file before encryption

**Scenario 2: No communication record for HDRG** [derived]
Given the billing is for an HDRG data package
When the billing file is prepared
Then no communication record is appended to the billing file
