### User Story

As a **Vertragsarzt**, I want to reverse any manual or system-side changes to service documentation,
so that I maintain full control over my service records and can correct unintended modifications.

### Compliance Context

The physician must retain sovereignty over their service documentation. Both manual edits and automatic system modifications (e.g., from billing eligibility checks) must be reversible. An irreversible change would remove the physician's ability to correct errors or exercise clinical judgment on service documentation.

### Acceptance Criteria

**Scenario 1: Manual changes are reversible** [derived]
Given the user has made a manual change to service documentation
When the user decides to reverse the change
Then the system allows the reversal of the manual change

**Scenario 2: System-side changes are reversible** [derived]
Given the system has made an automatic change to service documentation based on billing eligibility checks
When the user decides to reverse the change
Then the system allows the reversal of the system-side change
