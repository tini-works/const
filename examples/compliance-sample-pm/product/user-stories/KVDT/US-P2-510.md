### User Story

As a **Abrechnungsverantwortliche**, I want billing-preparatory functions including trial billings, daily control lists, and step-by-step corrections,
so that I can review and correct billing data before final submission.

### Compliance Context

Billing-preparatory functions are mandatory to ensure the accuracy of submitted billing data. Without trial billings and control lists, errors would only be discovered after submission, leading to rejections and delayed reimbursement.

### Acceptance Criteria

**Scenario 1: Execute trial billing at any time**
Given the user is preparing billing data
When the user requests a trial billing (Probeabrechnung)
Then the system executes a trial billing and presents the results

**Scenario 2: Create daily control lists at any time**
Given the user wants to review daily activity
When the user requests a daily control list (Tageskontrollliste)
Then the system generates a list with physician-patient contacts on a calendar day including patient personal details, services rendered, and diagnoses

**Scenario 3: Step-by-step or quarter-wide correction**
Given errors are identified in billing data
When the user initiates corrections
Then the software allows correcting billing step by step or across the entire quarter as needed
