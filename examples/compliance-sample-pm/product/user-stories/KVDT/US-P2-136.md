### User Story

As the **System**, the system must **correctly transform and adopt the cost carrier name from KVK and eGK cards into field 4134, and ensure it is not used for panel physician form issuance**,
so that **billing data contains the correct cost carrier name per Anlage 4a BMV-Ae while forms use the name from the KT-Stammdatei**.

### Compliance Context

Anlage 4a BMV-Ae and its Technische Anlage define the transformation specifications. The cost carrier name from the insurance card must always be transmitted when reading a card, but must not be used for form issuance -- different name sources apply for forms versus billing.

### Acceptance Criteria

**Scenario 1: Cost carrier name from KVK**
Given a KVK is read
When the software processes the KrankenKassenName field (Objekttag '80')
Then the content must be adopted per the "Technische Anlage zur Anlage 4a" [KBV_ITA_VGEX_Mapping_KVK] into field 4134

**Scenario 2: Cost carrier name from eGK with priority rules**
Given an eGK is read
When the software processes the cost carrier name
Then Priority 1 is the content of `UC_AllgemeineVersicherungsdatenXML/Versicherter/Versicherungsschutz/Kostentraeger/AbrechnenderKostentraeger/Name`
And Priority 2 is the content of `UC_AllgemeineVersicherungsdatenXML/Versicherter/Versicherungsschutz/Kostentraeger/Name`
And the value is adopted into field 4134

**Scenario 3: Cost carrier name not used for form issuance**
Given insurance card data has been read
When panel physician forms are issued
Then the cost carrier name from the insurance card must not be used
And the rules of P2-210, P2-220, P2-260, or P2-270 must be observed

**Scenario 4: Manual entry of cost carrier name**
Given the user needs to enter cost carrier information manually
When the cost carrier name input is available
Then the software must allow manual entry
