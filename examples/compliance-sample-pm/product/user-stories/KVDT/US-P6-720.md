### User Story

As a **Praxisadministrator**, I want to modify or extend the EBM data basis,
so that mid-quarter contract closures or errors in the delivered EBM file can be corrected without waiting for the next quarterly update.

### Compliance Context

Contract closures during a quarter or errors in the delivered EBM master data file may necessitate corrections. Without the ability to extend or modify the EBM data, the practice would be unable to bill newly introduced services or correct erroneous fee data, leading to revenue loss.

### Acceptance Criteria

**Scenario 1: User can modify or extend EBM data**
Given the user needs to correct or extend the EBM data basis
When the user accesses the EBM master data management
Then the software provides the ability to modify or extend the EBM data basis
