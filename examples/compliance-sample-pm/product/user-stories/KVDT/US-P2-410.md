### User Story

As a **Abrechnungsverantwortliche**, I want to identify a cost unit in the Kostentraeger-Stammdatei during manual entry or Ersatzverfahren,
so that I can verify the billing capability of the cost unit per § 1 Anlage 6 BMV-Ae.

### Compliance Context

Per § 1 of Anlage 6 BMV-Ae, the user must be able to identify a cost unit in the Kostentraeger-Stammdatei to verify billing capability. Without this capability, billing to invalid or non-existent cost units could occur, leading to rejected claims.

### Acceptance Criteria

**Scenario 1: Identify cost unit by IK**
Given an IK for identification of a cost unit is available
When the user enters the IK manually
Then the software identifies the cost unit in the KT-Stammdatei (see also P2-200)

**Scenario 2: Additional search criteria provided**
Given the user is searching for a cost unit
When the user uses search criteria
Then the software provides search by VKNR, Kassenname, Kassensuchname, and/or Ortssuchname

**Scenario 3: Verify billing capability after identification**
Given a cost unit has been identified in the KT-Stammdatei
When the identification is complete
Then the software verifies the billing capability of the cost unit with a KT-Abrechnungsbereich
AND implements all rules and case distinctions per Kapitel 2.2.2.1 for processing Kassenname and VKNR
