### User Story

As a **System**, The system must prevent users from modifying the data records of the delivered KV-Specifika master data file,
so that the integrity of KBV-provided KV-specific billing rules is preserved.

### Compliance Context

The KV-Specifika master data file from KBV has binding character. User modifications would compromise the integrity of billing rules, potentially causing incorrect billing behavior and compliance violations across all affected cases.

### Acceptance Criteria

**Scenario 1: KV-Specifika records are read-only** [derived]
Given the KV-Specifika master data file has been imported
When the user attempts to modify any existing data record
Then the system prevents the modification
And the data records remain unchanged from the KBV delivery
