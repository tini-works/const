### User Story

As a **Praxisadministrator**, I want the PVS to manage all physicians per practice site number (BSNR) with their LANR and transmit this in the besa data record,
so that multi-location practices correctly attribute services and billing to the right site and physician.

### Compliance Context

The KVDT data record description requires that all physicians at each BSNR are managed with their LANR and transmitted in the besa data record. For multi-location (mobile or networked) PVS operations, a practice site assignment is also required. Incorrect attribution leads to billing rejections by KVen.

### Acceptance Criteria

**Scenario 1: Manage physicians per BSNR [derived]**
Given multiple physicians practice at a BSNR
When the administrator configures the practice site
Then the PVS stores each physician's LANR associated with the BSNR
AND transmits this information in the besa data record per [KBV_ITA_VGEX_Datensatzbeschreibung_KVDT]

**Scenario 2: Multi-location practice site assignment [derived]**
Given the PVS is operated at multiple locations (mobile or networked)
When a user documents a service
Then the system implements a practice site assignment in addition to the user notification
