### User Story

As a **Vertragsarzt** (contract physician), I want the system to warn me when a GNR is not present in the master data file and allow storage only after explicit confirmation while also permitting addition to the master data,
so that unknown fee schedule numbers are flagged but can still be used when the physician determines they are valid.

### Compliance Context

When a GNR is not found in the EBM master data file, it may indicate a data entry error or a legitimately new/regional code not yet in the master file. The system must warn the user but not block storage outright, since the physician bears ultimate billing responsibility. The ability to add the GNR to the master data supports ongoing data completeness.

### Acceptance Criteria

**Scenario 1: Warn when GNR not in master data file**
Given the user enters a GNR that is not present in the master data file
When the existence check is performed
Then a warning notice is issued

**Scenario 2: Store unknown GNR only with explicit confirmation**
Given a warning has been issued for an unknown GNR
When the user explicitly confirms the entry
Then the GNR is stored in the billing data record

**Scenario 3: Add unknown GNR to master data**
Given a GNR is not present in the master data file
When the user determines it should be added
Then the addition of the GNR to the master data is possible
