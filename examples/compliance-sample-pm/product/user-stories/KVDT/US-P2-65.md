### User Story

As a **Praxisadministrator**, I want to capture, store, and use the pseudo physician number "999999900",
so that billing and form issuance can proceed even when no regular LANR is applicable per KBV guideline § 75 Abs. 7 SGB V.

### Compliance Context

The KBV guideline pursuant to § 75 Absatz 7 SGB V in conjunction with Anlage 6 BMV-Ae requires support for a pseudo physician number for cases where no regular LANR is available. Without this capability, billing and form issuance for these cases would be blocked.

### Acceptance Criteria

**Scenario 1: Capture and store pseudo physician number**
Given the user needs to configure a pseudo physician number
When the user enters "999999900"
Then the software captures and stores the pseudo physician number

**Scenario 2: Transmit pseudo LANR in ADT billing**
Given the pseudo physician number "999999900" is stored
When ADT billing is generated
Then the software transmits the pseudo physician number in fields 4241, 4242, and 5099

**Scenario 3: Use pseudo LANR on panel physician forms**
Given the pseudo physician number is applicable
When a panel physician form is issued
Then the software uses the pseudo physician number on the form
