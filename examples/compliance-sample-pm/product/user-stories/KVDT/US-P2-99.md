### User Story

As a **System**, The system must automatically pre-populate the fee schedule (FK 4121) from cost carrier master data,
so that billing data entries default to the correct fee schedule and reduce user error in the billing process.

### Compliance Context

Incorrect fee schedule assignments cause billing rejections at the KV level. The KVDT specification requires automatic pre-population from the cost carrier master data to minimize human error, as the fee schedule must correspond to the cost carrier against which services are claimed.

### Acceptance Criteria

**Scenario 1: Automatic pre-population of fee schedule from cost carrier**
Given the billing VKNR (FK 4104) has been determined
When the ADT billing data record is being prepared
Then the software automatically pre-populates the fee schedule (FK 4121) with the value of `/kostentraeger/gebuehrenordnung/@V` of the cost carrier whose `/kostentraeger/@V` corresponds to the determined billing VKNR (FK 4104)

**Scenario 2: User can adjust pre-populated fee schedule**
Given the fee schedule (FK 4121) has been pre-populated
When the user reviews the billing data record before finalizing
Then the software allows the user to adjust the pre-populated value of the fee schedule (FK 4121) before adopting the billing data record data into the ADT billing
