### User Story

As a **System**, the system must ensure that once an eGK has been read for a Bundespolizei Heilfuersorge (BPol) insured person, no further outdated KVKs can be read for that person,
so that the BPol eGK rollout is enforced and outdated KVKs are phased out.

### Compliance Context

The cost carrier Bundespolizei Heilfuersorge is replacing old KVKs with eGKs, introducing VKNR 74860 alongside the legacy VKNR 27860. Once a BPol insured person has been read via eGK, the system must reject subsequent KVK reads for that person in the current and all subsequent quarters. VKNR 27860 will be marked invalid in the master data file as of October 1, 2025.

### Acceptance Criteria

**Scenario 1: Block outdated KVK after eGK read for BPol insured**
Given an eGK has been first read for an insured person of BPol (VKNR = 74860)
When a KVK read is subsequently attempted for the same person (also for VKNR 27860)
Then the software rejects the KVK reading in the current quarter and all subsequent quarters
AND the user is informed that the insured person already has an eGK
