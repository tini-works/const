### User Story

As a **System**, The system must validate all date entries for correctness and reject invalid dates,
so that incorrect date entries do not cause problems in the billing process.

### Compliance Context

KVDT requires strict date validation to prevent billing rejections by KVs. Invalid dates in fields like treatment periods or validity ranges (FK 4125, FK 4233) cause downstream processing failures and potential financial loss for the practice.

### Acceptance Criteria

**Scenario 1: User enters a valid date**
Given the user is entering a date in a field with field type "d"
When the user inputs a valid date value
Then the software accepts the date entry

**Scenario 2: Date is transmitted in correct format JJJJMMTT**
Given the user has entered a valid date
When the date is included in the billing
Then the software transmits the date in the format JJJJMMTT, observing the defined value range in the corresponding fields

**Scenario 3: Date range fields FK 4125 and FK 4233 use JJJJMMTTJJJJMMTT format**
Given the user is entering a date in field FK 4125 (Gültigkeitszeitraum von ... bis) or FK 4233 (Stationäre Behandlung von ... bis...)
When the date is captured and transmitted
Then the date format JJJJMMTTJJJJMMTT is used
And the software validates the date for correctness and rejects invalid entries
