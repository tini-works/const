### User Story

As the **System**, the system must **transform KVK data fields into eGK/KVDT-compliant structures per the Mappingtabelle_KVK**,
so that **data read from legacy KVK cards can be correctly processed for billing purposes**.

### Compliance Context

Anlage 4a BMV-Ae and its Technische Anlage define the transformation requirements. Without correct KVK-to-eGK mapping, patient data from legacy cards cannot be used for billing, and downstream mandatory functions that assume a valid eGK data record will fail.

### Acceptance Criteria

**Scenario 1: KVK data transformation per Mappingtabelle**
Given insurance card data has been read from a KVK
When the data is processed for billing
Then the software must implement all required requirements according to the Mappingtabelle_KVK in the current version [KBV_ITA_VGEX_Mapping_KVK]
