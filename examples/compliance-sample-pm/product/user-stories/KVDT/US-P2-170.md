### User Story

As a **Praxisadministrator**, I want **the system to show all changes between newly read insurance card data and existing patient master data before adoption, per the gematik VSDM-A_2538 requirement**,
so that **I can review and confirm updates to patient data before they take effect**.

### Compliance Context

Anlage 4a BMV-Ae, Anhang 1, Nr. 1.5 defines that card data must only be adopted when field values conform to the "Technische Anlage zur Anlage 4a". The delta display requirement (VSDM-A_2538) ensures the user is aware of all changes, preventing unintended data overwrites.

### Acceptance Criteria

**Scenario 1: Delta display before adoption** [derived]
Given billing-relevant insurance card information has been read from an eGK via the connector
And the patient already has existing master data
When the system prepares to adopt the card data
Then the user must be shown all changes in comparison to the existing patient master data (per VSDM-A_2538)

**Scenario 2: Adoption into patient master data** [derived]
Given the user has reviewed the delta display
When the user confirms adoption
Then the insurance card data must be adopted into the patient master data
