### User Story

As a **Vertragsarzt**, I want the system to log and display all changes made to service documentation based on billing eligibility checks,
so that I have full transparency over system-initiated modifications to my service records.

### Compliance Context

When the billing system modifies service documentation based on EBM eligibility checks (e.g., removing ineligible services, adjusting quantities), these changes must be visible to the physician. Without logging and display, the physician cannot verify correctness or contest inappropriate automatic modifications.

### Acceptance Criteria

**Scenario 1: System-side changes are logged and displayed** [derived]
Given the system has made a change to service documentation based on billing eligibility checks
When the change is applied
Then the change is logged
And the change is displayed to the user
