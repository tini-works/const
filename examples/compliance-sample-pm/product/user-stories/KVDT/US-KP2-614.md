### User Story

As a **Abrechnungsverantwortliche**, I want **to specify at least one HGNC gene symbol (FK 5077) and one type of disease (FK 5079) when capturing GOPen 11513[G-alpha] and 11522[G-alpha], with the option for multiple entries**,
so that **the billing justification for these genetic examinations supports both single and multi-gene analyses per EBM requirements**.

### Compliance Context

GOPen 11513 and 11522 may involve multi-gene analyses, requiring the ability to specify multiple HGNC gene symbols and disease types. HGNC gene symbol chains per [KP2-623] can be used. Non-compliance risks billing rejections for these genetic services.

### Acceptance Criteria

**Scenario 1: At least one HGNC gene symbol and type of disease required**
Given the user is capturing GOPen 11513[G-alpha] or 11522[G-alpha]
When the service is being documented
Then the software requires at least one HGNC gene symbol (FK 5077) and one "Type of disease" (FK 5079)

**Scenario 2: Multiple entries allowed**
Given the user is documenting GOPen 11513[G-alpha] or 11522[G-alpha]
When the user needs to specify additional values
Then the user may specify multiple HGNC gene symbols and multiple entries for "Type of disease"
And if multiple entries for "Type of disease" are made, the software transmits field 5079 multiple times in the billing file

**Scenario 3: HGNC gene symbol chains usable**
Given the user needs to enter multiple HGNC gene symbols
When entering data for GOPen 11513[G-alpha] or 11522[G-alpha]
Then HGNC gene symbol chains according to function [KP2-623] can be used
