### User Story

As a **System**, The system must deploy the KV-Specifika master data file update only at the beginning of the following quarter, not during the current billing quarter,
so that billing rules remain stable within a quarter and the correct version is used at the right time.

### Compliance Context

Unlike the cost carrier master data file (which can be deployed immediately), the KV-Specifika master data file must not be deployed before the start of the next quarter. Premature deployment would apply future billing rules to the current quarter, causing incorrect billing calculations and potential rejections.

### Acceptance Criteria

**Scenario 1: SDKV deployed at correct quarter boundary** [derived]
Given a new KV-Specifika master data file has been delivered during the current quarter
When the system manages master data deployment
Then the update version is deployed at the beginning of the following quarter
And it is not deployed earlier than the start of the following quarter
