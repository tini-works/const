### User Story

As a **Vertragsarzt**, I want **the system to automatically generate and transfer the seven participant-count GOPs for approved group therapy procedures into fields 4253/4256**,
so that **the remuneration structure for group therapy is correctly represented based on participant counts**.

### Compliance Context

Group therapy remuneration depends on the number of participants (3-9), creating seven GOPs per procedure. When a short-term or long-term therapy is approved, all seven associated GOPs must be transferred. Manual handling of this is error-prone and risks incomplete billing.

### Acceptance Criteria

**Scenario 1: Automatic GOP generation for approved group therapy**
Given a psychotherapy with GOPs 3550X, 3551X, 3552X, 3553X, 3554X, 3555X, 3570X, or 3571X is approved
When participant counts 3-9 are substituted for X
Then seven GOPs are created and automatically transferred in field 4253 (insured) or field 4256 (accompanying person)

**Scenario 2: User can deactivate automatic transfer**
Given the automatic GOP transfer is active
When the user prefers manual control
Then the user can deactivate the default automatic transfer

**Scenario 3: User can manually select approved GOPs**
Given the seven possible GOPs for a procedure exist
When the user accesses the selection
Then the user can manually select which approved GOPs to transfer

**Scenario 4: User can edit entered GOPs**
Given GOPs have been entered for group therapy
When the user needs to modify them
Then the user can delete or add GOPs
