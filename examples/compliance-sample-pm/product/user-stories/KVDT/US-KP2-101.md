### User Story

As a **System**, the system must reject reading of a Krankenversichertenkarte (KVK) for statutorily insured persons while continuing to support KVK reading for "original" Other Cost Carriers (VKNR serial number digits 3-5 >= 800),
so that the eGK mandate for statutory insurees is enforced while maintaining backward compatibility for non-statutory cost carriers.

### Compliance Context

Since January 1, 2015, only the electronic Gesundheitskarte (eGK) serves as proof of entitlement for statutorily insured persons. The KVK is no longer valid for this group. However, "original" Other Cost Carriers with VKNR digits 3-5 >= 800 and KTAB 00-09 still use KVK. The system must enforce this distinction to prevent invalid proof of entitlement from being processed, while not blocking legitimate KVK usage.

### Acceptance Criteria

**Scenario 1: Reject KVK for statutory insured persons**
Given a KVK is read for a statutorily insured person (digits 3-5 of field 4104 < 800)
When the system processes the card
Then the reading is rejected with an error message
AND no automatic further processing of the rejected data occurs
AND the user can manually enter card data into the PVS at their own discretion per [KP2-102](../../compliances/KVDT/KP2-102.md)

**Scenario 2: Accept KVK for original Other Cost Carriers**
Given a KVK is read for an "original" Other Cost Carrier (digits 3-5 of field 4104 >= 800 and field 4106 KTAB is 00-09)
When the system processes the card
Then the KVK reading and automatic further processing is supported
