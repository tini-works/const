### User Story

As a **Praxisadministrator**, I want a user and rights management system in the PVS,
so that all services within KV billing can be assigned to a service location and a panel physician/psychotherapist.

### Compliance Context

KV billing requires that every service is attributable to a specific service location and physician. Without proper user and rights management, service attribution becomes unreliable, which can lead to billing rejections and compliance violations.

### Acceptance Criteria

**Scenario 1: User management for service attribution [derived]**
Given the PVS is configured for KV billing
When a user documents a service
Then the system assigns the service to a service location and a panel physician/psychotherapist based on the user and rights management

**Scenario 2: Rights-based access control [derived]**
Given multiple users operate the PVS
When a user logs in
Then the system enforces role-based access ensuring services are correctly attributed per KV billing rules
