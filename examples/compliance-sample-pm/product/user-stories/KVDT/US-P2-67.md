### User Story

As a **System**, the system must automatically pre-populate and transmit proof of TI application support and TI component equipment into the ADT billing,
so that KVen can verify TI lump sum eligibility per § 378 Abs. 2 SGB V.

### Compliance Context

For KVen to ensure TI lump sum payments, practices must transmit proof of TI application support (eRezept, NFDM, eMP, KIM, eAU, eArztbrief, eVDGA, TIM) and TI component equipment (Kartenterminal, SMC-B, eHBA) within billing. The gematik confirmation procedure must also be completed. Without this data, lump sum payments cannot be made.

### Acceptance Criteria

**Scenario 1: Auto-populate TI application values (FK 0225)**
Given the software prepares ADT billing
When TI application values are determined
Then FK 0225 is pre-populated with values for eRezept (1), NFDM (3), eMP (4), KIM (5), eAU (6), eArztbrief (7), Kartenterminal (8), SMC-B (9), eHBA (10), eVDGA (12), and TIM (13)
AND the user can review these before adoption into ADT billing

**Scenario 2: Auto-populate system support/equipment status (FK 0226)**
Given the software checks TI application support and TI component equipment
When the check completes
Then FK 0226 is automatically pre-populated based on available information
AND the currently available information is pre-populated in daily steps
AND if the previous quarter had "ja" (1) but current support is absent, "ja" (1) is pre-populated
AND if the previous quarter had "nein" (0) or no value and current support is absent, "nein" (0) is pre-populated
AND changes from previous quarter trigger user notification before ADT billing transmission

**Scenario 3: No user modification of FK 0225**
Given FK 0225 values are pre-populated
When the user reviews the values
Then the software does not allow modification of FK 0225

**Scenario 4: User may modify FK 0226**
Given FK 0226 values are pre-populated
When the user reviews the values
Then the software allows the user to modify FK 0226 before storing in the besa data record

**Scenario 5: ePA support detection and transmission**
Given the software checks ePA support
When ePA support status is determined
Then values for "keine ePA Unterstützung" (FK 0225=11, FK 0226=0) or "Unterstützung von ePA Stufe 3" (FK 0225=11, FK 0226=1) are transmitted per BSNR or hospital IK

**Scenario 6: gematik confirmation procedure**
Given the PVS communicates with the connector
When the confirmation procedure is evaluated
Then the successful completion of the gematik confirmation procedure for VSDM must be demonstrated
AND the confirmation letter must be submitted to KBV as part of "1ClickAbrechnung" certification
