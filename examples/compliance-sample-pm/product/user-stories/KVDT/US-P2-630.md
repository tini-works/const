### User Story

As a **Vertragsarzt**, I want an explicit day separation function for multiple physician-patient contacts on the same day,
so that services from separate contacts are correctly documented per EBM provisions without automatic system interference.

### Compliance Context

EBM provisions govern whether a day separation is required for multiple physician-patient contacts on the same day. Automatic day separation is explicitly prohibited; the physician must decide. Time documentation (FK 5006) is mandatory for further contacts to ensure proper billing of same-day encounters.

### Acceptance Criteria

**Scenario 1: No automatic day separation**
Given multiple physician-patient contacts occur on the same treatment day
When the billing system processes the contacts
Then automatic day separation is not performed
AND an explicit "DAY SEPARATION" function is offered to the user

**Scenario 2: Day separation creates additional treatment day with same date**
Given the user performs a day separation
When the separation is executed
Then an additional treatment day with the same date (FK 5000) is transmitted
AND it contains only the fee schedule numbers rendered during the subsequent physician-patient contact

**Scenario 3: Time entry for first GNR of first contact (optional, KV-specific)**
Given a day separation is performed
When the user reviews the first physician-patient contact
Then the system allows subsequent entry of a time (FK 5006) for the first GNR of the first contact
AND if the KV-specific rule for this is not observed, a warning is issued

**Scenario 4: Mandatory time entry for further contacts**
Given a day separation is performed
When the user documents further physician-patient contacts
Then specification of a time (FK 5006) for the respective first GNR is mandatory
AND if this rule is not observed, an error message is issued
