### User Story

As a **System**, The system must deploy the PLZ master data file update no later than the beginning of the following quarter, though it may be deployed within the current quarter,
so that postal code validations transition to the new version at the correct time.

### Compliance Context

Unlike the KV-Specifika file, the PLZ master data file can be deployed early within the current quarter but must be deployed no later than the next quarter start. The delivered version remains valid until a new version is delivered, ensuring continuous validation coverage.

### Acceptance Criteria

**Scenario 1: PLZ file deployed within allowed timeframe** [derived]
Given a new PLZ master data file has been delivered during the current quarter
When the system manages master data deployment
Then the PLZ file can be deployed in the current quarter
And the deployment must occur no later than the beginning of the following quarter

**Scenario 2: PLZ file remains valid until replaced** [derived]
Given a PLZ master data file version has been deployed
When no new version has been delivered
Then the current version remains valid and in use
