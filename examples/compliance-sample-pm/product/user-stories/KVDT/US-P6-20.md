### User Story

As a **Praxisadministrator**, I want the system to ensure timely deployment of the current valid KT master data file each quarter,
so that billing and contract physician form printing use correct cost carrier data.

### Compliance Context

The cost carrier (KT) master data file is mandatory for quarterly billing and form printing. An outdated KT file causes incorrect billing assignments and form errors. When an update is delivered, it must be deployed no later than the start of the new quarter, even if the previous quarter's billing is not yet complete.

### Acceptance Criteria

**Scenario 1: KT master data file available at quarter start** [derived]
Given a new quarter is beginning
When the user needs to perform billing or print contract physician forms
Then the currently valid KT master data file is available in the billing software

**Scenario 2: KT update deployed at quarter boundary** [derived]
Given an update to the KT master data file has been delivered
When the new quarter begins
Then the new KT master data file is deployed no later than the beginning of the new quarter
And the deployment occurs even if the billing of the previous quarter has not yet been completed
