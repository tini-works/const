### User Story

As the **System**, the system must **validate that a procedure code entered by the user exists in the current OPS master data file before it can be used for billing**,
so that **only valid OPS codes are transmitted and billing rejections due to invalid codes are prevented**.

### Compliance Context

Section 295 Abs. 1 SGB V requires procedure coding with the valid version. Transmitting a non-existent OPS code would cause billing rejection by the KV and potentially delay reimbursement for the practice.

### Acceptance Criteria

**Scenario 1: Procedure code exists in OPS master data**
Given the user enters a procedure code
When the system validates the code against the XML element `../opscode_liste/opscode/@V` of the current OPS master data file
Then the code is accepted if it exists

**Scenario 2: Procedure code does not exist in OPS master data**
Given the user enters a procedure code that is not present in the OPS master data file
When the system validates the code
Then it must be apparent to the user that the procedure code does not exist and may not be used for billing
And the software must prevent the transfer of the non-existent procedure code into the billing file
