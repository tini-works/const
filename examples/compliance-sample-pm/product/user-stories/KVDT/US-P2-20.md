### User Story

As the **System**, the system must **use the operating system date and must not provide any application-level function to modify the system date**,
so that **the principles of proper data processing are upheld and date integrity is maintained for billing**.

### Compliance Context

A modification of the system date violates the principles of proper data processing. Providing date modification functions in the application software could enable any user to manipulate dates, undermining the integrity of read-in dates, billing timestamps, and other date-dependent compliance functions.

### Acceptance Criteria

**Scenario 1: System date provided by operating system** [derived]
Given the billing software is running
When any function requires the current date
Then the system date must be provided by the operating system

**Scenario 2: No application-level date modification function** [derived]
Given the billing software is running
When the user navigates the application
Then no function must exist that enables modification of the system date
