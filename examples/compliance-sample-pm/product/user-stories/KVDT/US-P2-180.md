### User Story

As the **System**, the system must **transmit the approval number of a mobile card terminal in FK 4108 when insurance card data is transferred from a mobile reader, and correctly handle transitions from mobile to stationary reading**,
so that **billing records accurately reflect the card reading device used, per KVDT requirements**.

### Compliance Context

Mobile card terminals have specific identification requirements in the KVDT billing format. The approval number (FK 4108) must be correctly transmitted or suppressed depending on whether a stationary reader subsequently reads the same card. Incorrect handling leads to billing data inconsistencies.

### Acceptance Criteria

**Scenario 1: Mobile reader approval number transmitted**
Given insurance card data is transferred from a mobile reader (return code 9500 or 9501 at RESET_CT)
When the data is stored in the billing system
Then the approval number of the mobile card terminal (tag '92') must be transmitted in FK 4108 of the respective record type 010x

**Scenario 2: Existing approval number overwritten**
Given an approval number is already present in the record
When new data is transferred from a mobile reader
Then the existing approval number is overwritten by the currently transmitted one (only one per data record 010x)

**Scenario 3: Stationary reader overrides mobile reader data**
Given the same insurance card was first read via a mobile reader in the current quarter
When the same card is subsequently read via a stationary reader
Then the software must overwrite the existing read-in date (FK 4109) in the respective record type "010x"
And the software does not transmit field FK 4108 ("Zulassungsnummer") with the billing
