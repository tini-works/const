### User Story

As a **System**, the system must store name and address changes deviating from the insurance card separately and exclude them from billing,
so that billing always uses the official insurance card data while the practice retains current patient contact information.

### Compliance Context

For billing purposes, the official data from the insurance card must be used because insured person data must match the data registered with the health insurers. Transmitting deviating data causes mismatches and potential billing rejections. The practice still needs current contact data for correspondence and medical purposes.

### Acceptance Criteria

**Scenario 1: Separate storage of deviating data**
Given name and/or address changes arise for an insured person in the current quarter that deviate from the insurance card
When the changes are recorded
Then the software stores and manages these deviating name and address data separately from the official card data

**Scenario 2: Exclude deviating data from billing**
Given deviating name and/or address data exists for a patient
When the ADT billing is generated
Then the software does not transfer the deviating data for billing purposes
AND the official data from the insurance card is transferred for billing (see P2-120)
