### User Story

As a **System**, the system must check GNRs present in the EBM master data file using the recommended sequence of Conditions, KV conditions, and Rules, applying the principle that the physician should be billed as favorably as possible,
so that EBM compliance checks are thorough and physician-favorable per the fundamental EBM principle.

### Compliance Context

The EBM master data file contains conditions, KV-specific conditions, and rules for each GNR. The recommended check sequence ensures systematic validation. When multiple rules exist, the EBM principle of most-favorable billing for the physician applies. Supplementary numbers (letter suffixes) inherit the rules of their base 5-digit GNR unless explicitly specified otherwise.

### Acceptance Criteria

**Scenario 1: Apply recommended check sequence** [derived]
Given a GNR exists in the EBM master data file
When the system performs checks
Then the checks follow the sequence: 1) Conditions, 2) KV conditions, 3) Rules

**Scenario 2: Apply most-favorable billing principle** [derived]
Given multiple rules exist for a GNR
When the rules are evaluated
Then the system applies the principle that the physician should be billed as favorably as possible
AND the order of rules is considered relevant
