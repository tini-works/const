### User Story

As a **Abrechnungsverantwortliche**, I want billing to be practice-site-based by default with the KVDT filename formed from the BSNR,
so that billing files are correctly structured per the KVDT data record description.

### Compliance Context

The KVDT data record description mandates that billing is practice-site-based by default, with the billing file named from the practice site number (FK 0201). This ensures correct file identification and processing by KVen. Optional partial billings by additional criteria are permitted but not the default.

### Acceptance Criteria

**Scenario 1: Default practice-site-based billing [derived]**
Given the user initiates billing
When the KVDT billing file is generated
Then the filename is formed from the practice site number (FK 0201, besa data record) per the KVDT data record description

**Scenario 2: Optional partial billing by additional criteria [derived]**
Given the user needs a partial billing
When the user selects additional criteria (e.g. LANR FK 5099, GOPen FK 5001, VKNR+KTAB)
Then the software exports a partial billing based on those criteria
