### User Story

As a **Vertragsarzt**, I want the system to warn me when I record an EBM service requiring ICD justification and no matching diagnosis is present,
so that I am aware of the billing justification requirement without being blocked from documenting the service.

### Compliance Context

Certain EBM services are only billable with justification through specific treatment diagnoses. The GO master file (SDEBM) defines which services require ICD codes. Warning the user supports correct billing without blocking service documentation, as the diagnosis may be added later.

### Acceptance Criteria

**Scenario 1: Warning when required ICD code is missing**
Given the user records an EBM service with a GNR that has an active list of justification-eligible diagnoses in the SDEBM (`../gnr/bedingung/begruendungen_liste/icd_liste` with `@V='true'`)
AND none of the ICD-10-GM codes from the list is recorded in FK 6001 or FK 3673
When the service is documented
Then the software displays a warning that the service is only billable with justification through at least one of the listed treatment diagnoses

**Scenario 2: Allow recording despite missing ICD code**
Given the warning per Scenario 1 has been displayed
When the user chooses to proceed
Then the software allows the user to record the EBM service even without the required ICD-10-GM code in FK 6001 or FK 3673
