### User Story

As a **Abrechnungsverantwortliche**, I want **to specify exactly one HGNC gene symbol (FK 5077), optionally one type of disease (FK 5079), and a valid ICD-10-GM code (FK 6001) when capturing GOPen 19421, 19451, and 19452[G-alpha]**,
so that **the billing justification for these genetic examinations is complete with exactly one gene symbol and disease coding per EBM provisions**.

### Compliance Context

EBM provisions for GOPen 19421, 19451, and 19452 require exactly one HGNC gene symbol and ICD-10-GM coded disease type. The "Type of disease" (FK 5079) is optional but limited to exactly one entry. Non-compliance causes billing rejections.

### Acceptance Criteria

**Scenario 1: Exactly one HGNC gene symbol required**
Given the user is capturing GOPen 19421[G-alpha], 19451[G-alpha], or 19452[G-alpha]
When the service is being documented
Then the software requires exactly one HGNC gene symbol (FK 5077)

**Scenario 2: Type of disease optional, limited to one entry**
Given the user is capturing GOPen 19421[G-alpha], 19451[G-alpha], or 19452[G-alpha]
When the service is being documented
Then the "Type of disease" (FK 5079) does not need to be mandatorily captured
And the user must be given the ability to capture exactly one "Type of disease"
