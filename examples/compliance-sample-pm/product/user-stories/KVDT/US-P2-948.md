### User Story

As a **Abrechnungsverantwortliche**, I want to search for existing patients by insured person's ID, insurance number, SKT additional information, and date of birth,
so that I can identify patients without using names, supporting data protection in KV-physician communication.

### Compliance Context

For data protection reasons, KV-physician communication may use only the insured person's ID instead of patient names. The software must support identifying patients via non-name-based search criteria to comply with these data protection requirements.

### Acceptance Criteria

**Scenario 1: Search by insured person's ID**
Given the user needs to find a patient
When the user searches by Versicherten_ID
Then the software returns matching patient records

**Scenario 2: Search by insurance number**
Given the user needs to find a patient
When the user searches by Versichertennummer
Then the software returns matching patient records

**Scenario 3: Search by SKT additional information**
Given the user needs to find a patient
When the user searches by SKT-Zusatzangaben
Then the software returns matching patient records

**Scenario 4: Search by date of birth**
Given the user needs to find a patient
When the user searches by Geburtsdatum
Then the software returns matching patient records
