### User Story

As a **System**, The system must **set FK 4112 to value 1 when insured person data are adopted from an eEB, and omit FK 4112 otherwise**,
so that **billing records accurately trace the source of insured person data for eEB verification**.

### Compliance Context

FK 4112 provides traceability of the eEB as the data source in billing. Without correct marking, the KV cannot verify whether insurance data originated from an eEB, risking billing rejections and compliance violations.

### Acceptance Criteria

**Scenario 1: eEB data adopted — FK 4112 set**
Given insured person data are adopted from an eEB
When the software creates the billing data record
Then the software marks the respective billing data record with the value 1 in FK 4112

**Scenario 2: eEB data not adopted — FK 4112 omitted**
Given insured person data are not adopted from an eEB
When the software creates the billing data record
Then the software ensures that FK 4112 is not transferred in the respective billing data record
