### User Story

As a **Praxisadministrator** (practice administrator), I want to extend existing KT master data records with new IK numbers and associated KT-Abrechnungsbereiche when a billing-VKNR for an unknown IK is already present in the official KT-Stammdatei,
so that cost carrier assignments remain accurate and up-to-date for billing.

### Compliance Context

The KT-Stammdatei must stay current to ensure correct cost carrier assignment during billing. If unknown IKs cannot be added to existing records, billing data may be routed incorrectly, causing rejected claims or delayed reimbursement. Erroneous extensions must also be reversible to prevent persistent data corruption.

### Acceptance Criteria

**Scenario 1: Add unknown IK to existing KT master record** [derived]
Given a billing-VKNR transmitted by the Kassenaerztliche Vereinigung references an unknown IK
And the billing-VKNR is already present in the official KT-Stammdatei
When the administrator initiates the extension
Then the IK is added to the corresponding master data record together with the associated KT-Abrechnungsbereich

**Scenario 2: Correct erroneous extensions** [derived]
Given an IK was erroneously added to a KT master data record
When the user identifies the error
Then the user can correct or remove the erroneous extension
