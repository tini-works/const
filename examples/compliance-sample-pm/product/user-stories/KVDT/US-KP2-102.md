### User Story

As a **Praxisadministrator** (practice administrator), I want to view the card data of a rejected KVK in a copyable format so that I can manually transfer the data if needed,
so that data from rejected cards is not lost, especially after mobile terminal adoption.

### Compliance Context

When a KVK is rejected per KP2-101, the card data must still be viewable in a copyable format to prevent data loss (e.g., from mobile card terminals). However, automatic adoption into the Ersatzverfahren (substitute procedure) is not permitted, and no Einlesedatum (read-in date) may be generated. This ensures the user can access the data while maintaining the integrity of the entitlement verification process.

### Acceptance Criteria

**Scenario 1: Display rejected card data in copyable format**
Given a KVK has been rejected per [KP2-101](../../compliances/KVDT/KP2-101.md)
When the user requests to view the rejected data
Then the rejected card data is displayed in "copyable" format
AND no automatic adoption into the Ersatzverfahren occurs

**Scenario 2: No read-in date generated**
Given rejected card data is being viewed
When the data is displayed
Then no Einlesedatum per [P2-140](../../compliances/KVDT/P2-140.md) is generated
