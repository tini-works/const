### User Story

As a **Vertragsarzt** (contract physician), I want the system to store the personal data of the patient for pregnancy termination cases when documentation is performed exclusively electronically,
so that the physician's treatment documentation obligation is fulfilled despite the pseudonymization in billing.

### Compliance Context

Even though pregnancy termination billing uses pseudonymized identifiers, the physician is still obligated to maintain treatment documentation that includes patient personal data. When documentation is exclusively electronic in the PVS, the system must store this data to fulfill the physician's documentation duty. Without this, the physician would lack legally required treatment records.

### Acceptance Criteria

**Scenario 1: Store patient personal data for electronic documentation** [derived]
Given a pregnancy termination case is being documented exclusively electronically in the PVS
When the physician documents the treatment
Then the personal data of the patient is stored in the system for the physician's own treatment documentation
