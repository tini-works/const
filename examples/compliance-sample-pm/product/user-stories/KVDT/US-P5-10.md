### User Story

As a **Praxisadministrator**, I want the software to ensure timely availability and correct use of the KVDT validation module (XPM) and KBV crypto module (XKM),
so that billing files are validated for data quality and encrypted with the correct billing keys before submission.

### Compliance Context

KVDT requires all billing files to be validated and encrypted before submission. The validation module ensures data quality, and the crypto module ensures data protection. Using outdated modules or incorrect encryption keys results in billing rejection at the KV level.

### Acceptance Criteria

**Scenario 1: Current KBV crypto module available**
Given a new billing period is approaching
When the user prepares for billing
Then the software ensures the user can use the currently valid version of the KBV crypto module (XKM) in time for billing

**Scenario 2: Correct billing keys used for ADT/KADT/SADT**
Given the user is billing ADT, KADT, and/or SADT data packages
When the billing file is encrypted
Then the valid billing key "Oeffentlich_KV_VXX.pub" (working mode "Abrechnungs_Verschluesselung") is always used

**Scenario 3: Correct billing key used for HDRG**
Given the user is billing the HDRG data package
When the billing file is encrypted
Then the valid billing key "Oeffentlich_HDRG_VXX.pub" (working mode "HDRG_Verschluesselung") is always used

**Scenario 4: Current KVDT validation module available**
Given a new billing period is approaching
When the user prepares for billing
Then the software ensures the user can use the currently valid version of the KVDT validation module (KVDT-XPM) in time for billing
