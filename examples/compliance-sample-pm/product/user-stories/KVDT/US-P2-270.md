### User Story

As an **Abrechnungsverantwortliche**, I want **the system to warn me when an IK from the insurance card is not found in the KT-Stammdatei and allow me to create a temporary master data record or add the IK to an existing record**,
so that **billing can proceed for patients with new or unrecognized cost units**.

### Compliance Context

An unknown IK typically indicates a new cost unit not yet reflected in the KT-Stammdatei. The system must warn the user and facilitate creation of temporary records so billing is not blocked while the KV is contacted for clarification.

### Acceptance Criteria

**Scenario 1: Warning for unknown IK**
Given IK_v from the insurance card does not match any IK under `/kostentraeger/ik_liste/ik/@V` in the KT-Stammdatei
When the system processes the cost unit
Then a system-generated WARNING is issued requesting the user to contact the respective Kassenaerztliche Vereinigung

**Scenario 2: Temporary master data record creation**
Given the unknown IK warning has been displayed
When the user needs to proceed with billing
Then the user can manually create a temporary KT-Stammsatz based on the insurance card and/or KV information
Or add the corresponding IK to an existing master data record
