### User Story

As the **System**, the system must **use the 9-digit Institutionskennzeichen (IK) from the insurance card as the mandatory search key for identifying a cost unit in the KT-Stammdatei**,
so that **cost units are uniquely and correctly identified for billing and form issuance**.

### Compliance Context

The IK is the authoritative identifier for cost units in the German healthcare billing system. Using the VKNR from the KVK for billing is explicitly prohibited. If an AbrechnenderKostentraeger element exists on the eGK, that IK takes precedence. Incorrect cost unit identification leads to billing routed to the wrong insurer.

### Acceptance Criteria

**Scenario 1: IK used as search key regardless of source**
Given an IK is available from the insurance card or manual entry
When the system searches for a cost unit
Then the 9-digit IK must be used as the search key
And if the IK is found in the KT-Stammdatei under `/kostentraeger/ik_liste/ik/@V`, a unique Kostentraeger-Stammsatz must be identifiable

**Scenario 2: VKNR from KVK not used for billing**
Given a KVK has been read
When billing data is prepared
Then the VKNR from the KVK must not be used for billing

**Scenario 3: AbrechnenderKostentraeger IK takes precedence**
Given an eGK contains an element `////AbrechnenderKostentraeger/Kostentraegerkennung`
When the system identifies the cost unit
Then this IK must be used as the mandatory search key
