### User Story

As a **Abrechnungsverantwortliche**, I want **to input, store, and transfer the original Auftragstext from Muster 39 referral forms for cervical carcinoma screening**,
so that **differentiated screening orders (Zytologie, HPV-Test, KO-Test) are correctly coded and transmitted per oKFE-RL**.

### Compliance Context

The organized cervical carcinoma screening program (oKFE-RL) requires differentiated orders for Primärscreening or Abklärungsdiagnostik. The Auftragstext in FK 4205 must encode the combination (e.g. "P-HPV") to ensure correct billing. Incorrect coding risks misclassification of screening procedures.

### Acceptance Criteria

**Scenario 1: Muster 39 Auftragstext capture with structured entry**
Given a Muster 39 (Satzart 0102 / Scheinuntergruppe 21) is captured
When the user enters the Auftragsinformationen manually
Then the software prompts the user to select:
  - Abrechnungsart: P (Primärscreening) or A (Abklärungsdiagnostik)
  - Auftrag: Zyto, HPV, or KoTest
And the software combines both with a hyphen for transfer into FK 4205 (e.g. "P-HPV")

**Scenario 2: Direct value entry allowed**
Given a Muster 39 is being captured
When the user prefers direct entry
Then the software also allows direct entry of P-HPV, P-Zyto, P-KoTest, A-HPV, A-Zyto, or A-KoTest

**Scenario 3: No default value pre-populated**
Given the entry field for Auftragstext is displayed
When the user views the field
Then the software does not pre-populate with a default value (e.g. "Zytologische Untersuchung")
And the billing does not automatically transfer a default value

**Scenario 4: Automatic adoption from LDT 3 import**
Given Auftragsinformationen are imported via Satzart 8215 (LDT 3)
When the import is processed
Then the software may automatically adopt content from FK 8630 and FK 8629 of Obj_0034 combined with a hyphen into FK 4205 and transfer with billing
