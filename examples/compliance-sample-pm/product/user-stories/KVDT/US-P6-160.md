### User Story

As a **System**, The system must not apply record type "kvx3" additional information or restrictions when billing is based on insurance card data,
so that insurance card-based billing for special population groups is not incorrectly blocked by SKT rules.

### Compliance Context

Since 01.04.2000, insurance cards have been introduced for special population groups (e.g., BVG) of "Other Cost Carriers." These cards do not contain SKT additional information like file reference numbers. Applying kvx3 restrictions to insurance card cases would incorrectly block valid billing for these patients.

### Acceptance Criteria

**Scenario 1: Warning displayed only when insurance card data is absent**
Given the billing data meets all conditions: FK 4109 is not present, AND the serial number of VKNR >= 800 (with KTAB = 00-09) or VKNR < 800 with KTAB not equal to 00, AND neither FK 3105 nor FK 3119 is present
When the system evaluates kvx3 requirements
Then the software displays a warning notice for capturing additionally required billing information or restrictions according to record type "kvx3"
