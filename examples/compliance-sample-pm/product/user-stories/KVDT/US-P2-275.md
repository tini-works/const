### User Story

As an **Abrechnungsverantwortliche**, I want **to create, modify, and manage temporary cost unit master data records with all required billing fields, including validation of the Abrechnungs-VKNR**,
so that **billing can proceed for new cost units not yet in the official KT-Stammdatei**.

### Compliance Context

New cost units may appear before the KT-Stammdatei is updated. Without the ability to create temporary records, billing would be blocked until the next master data update. Temporary records must contain all required billing fields and be properly validated to avoid billing errors.

### Acceptance Criteria

**Scenario 1: Required data for temporary cost unit record**
Given a new cost unit needs to be added
When the user creates a temporary KT-Stammsatz
Then the following must be recorded: Abrechnungs-VKNR, KT-Abrechnungsbereich, Kassenname zur Bedruckung, and Gebuehrenordnung alongside the valid IK

**Scenario 2: Searchability of temporary records**
Given a temporary cost unit exists
When the user searches the KT-Stammdatei in the same service quarter
Then the cost unit must be findable by Kassensuchname and Ortssuchname

**Scenario 3: Abrechnungs-VKNR validation**
Given the user manually enters an Abrechnungs-VKNR
When the entry is validated
Then the system must check: length is 5 digits, type is numeric, and format rule 017 is applied

**Scenario 4: Temporary records not auto-assigned across quarters**
Given temporary data records exist from the previous quarter
And the corresponding IKs are not contained in the updated KT-Stammdatei
When the new KT-Stammdatei is loaded
Then the temporary records must not be automatically assigned to the new KT-Stammdatei

**Scenario 5: User can modify temporary records**
Given temporary master data records exist
When the user edits a record
Then the user must have the ability to modify temporary records or amend KTABs

**Scenario 6: Temporarily created IK auto-assignment**
Given a temporarily created IK is not in the subsequent KT-Stammdatei version
When the new version is loaded
Then this IK may be automatically assigned to the new KT-Stammdatei
