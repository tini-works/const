### User Story

As a **Vertragsarzt**, I want **to capture billing justifications (distance, visit location, zone) for visits outside the medical practice**,
so that **I can bill travel allowance for home visits with the required documentation per KV-specific fee schedule regulations**.

### Compliance Context

Travel allowance billing for home visits requires specification of either a zone, one-way distance in kilometers, or the visit location. These fields are subject to regional KV regulatory authority. Without this data, travel allowance claims will be rejected.

### Acceptance Criteria

**Scenario 1: Input fields for visit billing justifications provided**
Given the user is documenting a visit outside the medical practice
When the user accesses the visit billing fields
Then the software provides input fields for:
  - Double kilometers (DKM) — FK 5008
  - Visit location for home visits — FK 5017
  - Zone for visits — FK 5018

**Scenario 2: Visit data transmitted in billing**
Given the user has captured visit justification data
When the billing is created
Then the software transmits the captured data in the respective field identifiers (FK 5008, 5017, 5018)
