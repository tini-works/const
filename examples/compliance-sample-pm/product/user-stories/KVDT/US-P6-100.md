### User Story

As a **Praxisadministrator**, I want the billing software to provide the currently valid KV-Specifika master data file at the beginning of each quarter,
so that all KV-specific billing data is available and the system only exposes data from the responsible KV.

### Compliance Context

The KV-Specifika master data file contains region-specific billing rules and parameters. Each KV has unique requirements, and the system must restrict access to the responsible KV's data only. Using outdated or incorrect KV data leads to billing rejections and potential compliance violations.

### Acceptance Criteria

**Scenario 1: Timely availability of KV-Specifika master data** [derived]
Given a new quarter is beginning
When the user prepares for billing
Then the currently valid KV-Specifika master data file is available for the billing software

**Scenario 2: All KV-specific data accessible** [derived]
Given the KV-Specifika master data file has been loaded
When the user accesses billing data
Then all KV-specific data of the respectively responsible KV is available in the billing system

**Scenario 3: Access restricted to responsible KV** [derived]
Given the user is accessing KV-specific data
When the system displays specifications
Then the billing software only grants the user access to the specifications of the responsible Kassenärztliche Vereinigung
