### User Story

As a **Abrechnungsverantwortliche**, I want to search for and select the correct cost unit even when no IK is available,
so that billing is directed to the correct insurer and claim rejections are avoided.

### Compliance Context

Due to the high number of possible cost units, users must be supported in selecting the correct one during manual entry or Ersatzverfahren. Selecting the wrong cost unit causes billing problems and potential claim rejections.

### Acceptance Criteria

**Scenario 1: Search by Kassensuchname, Ortssuchname, or VKNR**
Given no IK for identification of a cost unit is available
When the user searches for a cost unit
Then the software provides search at minimum via Kassensuchname (`/kostentraeger/bezeichnung/suchname/@V`) and/or Ortssuchname (`/kostentraeger/ortssuchname_liste/ortssuchname/@V`) or the 5-digit VKNR

**Scenario 2: Use Abrechnungs-IK for cost units with multiple IKs**
Given the user selects a cost unit with multiple valid IKs (`/kostentraeger/ik_liste/ik/@V`)
When the cost unit is selected
Then the Abrechnungs-IK (attribute `R="abrechnungs_ik"`) is used for billing and issuance of contractual physician forms

**Scenario 3: Verify billing capability after selection**
Given a cost unit is selected by the user
When selection is confirmed
Then the rules for verifying billing capability with a KT-Abrechnungsbereich and processing Kassenname and VKNR per Kapitel 2.2.2.1 apply
