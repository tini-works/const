### User Story

As a **Praxisadministrator** (practice administrator), I want the software vendor to be able to temporarily extend the KT master data file via software updates with new cost carrier records or new IKs,
so that the practice can bill for newly added cost carriers before the next official KBV master data file release.

### Compliance Context

Between official KBV master data file releases, new cost carriers or IKs may emerge. The software vendor is permitted to temporarily extend the KT master data file via updates to bridge the gap. Without this capability, practices would be unable to bill for patients insured with newly created health insurers until the next official KBV release.

### Acceptance Criteria

**Scenario 1: Add new cost carrier master records via update** [derived]
Given the software vendor provides an update
When the update includes new cost carrier master records
Then the records are added to the user's KT master data file temporarily

**Scenario 2: Add new IKs to existing KT master records via update** [derived]
Given the software vendor provides an update
When the update includes new IKs for an existing KT master record
Then the new IKs are added to the existing record
