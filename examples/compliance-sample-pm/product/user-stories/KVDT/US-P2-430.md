### User Story

As a **System**, the system must accept and process the full value range of the date of birth field (FK 3103),
so that partial or unknown birth dates can be correctly captured and transmitted per KVDT format.

### Compliance Context

KVDT defines a special value range for FK 3103 (Geburtsdatum) that includes partial dates (JJJJMM00, JJJJ0000) and unknown dates (00000000). If the system only accepts standard date formats, valid patient records would be rejected, causing billing failures.

### Acceptance Criteria

**Scenario 1: Full date of birth [derived]**
Given a patient's date of birth is fully known
When the user enters the date in format JJJJMMTT
Then the system accepts and stores the value in FK 3103

**Scenario 2: Partial date of birth with unknown day [derived]**
Given a patient's exact day of birth is unknown
When the user enters the date in format JJJJMM00
Then the system accepts and stores the value in FK 3103

**Scenario 3: Partial date with only year known [derived]**
Given only the patient's birth year is known
When the user enters the date in format JJJJ0000
Then the system accepts and stores the value in FK 3103

**Scenario 4: Completely unknown date of birth [derived]**
Given the patient's date of birth is completely unknown
When the user enters 00000000
Then the system accepts and stores the value in FK 3103
