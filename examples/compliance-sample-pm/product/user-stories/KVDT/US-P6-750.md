### User Story

As a **Vertragsarzt**, I want the system to use my responsible KV's EBM master data file as the primary data basis, with fallback to the nationwide EBM file,
so that regional fee numbers are available while ensuring access to nationally defined services not yet in the regional file.

### Compliance Context

Each KV may have region-specific fee numbers in their EBM file. However, short-term Bewertungsausschuss decisions may appear only in the nationwide EBM file. The system must prioritize the regional file but fall back to the national file to ensure complete service coverage and prevent billing gaps.

### Acceptance Criteria

**Scenario 1: Regional EBM file used as primary basis**
Given the user's responsible KV has a regional EBM master data file
When the system loads the EBM data basis
Then the data of the EBM master data file of the responsible KV is provided as the primary EBM data basis

**Scenario 2: Fallback to nationwide EBM file**
Given the user records a fee number for service documentation
When the fee number does not exist in the regional EBM master data file
Then the software accesses the nationwide unified EBM master data file
