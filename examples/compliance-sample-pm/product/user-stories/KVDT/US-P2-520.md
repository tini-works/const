### User Story

As a **Abrechnungsverantwortliche**, I want the software to handle quarter transitions so that billing data from different quarters is processed true to the quarter,
so that billing deadlines after quarter boundaries do not cause data integrity issues.

### Compliance Context

Billing submission deadlines do not coincide with quarter boundaries, so the software must allow concurrent processing of current and previous quarter data. Failure to keep data quarter-true risks mixing billing periods, leading to rejected or inaccurate submissions.

### Acceptance Criteria

**Scenario 1: Bill previous quarter in subsequent quarters**
Given the billing deadline for the previous quarter has not yet passed
When the user needs to submit billing
Then the software provides the ability to bill the previous quarter also in subsequent quarters

**Scenario 2: Process previous quarter while new quarter is active**
Given processing of the previous quarter is not yet complete
When the user begins data entry for the new quarter
Then the software allows processing the previous quarter concurrently

**Scenario 3: Quarter-true data separation for Satzarten "010x"**
Given data entry for the new quarter occurs while the previous quarter billing is not yet closed
When the software stores Satzarten "010x" data
Then all data is kept quarter-true on a multiple basis
AND the date of the last Einlesetag of the insurance card (FK 4109) is maintained per quarter
