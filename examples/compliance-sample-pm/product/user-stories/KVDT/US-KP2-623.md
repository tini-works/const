### User Story

As a **Vertragsarzt**, I want **to assign defined HGNC gene symbol chains to a GOP during service documentation**,
so that **multiple gene symbols are efficiently transmitted in billing without individual entry for each symbol**.

### Compliance Context

HGNC gene symbol chains reduce the documentation burden for extensive genetic examinations. The system must automatically expand chains into individual FK 5077 fields for billing transmission. Only chains defined per [KP2-622] may be used, ensuring data integrity.

### Acceptance Criteria

**Scenario 1: Only valid chains usable**
Given the user wants to assign an HGNC gene symbol chain to a GOP
When selecting a chain
Then only chains defined according to requirement [KP2-622] may be used

**Scenario 2: Chain assigned to GOP with automatic field expansion**
Given the user assigns an HGNC gene symbol chain to a GOP
When the billing is generated
Then the software automatically transmits each contained HGNC gene symbol in a separate field 5077

**Scenario 3: Additional symbols and chains addable after adoption**
Given an HGNC gene symbol chain has been adopted into service documentation
When the user needs to add more symbols
Then the user can add further HGNC gene symbols per [KP2-621] and/or further HGNC gene symbol chains to the GOP
