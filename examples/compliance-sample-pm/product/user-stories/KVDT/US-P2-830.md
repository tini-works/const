### User Story

As a **Vertragsarzt**, I want the point value for the patient receipt to be auto-populated with the current orientation value and be editable,
so that the estimated physician's fee on the patient receipt is calculated correctly per § 305 (2) SGB V.

### Compliance Context

The point value (Orientierungswert) is the basis for calculating the estimated physician's fee on the patient receipt. It is updated annually by regulation. Pre-populating the current value reduces errors, while editability accommodates practice-specific deviations.

### Acceptance Criteria

**Scenario 1: Auto-populate with current orientation value**
Given the user prepares a patient receipt
When the point value field is displayed
Then it is pre-populated with the currently valid orientation value (e.g. 12.7404 Cent from 01.01.2026)

**Scenario 2: User may change the point value**
Given the point value is pre-populated
When the user needs a practice-specific value
Then the software allows the user to change the content of the point value field
