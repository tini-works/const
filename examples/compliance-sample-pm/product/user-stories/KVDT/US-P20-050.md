### User Story

As a **Vertragsarzt**, I want to mark each RV analyte-GOP combination with a certificate status (present, not present, or pnSD/uu),
so that the ring trial certification status is correctly documented and transmitted in the RVSA data record.

### Compliance Context

Each material-analyte-GOP combination must carry a certificate status for the RVSA data record. The default assignment depends on the pnSD/uu status (FK 0301), and the system must support both bulk and individual certificate marking. Incorrect certificate status leads to non-compliant billing data.

### Acceptance Criteria

**Scenario 1: Certificate status options available** [derived]
Given the user is managing facility-individual RV analyte-GOP combinations
When the user accesses the certificate marking
Then the system offers options: "Certificate present", "Certificate not present", and "pnSD/uu"
And all analyses default to "Certificate present"

**Scenario 2: Bulk marking of certificate status** [derived]
Given the user wants to apply a certificate status to all analyses
When the user selects a bulk action for "Certificate not present" or "pnSD/uu"
Then the status is applied collectively to all analyses with a single action

**Scenario 3: Individual deviation from bulk marking** [derived]
Given a bulk certificate status has been applied
When the user needs to mark individual certificates differently
Then the system allows individual certificate markings that deviate from the overall marking

**Scenario 4: Default values based on pnSD/uu-Info FK 0301** [derived]
Given the pnSD/uu-Info (FK 0301) has been set per [KP20-030](../../compliances/KVDT/KP20-030.md)
When RV certificates (FK 0305) are assigned
Then the default value is: Code 1 (Certificate present) when FK 0301 = 0 (no pnSD/uu)
And Code 2 (pnSD/uu analysis) when FK 0301 = 1 (exclusively pnSD/uu)
And Code 1 (Certificate present) when FK 0301 = 2 (partially pnSD/uu)

**Scenario 5: Certificate markings are persistent and editable** [derived]
Given the user has marked RV certificates
When the user saves the configuration
Then the facility-individual RV certificates are permanently stored and editable
