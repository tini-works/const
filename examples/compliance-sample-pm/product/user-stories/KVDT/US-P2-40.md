### User Story

As a **System**, the system must prevent automatic population of substitute values in mandatory fields without individual user confirmation,
so that substitute values are only used when genuinely no data is available in the practice.

### Compliance Context

KVDT defines substitute values for specific mandatory fields (e.g. Geburtsdatum FK 3103, LANR FK 5099) that may only be transmitted when no actual value exists. Auto-populating these without confirmation risks submitting incorrect billing data, leading to claim rejections and potential audit findings.

### Acceptance Criteria

**Scenario 1: No auto-population of substitute values [derived]**
Given the fields listed in Table 1 (FK 3103, 4219, 4220, 4243, 4241, 4242, 5099, 5077)
When the system prepares data for billing transmission
Then substitute values must not be automatically populated without individual user confirmation

**Scenario 2: User explicitly confirms substitute value [derived]**
Given a mandatory field listed in Table 1 has no actual value in the practice
When the user individually confirms the use of a substitute value
Then the system populates the field with the defined substitute value from Chapter 7 "Feldverzeichnis"
