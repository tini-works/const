### User Story

As a **System**, the system must create a digital Form 6 for doctor-to-doctor referral commissioning according to the provisions of the BMV-Ae, Annex 2b, and the technical handbook for digital preprints,
so that electronic referrals conform to the regulated format and can be processed by receiving systems.

### Compliance Context

The BMV-Ae and its annexes regulate electronic referral commissioning. The technical handbook for digital preprints [KBV_ITA_VGEX_Technisches_Handbuch_DiMus] specifies precise requirements for form field population, file naming, electronic signatures, verification numbers, PDF/A-2a format, and metadata. Non-compliant forms will be rejected by receiving systems.

### Acceptance Criteria

**Scenario 1: Populate required PDF form fields**
Given a digital Form 6 is being created
When the system generates the form
Then all required PDF form fields are populated per Chapter "2.10 Formularfelder" and Chapter "3.1.2 Formularfelder digitales Muster 6"
AND the properties of the PDF form fields conform to the technical handbook provisions

**Scenario 2: Apply correct file naming**
Given a digital Form 6 has been created
When the file is saved
Then the file naming follows Chapter "2.5 Dateinamen"

**Scenario 3: Apply qualified electronic signature**
Given a digital Form 6 requires signing
When the signature is applied
Then the qualified electronic signature conforms to Chapter "2.7 Qualifizierte elektronische Signatur"

**Scenario 4: Apply KBV verification number**
Given a digital Form 6 is being finalized
When the verification number is applied
Then the KBV-Pruefnummer is applied per Chapter "2.10.7 KBV-Pruefnummer"

**Scenario 5: Maintain PDF/A-2a format and metadata**
Given a digital Form 6 is created
When the form file is generated
Then the file format is PDF/A-2a per Chapter "2.8"
AND metadata values per Table 2 of Chapter "2.9 Metadaten" are not altered
AND metadata values per Table 1 of Chapter "2.9 Metadaten" are populated with respective values
