### User Story

As a **Abrechnungsverantwortliche**, I want the DMP-Kennzeichen field to be pre-populated with the correct default in the Ersatzverfahren,
so that the mandatory FK 4132 is always transmitted correctly even without an insurance card.

### Compliance Context

Since July 1, 2018, transmission of DMP-Kennzeichen (FK 4132) is mandatory. In the Ersatzverfahren, users need support to ensure this field is always filled. After an insurance switch, the value must default to "00" regardless of previous insurer data.

### Acceptance Criteria

**Scenario 1: Pre-population with existing value**
Given the Ersatzverfahren is active
AND a value for DMP-Kennzeichen already exists for the patient in the software
When the entry form is displayed
Then FK 4132 is pre-populated with the existing value

**Scenario 2: Pre-population with default "00" for new or unknown patients**
Given the Ersatzverfahren is active
AND no value exists for the patient, or the patient does not yet exist, or there is a simultaneous insurance switch (see P2-530)
When the entry form is displayed
Then FK 4132 is pre-populated with the default value "00" for billing

**Scenario 3: User may overwrite the pre-populated value**
Given the DMP-Kennzeichen is pre-populated in the Ersatzverfahren
When the user enters a different value
Then the user-entered value replaces the pre-populated value

**Scenario 4: Fallback transfer when user does not enter a value**
Given the user does not enter a DMP-Kennzeichen during Ersatzverfahren data entry
When billing is generated
Then the software transfers either the already known value (preferred) or the default value "00" in FK 4132
