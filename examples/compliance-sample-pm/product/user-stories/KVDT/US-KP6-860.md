### User Story

As the **System**, the system must **ensure that the data records of the OPS master data file [SDOPS] cannot be changed in terms of content**,
so that **the integrity of the official procedure code catalog is preserved as required by section 295 Abs. 1 SGB V**.

### Compliance Context

Section 295 Abs. 1 SGB V mandates the use of the official procedure code version. Any alteration of the OPS master data -- whether by user action or during format transformation -- would mean the software no longer uses the valid version, resulting in non-compliant coding and billing.

### Acceptance Criteria

**Scenario 1: User cannot modify OPS master data content**
Given the OPS master data file [SDOPS] is integrated
When the user attempts to modify the data content
Then the software must prevent the modification

**Scenario 2: Format transformation preserves content integrity**
Given the OPS master data file requires transformation into another format
When the transformation is performed
Then the software must ensure no content changes occur during the transformation
