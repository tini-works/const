### User Story

As a **Praxisadministrator**, I want file-system-level access to the unvalidated and unencrypted billing file generated by the PVS,
so that I can use the stand-alone version of the KBV validation module (XPM) to validate billing files externally.

### Compliance Context

When the KBV-provided stand-alone XPM is used instead of an integrated validation module, the user needs direct access to the raw billing file. Without this access, external validation is impossible, and the billing workflow is blocked.

### Acceptance Criteria

**Scenario 1: Unencrypted billing file accessible** [derived]
Given the stand-alone version of the XPM provided by the KBV is used
When the PVS has completed the billing run
Then the user has access at the file system level to the unvalidated and unencrypted billing file generated by the PVS
