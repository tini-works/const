### User Story

As a **System**, The system must validate facility-individual RV analytes by comparing them against documented laboratory services and highlight potentially relevant analytes,
so that the user is guided toward correct analyte selections based on actual service documentation.

### Compliance Context

Color-highlighted validation against actual service documentation helps users identify which RV analytes are genuinely performed at their facility. This visual aid reduces selection errors in the RVSA data record. The highlighting is explicitly not an error indicator but a guidance mechanism.

### Acceptance Criteria

**Scenario 1: Validation against service documentation** [derived]
Given the user has documented facility-individual analytes
When the user requests validation against service documentation
Then the system parses the billing (facility-related) for potential GOPs linked to RV analytes according to key table [S_NVV_RV_ZERTIFIKAT, OID 1.2.276.0.76.3.1.1.5.2.22]
And potentially relevant analytes are highlighted in color, taking into account the previously specified RV materials

**Scenario 2: Color highlighting is informational only** [derived]
Given the system has highlighted potentially relevant analytes
When the user reviews the highlighted results
Then the highlighting represents a visual aid only and is not interpreted as an error detection
