### User Story

As a **Abrechnungsverantwortliche**, I want **to search for and validate HGNC gene symbols from the maintained key table when coding genetic services for billing**,
so that **genetic examinations are coded with standardized HGNC symbols per EBM billing provisions**.

### Compliance Context

EBM billing provisions mandate HGNC gene symbol coding for genetic services. The system must maintain the HGNC key table and validate entries in real-time. Invalid symbols trigger a replacement value (999999) with mandatory free-text gene name. Non-compliance causes billing rejections for genetic services.

### Acceptance Criteria

**Scenario 1: HGNC key table maintained in system**
Given the software is configured for genetic service billing
When the system starts
Then the HGNC key table (OID) data is maintained in the system

**Scenario 2: Search and auto-adopt HGNC gene symbol**
Given the user needs to specify an HGNC gene symbol
When the user searches for a gene symbol
Then the user can search for and automatically adopt the selected value into field 5077

**Scenario 3: Real-time validation — symbol found**
Given the user enters an HGNC gene symbol
When the symbol exists in the HGNC key table (element `/key/@DN`)
Then the system accepts the entry
And the user must not have the ability to populate field 5078

**Scenario 4: Real-time validation — symbol not found**
Given the user enters an HGNC gene symbol
When the symbol does not exist in the HGNC key table
Then the system displays a warning that the entered HGNC gene symbol does not exist in the key table
And the system ensures the replacement value "999999" is transmitted in KVDT field 5077
And the user must provide a free text for the gene name in field 5078 (Gen-Name)

**Scenario 5: HGNC gene symbol transmitted in billing**
Given the user has specified an HGNC gene symbol
When the billing is created
Then the software transmits the HGNC gene symbol in field 5077
And transmits the gene name in field 5078 if a value is present

**Scenario 6: Mandatory HGNC symbol for configured GOPen**
Given the EBM-Stammdatei stores value V="5077" for a GOP under gnr_zusatzangaben
When the user captures such a GOP with HGNC gene symbol as justification type
Then the system requires at least one HGNC gene symbol

**Scenario 7: Key table not extensible**
Given the HGNC key table is loaded
When the contract physician or PVS manufacturer attempts to add entries
Then the system does not allow extension of the key table with additional HGNC symbols
