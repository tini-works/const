### User Story

As a **Abrechnungsverantwortliche**, I want to capture and transmit additional data required by my KV for billing Sonstige Kostentraeger,
so that special cost unit billing complies with regional KV-specific regulations.

### Compliance Context

Billing rules for Sonstige Kostentraeger are not uniform nationwide; each Kassenaerztliche Vereinigung defines its own requirements via the KV-Spezifika-Stammdatei (SDKV, Satzart "kvx3"). Failure to capture and transmit these KV-specific additional data results in rejected claims for these special cost units.

### Acceptance Criteria

**Scenario 1: Capture KV-specific additional data [derived]**
Given a patient is assigned to a Sonstiger Kostentraeger
When the user documents the billing case
Then the software provides entry fields for all additional data required by the responsible KV according to Satzart "kvx3" of the KV-Spezifika-Stammdatei

**Scenario 2: Transmit additional data in billing [derived]**
Given KV-specific additional data has been captured for a Sonstiger Kostentraeger
When the ADT billing is generated
Then the captured additional data is transmitted in the billing file
