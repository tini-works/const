### User Story

As an **Abrechnungsverantwortliche**, I want to add temporary cost carrier master records to the KT master data file when new health insurance establishments appear after the editorial deadline,
so that patients with newly created cost carriers can still be billed without waiting for the next quarterly KT file update.

### Compliance Context

New health insurance establishments can be created after the editorial deadline for the quarterly KT master data file. Without temporary extension capability, patients insured by these new carriers cannot be billed, causing revenue loss and patient care documentation gaps. An IK must not be added to an existing KT master record to prevent data corruption.

### Acceptance Criteria

**Scenario 1: Temporary cost carrier records can be added** [derived]
Given a new health insurance establishment has been created after the KT master data editorial deadline
When the user encounters an IK from an insurance card or substitute procedure
Then new cost carriers can be added as temporary cost carrier master records to the KT master data file

**Scenario 2: IK cannot be added to existing KT records** [derived]
Given an IK needs to be recorded
When the user attempts to add it to an existing KT master record
Then the system prevents the addition of the IK to the existing record
