### User Story

As a **Vertragsarzt** (contract physician), I want to search for other physicians using the KBV Webservice "Kollegensuche",
so that I can efficiently find colleagues for referrals and consultations.

### Compliance Context

The KBV provides the Webservice "Kollegensuche" to support physicians in finding other physicians for referral purposes. Integration of this service streamlines the referral workflow and ensures that physician search data is sourced from an authoritative, up-to-date registry.

### Acceptance Criteria

**Scenario 1: Use Webservice Kollegensuche per specification**
Given the physician needs to search for a colleague
When the physician initiates a search via the Kollegensuche function
Then the software uses the Webservice-Kollegensuche according to [KBV_ITA_VGEX_SST_Kollegensuche] in the current version

**Scenario 2: Fulfill sub-requirements** [derived]
Given the Kollegensuche feature is implemented
When the software is evaluated for compliance
Then the software fulfills requirements [KP2-507](../../compliances/KVDT/KP2-507.md), [KP2-508](../../compliances/KVDT/KP2-508.md), and [KP2-509](../../compliances/KVDT/KP2-509.md)
