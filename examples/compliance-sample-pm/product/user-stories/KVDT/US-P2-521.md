### User Story

As a **Abrechnungsverantwortliche**, I want to incorporate treatment cases from previous quarters (Nachzüglerfälle) into the current quarterly billing,
so that retroactive billing of previously unbilled cases is possible without data loss.

### Compliance Context

Treatment cases from previous quarters that were not billed in time must still be billable. Without this capability, the practice loses revenue for services already rendered, and compliance with retroactive billing requirements is not met.

### Acceptance Criteria

**Scenario 1: Bill Nachzüglerfall in current quarter**
Given the quarter of the treatment case (FK 4101) lies before the current Abrechnungsquartal (FK 9204)
AND the case was not previously billed
When the user initiates billing
Then the software enables the user to bill the case as a Nachzüglerfall in the current quarter
