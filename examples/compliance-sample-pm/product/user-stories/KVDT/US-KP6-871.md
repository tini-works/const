### User Story

As the **System**, the system must **ensure that a documented OPS code is not transmitted without side localization when the OPS master data requires it, and that no side localization is transmitted when the master data forbids it**,
so that **billing data contains correct side localization per section 295 Abs. 1 SGB V and the OPS coding rules**.

### Compliance Context

Section 295 Abs. 1 SGB V mandates correct procedure coding. Transmitting an OPS code without required side localization, or with an unauthorized one, leads to billing rejections. The OPS master data file defines per code whether side localization is required.

### Acceptance Criteria

**Scenario 1: Side localization required (kzseite = "J")**
Given an OPS code to be documented is defined in the OPS master data file [SDOPS] with `../opscode_liste/opscode/kzseite` equal to "J"
When the OPS code is prepared for billing
Then the software must ensure that one side localization is transmitted for the OPS code via FK 5041 (see also KP2-910, 2)

**Scenario 2: Side localization forbidden (kzseite = "N")**
Given an OPS code to be documented is defined in the OPS master data file [SDOPS] with `../opscode_liste/opscode/kzseite` equal to "N"
When the OPS code is prepared for billing
Then the software must ensure that no side localization is transmitted via FK 5041
