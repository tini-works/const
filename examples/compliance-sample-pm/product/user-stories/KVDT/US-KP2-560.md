### User Story

As a **Abrechnungsverantwortliche**, I want **to input, store, and transfer the original Auftragstext from Muster 6 and Muster 10 referral forms**,
so that **the executing physician's scope of services is correctly documented and billed per § 295 Abs. 4 and § 297 Abs. 2 SGB V**.

### Compliance Context

Per KBV Richtlinien and § 295 Abs. 4 SGB V, all billing-relevant data must be electronically transferable. The executing physician may only perform services specified under "Auftrag". Failure to capture and transmit the original Auftragstext in FK 4205 violates billing regulations and can lead to rejected claims.

### Acceptance Criteria

**Scenario 1: Auftragstext requested for Muster 6 Auftragsleistungen**
Given a referral per Muster 6 with marked "Ausführung von Auftragsleistungen" (Satzart 0102 / Scheinuntergruppe 21) is captured
When the user creates the billing record
Then the software requests the user to adopt the original Auftragstext from the "Auftrag" lines

**Scenario 2: Auftragstext requested for Muster 10**
Given a referral per Muster 10 (Satzart 0102 / Scheinuntergruppe 27) is captured
When the user creates the billing record
Then the software requests the user to adopt the original Auftragstext from the "Auftrag" lines

**Scenario 3: Auftragstext transferred in FK 4205**
Given the user has entered the Auftragstext
When the billing is created
Then the software transfers the Auftragstext into field FK 4205

**Scenario 4: No default value pre-populated**
Given the entry field for Auftragstext is displayed
When the user views the field
Then the software does not pre-populate the entry field with a default value (e.g. "Laboruntersuchung")
And the billing does not automatically transfer a default value

**Scenario 5: Auftragstext for deviating Scheinuntergruppen**
Given a referral per Muster 6 with a deviating Scheinuntergruppe is captured
When the user enters any existing Auftragstext
Then the software enables entry and transfers it analogously in FK 4205

**Scenario 6: Automatic transfer from LDT 3 import**
Given Auftragsinformationen are imported via Satzart 8215 "Auftrag" (LDT 3)
When the import is processed
Then the software may automatically transfer the content into field FK 4205 with the billing

**Scenario 7: Selection list support with user control**
Given the software supports selection lists as input aid
When the user accesses the Auftragstext entry field
Then the software enables the user to maintain these selection lists
And the visible entry field is initially empty with no pre-set value
And an entry adopted from a selection list is modifiable by the user at any time
