### User Story

As a **System**, the system must read the data from a received digital Form 6 according to the technical handbook for digital preprints, including signature verification, form field extraction, and metadata reading,
so that referral data can be correctly parsed and used in further processing.

### Compliance Context

Correctly reading the received digital Form 6 is essential for data integrity in the referral workflow. The technical handbook mandates verification of the qualified electronic signature, reading of all required PDF form fields, metadata extraction, and file naming validation. Failure to properly read the form could result in invalid or incomplete referral data entering the system.

### Acceptance Criteria

**Scenario 1: Verify qualified electronic signature**
Given a digital Form 6 has been received per [K26-06](../../compliances/KVDT/K26-06.md)
When the system reads the form
Then the qualified electronic signature is verified per Chapter "2.7 Qualifizierte elektronische Signatur"

**Scenario 2: Read all required PDF form fields**
Given a digital Form 6 has been received
When the system reads the form
Then all required PDF form fields are read per Chapter "2.10 Formularfelder" and Chapter "3.1.2 Formularfelder digitales Muster 6"

**Scenario 3: Read required metadata**
Given a digital Form 6 has been received
When the system reads the form
Then all required metadata are read per Chapter "2.9 Metadaten" and Chapter "3.1.1 Metadaten digitales Muster 6"

**Scenario 4: Observe file naming conventions**
Given a digital Form 6 has been received
When the system reads the form
Then the file naming conventions per Chapter "2.5 Dateinamen" are observed
