### User Story

As a **System**, The system must **require and verify OPS codes as billing justification when OP services from EBM chapters 31.2 or 36.2 are captured**,
so that **outpatient surgical procedures are properly documented and justified per § 295 SGB V since 01.04.2005**.

### Compliance Context

Since 01.04.2005, outpatient surgical procedures must be documented with procedure codes (OPS) for both documentation and billing purposes. Real-time verification against the EBM-Stammdatei ensures correctness. Incorrect or missing OPS codes lead to rejected billing claims for surgical services.

### Acceptance Criteria

**Scenario 1: OPS code required for configured GOPen**
Given the EBM-Stammdatei stores V="5035" for a GOP under gnr_zusatzangaben
When the user captures such a GOP with OPS code as justification type
Then the system requires the user to provide an OPS code

**Scenario 2: OPS code verified and transmitted**
Given the user enters an OPS code
When the code is validated against the EBM-Stammdatei ops_liste
Then the system verifies the OPS code for correctness
And transmits it in the billing file under FK 5035

**Scenario 3: Laterality L or R requested**
Given the user enters an OPS code defined with laterality "L" or "R" in the EBM-Stammdatei
When the entry is processed
Then the system requests the user to specify the laterality
And presents the corresponding lateralizations for selection
And transmits the laterality in FK 5041

**Scenario 4: Laterality P — automatic bilateral transmission**
Given the user enters an OPS code defined with laterality "P" in the EBM-Stammdatei
When the entry is processed
Then the OPS code is transmitted twice (two FK 5035 fields) automatically
And the system specifies lateralization "L" once and "R" once in FK 5041

**Scenario 5: OPS code not assigned to GOP — notice displayed**
Given the user enters an OPS code not assigned to the GOP in the EBM-Stammdatei
When the validation completes
Then the system displays a corresponding notice
And allows adoption of the OPS code(s) into the billing file despite the notice

**Scenario 6: OPS 5-983 suppresses notice**
Given an OPS code is not among those defined for a service number in the EBM-Stammdatei
When additionally OPS code 5-983 is specified
Then the system waives displaying the notice
