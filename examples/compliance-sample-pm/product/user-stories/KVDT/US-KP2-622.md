### User Story

As a **Vertragsarzt**, I want **to create, manage, and use HGNC gene symbol chains (grouped gene symbols) for billing documentation**,
so that **the effort for documenting extensive genetic examinations is reduced through reusable gene symbol groups**.

### Compliance Context

Extensive genetic examinations may require many individual HGNC gene symbols. Gene symbol chains reduce documentation effort and error risk. Chains must use only valid symbols from the current HGNC key table, and must be validated on quarter changes or key table updates.

### Acceptance Criteria

**Scenario 1: Unique identifier for each chain**
Given the user creates an HGNC gene symbol chain
When the chain is saved
Then each chain has a unique identifier

**Scenario 2: Patient-independent storage**
Given HGNC gene symbol chains are created
When stored in the system
Then the chains are stored patient-independently

**Scenario 3: Real-time validation during chain creation**
Given the user creates or edits an HGNC gene symbol chain
When HGNC gene symbols are entered or selected
Then the software checks each symbol for existence in the HGNC key table (element `/key/@DN`) in real-time

**Scenario 4: Display all defined chains**
Given HGNC gene symbol chains exist in the system
When the user accesses the chain management
Then the user can display all defined HGNC gene symbol chains

**Scenario 5: Edit self-defined chains**
Given the user has defined HGNC gene symbol chains
When the user edits a chain
Then the user can add further HGNC gene symbols, remove symbols from a chain, or remove the entire chain

**Scenario 6: Automatic validation on quarter change or key table import**
Given a quarter change occurs or a new HGNC key table is imported
When the system processes the change
Then the software automatically verifies all defined chains for valid HGNC gene symbols
And when individual symbols are no longer valid, the user is prompted to update the chain
