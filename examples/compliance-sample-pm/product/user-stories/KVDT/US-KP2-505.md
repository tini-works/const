### User Story

As a **Vertragsarzt**, I want **the software to implement the 116117 Terminservice Vermittlungscode functions and provide issuance of Überweisungen (Muster 6 and PTV 11)**,
so that **appointment mediation through Terminservice is properly documented and billed per the KBV Anforderungskatalog**.

### Compliance Context

The Terminservice requires standardized handling of Vermittlungscodes for separate remuneration of mediated appointments. The software must pass the kv.digital audit and submit the certificate to KBV. Non-compliance blocks certification and prevents practices from billing TSS-related surcharges.

### Acceptance Criteria

**Scenario 1: 116117 TSS functions implemented**
Given the software is certified for KVDT billing
When the user accesses Terminservice functions
Then the software implements the functions per Kapiteln 2 and 3 of the Anforderungskatalog Terminservice version 3.0.0 [KBV_ITA_VGEX_Anforderungskatalog_116117_TSS]
And provides the user with issuance of Überweisungen (Muster 6 and PTV 11)

**Scenario 2: Audit certification completed**
Given the software implements 116117 Terminservice Vermittlungscode functions
When the certification process is conducted
Then the software must have successfully completed the audit for the application "116117 Terminservice Vermittlungscode" at kv.digital
And the audit certificate must be submitted to KBV within the certification process
