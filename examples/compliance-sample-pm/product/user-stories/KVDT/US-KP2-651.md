### User Story

As a **Vertragsarzt**, I want **to capture and transmit the batch number (FK 5010) for every SARS-CoV-2 vaccination GOP**,
so that **vaccination batch traceability is ensured for reporting to the Robert Koch Institute per § 13 Abs. 5 Nr. 10 IfSG**.

### Compliance Context

The Infection Protection Act (IfSG) § 13 Abs. 5 Nr. 10 requires the batch number of every SARS-CoV-2 vaccination to be transmitted from the KV to the Robert Koch Institute. Without batch number capture, the practice violates statutory reporting obligations.

### Acceptance Criteria

**Scenario 1: Batch number mandatory for configured GOPen**
Given the user is capturing a GOP within service documentation
When the EBM-Stammdatei stores `@V="5010"` for the GOP under gnr_zusatzangaben
Then the software requires the user to provide the "Batch number" (FK 5010)

**Scenario 2: Batch number optionally capturable for other GOPen**
Given the user is capturing a GOP not listed under acceptance criterion 1
When the user wants to record a batch number
Then the software allows the user to capture the "Batch number" (FK 5010)
