### User Story

As a **System**, the system must automatically create a new "010x" data record when a patient's Besondere Personengruppe changes within the current quarter,
so that billing data is correctly separated for specific remuneration and billing regulations.

### Compliance Context

Membership in a Besondere Personengruppe (FK 4131) leads to specific remuneration and billing regulations from the point the membership becomes known. Without separating billing data, services could be remunerated under incorrect rules.

### Acceptance Criteria

**Scenario 1: New 010x record on Besondere Personengruppe change**
Given the Besondere Personengruppe (FK 4131) of a patient changes in the current quarter
When the change is detected
Then the software generates an additional "010x" data record

**Scenario 2: Preserve existing Einlesedatum**
Given a new 010x data record is created due to the change of Besondere Personengruppe
When the new record is generated
Then no changes to the already existing Einlesedatum (FK 4109) of previous records take place
AND the Einlesedatum is updated only for the newest data record
AND the Einlesedatum is stored and transmitted "true to data record" (see P2-150)
