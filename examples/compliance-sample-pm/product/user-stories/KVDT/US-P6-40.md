### User Story

As a **System**, The system must protect official KT master data fields from user modification while allowing address field changes with proper tracking,
so that the integrity of cost carrier reference data is maintained while permitting necessary address corrections.

### Compliance Context

Certain KT master data fields have "official" character and must remain immutable to prevent billing errors from user modifications. Address fields, however, may need correction by the practice. The system must track such changes and allow reverting to original KBV data to ensure data consistency across updates.

### Acceptance Criteria

**Scenario 1: Official fields are read-only** [derived]
Given the KT master data file has been imported
When the user views cost carrier data
Then the fields `/kostentraeger/@V`, `/kostentraeger/kt_abrechnungsbereich_liste/kt_abrechnungsbereich/bedruckungsname/@V`, `/kostentraeger/ik_liste/ik/@V`, `/kostentraeger/gebuehrenordnung/@V`, `/kostentraeger/kt_abrechnungsbereich_liste/kt_abrechnungsbereich/@V`, `/kostentraeger/bezeichnung/suchname/@V`, `/kostentraeger/ortssuchname_liste/ortssuchname/@V`, `/kostentraeger/gueltigkeit/@V`, `/kostentraeger/ik_liste/ik/gueltigkeit/@V`, `/kostentraeger/existenzbeendigung/aufnehmender_kostentraeger/@V`, `/kostentraeger/unz_kv_geltungsbereich_liste/unz_kv_geltungsbereich/@V`, and `/kostentraeger/kt_abrechnungsbereich_liste/kt_abrechnungsbereich/gueltigkeit/@V` are display-only and not changeable by the user

**Scenario 2: Address fields are editable with tracking** [derived]
Given the user needs to correct a cost carrier address
When the user modifies address fields (`/kostentraeger/adresse_liste/adresse/*`)
Then the software allows the change while observing format specifications
And the software marks the cost carrier as manually corrected
And the user has the option to restore the original data according to the KT master data file

**Scenario 3: Manual address corrections survive KT updates** [derived]
Given the user has manually corrected an address
When a new KT master data file is imported
Then address changes not included in the new version are retained
