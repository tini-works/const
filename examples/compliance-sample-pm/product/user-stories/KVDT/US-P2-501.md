### User Story

As a **Abrechnungsverantwortliche**, I want to create additional arbitrary 010x data records for the same patient in the current quarter,
so that treatments from Terminvermittlungen or special constellations (e.g. open consultation hours) per § 87a Abs. 3 Satz 5 SGB V can be separately marked and remunerated.

### Compliance Context

The TSVG and GKV-FinStG require separate remuneration for treatments resulting from appointment services or special constellations. Without the ability to create additional 010x records per patient per quarter, these treatments cannot be properly marked and billed.

### Acceptance Criteria

**Scenario 1: Create additional 010x data records**
Given a patient already has a 010x data record in the current quarter
When the user needs to document additional treatments (e.g. Terminvermittlung)
Then the user can create additional arbitrary 010x data records with the same and/or different Satzart, Scheinuntergruppe

**Scenario 2: Update Einlesedatum on card re-read**
Given the insurance card is re-read for a patient with multiple 010x data records
When the card data is processed
Then an already existing Einlesedatum in all 010x data records of the current quarter is updated (see P2-150)
