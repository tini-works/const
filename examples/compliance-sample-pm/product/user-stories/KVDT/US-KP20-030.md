### User Story

As a **Praxisadministrator**, I want **a configuration option to document whether unit-use reagents (pnSD/uu) are used at the practice (no, exclusively, or partially), with the setting permanently stored and editable**,
so that **the RVSA data record correctly reflects pnSD/uu usage and billing data is compliant**.

### Compliance Context

KVDT section 2.5.6 mandates that practices using specialist groups potentially employing pnSD/uu must document their reagent usage. If not implemented, FK 0301 defaults to "0" in the RVSA data record, potentially misrepresenting practice capabilities and causing billing discrepancies.

### Acceptance Criteria

**Scenario 1: Configuration dialog for pnSD/uu usage** [derived]
Given the billing software is used by specialist groups that could potentially use pnSD/uu
When the administrator accesses the pnSD/uu configuration
Then a dialog/configuration option must exist to document usage of unit-use reagents
And the options must include: no, exclusively, partially
And the setting must be permanently stored and editable

**Scenario 2: Default RVSA transmission when function not implemented** [derived]
Given the software does not implement the pnSD/uu function (e.g., pure LIS system)
When the RVSA data record is generated
Then FK 0301 is transmitted with the content "0"
