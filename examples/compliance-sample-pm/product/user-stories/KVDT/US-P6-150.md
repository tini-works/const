### User Story

As an **Abrechnungsverantwortliche**, I want the system to prompt me for SKT additional billing information (FKs 9402, 9403, 9404) without blocking billing when the data is unavailable,
so that I can complete billing even when optional SKT data is not present at the practice.

### Compliance Context

The additional SKT billing information defined in record type "kvx3" is only required when actually available at the practice (e.g., printed on paper billing certificates). The system must warn but never block billing over missing SKT data, and must never generate or transmit substitute values. Blocking billing would prevent timely submission.

### Acceptance Criteria

**Scenario 1: System warning for missing SKT additional information** [derived]
Given an SKT case is being billed
When the additional billing information per FKs 9402, 9403, 9404 of record type "kvx3" has not been entered
Then the system prompts the user with a warning to enter the additional billing information
And further processing is possible at any time after the warning

**Scenario 2: No billing-preventing error for missing SKT data** [derived]
Given the SKT additional information is not available at the practice
When the user proceeds past the warning
Then the system does not generate a billing-preventing error message
And the system does not generate or transmit undefined substitute values
