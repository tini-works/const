### User Story

As a **Abrechnungsverantwortliche**, I want **to specify exactly one type of disease (FK 5079) when capturing GOP 11233[G-alpha]**,
so that **the billing justification for genetic examination GOP 11233 is complete and transparent per EBM provisions**.

### Compliance Context

EBM billing provisions for GOP 11233[G-alpha] require the specification of the type of disease. Without this mandatory field, billing for this genetic examination will be rejected. FK 5077 must not be transmitted for this GOP.

### Acceptance Criteria

**Scenario 1: Exactly one type of disease required**
Given the user is capturing GOP 11233[G-alpha] within service documentation
When the service is being documented
Then the software requires the user to provide exactly one entry for "Type of disease" (FK 5079)

**Scenario 2: Type of disease transmitted in billing**
Given the user has provided the type of disease for GOP 11233[G-alpha]
When the billing is created
Then the software transmits the "Type of disease" in field FK 5079

**Scenario 3: FK 5077 not transmitted**
Given GOP 11233[G-alpha] is being billed
When the billing file is generated
Then the software does not transmit field FK 5077
