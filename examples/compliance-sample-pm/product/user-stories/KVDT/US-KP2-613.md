### User Story

As a **Abrechnungsverantwortliche**, I want **to specify exactly one HGNC gene symbol (FK 5077) and one type of disease (FK 5079) when capturing GOPen 11511, 11512, 11516-11518, and 11521[G-alpha]**,
so that **the billing justification for these genetic examinations includes both the examination type and disease type as required by EBM**.

### Compliance Context

EBM billing provisions for GOPen 11511, 11512, 11516-11518, and 11521 require both the HGNC gene symbol and type of disease. The user must not be able to specify multiple values for either field per service number. Non-compliance leads to billing rejections.

### Acceptance Criteria

**Scenario 1: Exactly one HGNC gene symbol and one type of disease required**
Given the user is capturing GOPen 11511[G-alpha], 11512[G-alpha], 11516[G-alpha] to 11518[G-alpha], or 11521[G-alpha]
When the service is being documented
Then the software requires exactly one entry of an HGNC gene symbol (FK 5077) and one "Type of disease" (FK 5079)
And the user must not have the ability to specify multiple HGNC gene symbols or multiple entries for "Type of disease" for a single service number
