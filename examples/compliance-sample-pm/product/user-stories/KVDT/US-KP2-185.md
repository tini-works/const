### User Story

As a **System**, the system must adopt the verification proof (Pruefungsnachweis) from the eGK after a VSDM online update and transmit it correctly in the ADT billing data,
so that the practice demonstrates compliance with the online verification obligation per SGB V and the verification proof is included in KV billing submissions.

### Compliance Context

Section 291b Abs. 2 SGB V requires proof of performing online eGK updates. Section 295 SGB V mandates transmission of the verification proof with billing documents to the KV. The proof includes timestamp (FK 3010), result (FK 3011), error code (FK 3012), and verification digit (FK 3013). Complex rules govern when to update, overwrite, or skip the proof based on status codes 1-6. The gematik confirmation procedure for VSDM must be completed.

### Acceptance Criteria

**Scenario 1: Adopt verification proof on eGK read with online verification**
Given an eGK is read and an online verification was performed (current EF.PN file present)
When the eGK data is processed
Then the verification proof is adopted into the billing and stored
AND the "Pruefnachweis lesen" flag was set during connector communication

**Scenario 2: No verification proof from mobile card terminals**
Given data is adopted from a mobile card terminal
When the data is processed
Then the software does not transmit the verification proof

**Scenario 3: Handle status 1 or 2 during first read**
Given during the first reading in the quarter a verification proof with status 1 or 2 is returned
When subsequent reading processes occur in the quarter
Then a VSD online verification and update is performed
AND no further verification proof (FK 3010, 3011, 3012, 3013) is adopted into the billing

**Scenario 4: Handle status 3-6 with subsequent correction**
Given during the first reading in the quarter a verification proof with status 3 to 6 is returned
When the next reading process returns a verification proof with status 1 or 2
Then the already existing verification proof (FK 3010, 3011, 3012, 3013) is overwritten in the current quarter

**Scenario 5: Transmit verification proof in billing records**
Given a verification proof has been obtained
When the billing data is transmitted
Then the proof is transmitted in data record "010x" with:
AND FK 3010 contains the content of `/PN/TS` (timestamp)
AND FK 3011 contains the content of `/PN/E` (result)
AND FK 3012 contains the content of `/PN/EC` (error code)
AND FK 3013 contains the content of `/PN/PZ` (verification digit, base64-encoded without CR/LF line breaks)

**Scenario 6: Apply verification proof per patient across all records**
Given a verification proof has been obtained per acceptance criteria 3 and 4
When the billing data is prepared
Then the last requested verification proof is adopted per patient in all data records "010x" of the current quarter
AND the software does not allow manual entry of this data
